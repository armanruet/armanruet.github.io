        <span class="badge badge-green">01</span>
        <h2 style="margin:0;font-size:24px;">Lambda Functions — Complete Reference</h2>
      </div>

      <div class="callout callout-purple">
        <p><strong>Functional Purity & Mathematics:</strong> The concept of lambda originates from <em>Lambda
            Calculus</em>, developed by Alonzo Church. In functional programming, lambdas are preferred because they
          discourage state mutations and side effects, making algorithms completely deterministic and mathematically
          provable.</p>
      </div>
      <p>A <strong>lambda</strong> is Python's syntax for an <em>anonymous function</em> — a function object created
        inline without a <code>def</code> statement. It originates from the mathematical concept of <em>lambda
          calculus</em> introduced by Alonzo Church in the 1930s, the same theoretical foundation underlying all of
        functional programming.</p>
      <h3>Core Syntax</h3>
      <div class="formula">
        <span class="sym">lambda</span> <span class="eq">arg₁, arg₂, ..., argₙ</span> <span class="sym">:</span> <span
          class="eq">expression</span>
      </div>
      <p>The body is limited to a <strong>single expression</strong> — no statements, no loops, no assignments. It
        returns the result of that expression implicitly.</p>
      <div>
        <p><strong>File: lambda_anatomy.py</strong></p>
        <pre><span class="cm"># ─── Anatomy of a lambda ───────────────────────────────────────</span>

<span class="cm"># Zero arguments</span>
<span class="var">greet</span>   = <span class="kw">lambda</span>: <span class="str">"Moien!"</span>                     <span class="cm"># → "Moien!"</span>

<span class="cm"># Single argument</span>
<span class="var">square</span>  = <span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span> ** <span class="num">2</span>                    <span class="cm"># → 9 (when x=3)</span>

<span class="cm"># Multiple arguments</span>
<span class="var">dot</span>     = <span class="kw">lambda</span> <span class="var">a</span>, <span class="var">b</span>: <span class="builtin">sum</span>(<span class="var">x</span>*<span class="var">y</span> <span class="kw">for</span> <span class="var">x</span>,<span class="var">y</span> <span class="kw">in</span> <span class="builtin">zip</span>(<span class="var">a</span>,<span class="var">b</span>))  <span class="cm"># dot product</span>

<span class="cm"># Default argument</span>
<span class="var">scale</span>   = <span class="kw">lambda</span> <span class="var">x</span>, <span class="var">k</span>=<span class="num">2</span>: <span class="var">x</span> * <span class="var">k</span>               <span class="cm"># → 6 (when x=3)</span>

<span class="cm"># *args — variadic positional</span>
<span class="var">vec_sum</span> = <span class="kw">lambda</span> *<span class="var">args</span>: <span class="builtin">sum</span>(<span class="var">args</span>)            <span class="cm"># → 15 (1+2+3+4+5)</span>

<span class="cm"># **kwargs — variadic keyword</span>
<span class="var">describe</span> = <span class="kw">lambda</span> **<span class="var">kw</span>: <span class="str">', '</span>.<span class="fn">join</span>(<span class="str">f'</span><span class="kw">{k}</span><span class="str">=</span><span class="kw">{v}</span><span class="str">'</span> <span class="kw">for</span> <span class="var">k</span>,<span class="var">v</span> <span class="kw">in</span> <span class="var">kw</span>.<span class="fn">items</span>())
<span class="builtin">print</span>(<span class="var">describe</span>(<span class="var">name</span>=<span class="str">'Arman'</span>, <span class="var">city</span>=<span class="str">'Luxembourg'</span>))  <span class="cm"># → name=Arman, city=Luxembourg</span>

<span class="cm"># Mixed: positional + *args + **kwargs</span>
<span class="var">mixed</span>   = <span class="kw">lambda</span> <span class="var">x</span>, *<span class="var">args</span>, **<span class="var">kw</span>: (<span class="var">x</span>, <span class="var">args</span>, <span class="var">kw</span>)
<span class="builtin">print</span>(<span class="var">mixed</span>(<span class="num">1</span>, <span class="num">2</span>, <span class="num">3</span>, <span class="var">y</span>=<span class="num">4</span>))               <span class="cm"># → (1, (2, 3), {'y': 4})</span>

<span class="cm"># Ternary conditional inside lambda</span>
<span class="var">relu</span>    = <span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span> <span class="kw">if</span> <span class="var">x</span> &gt; <span class="num">0</span> <span class="kw">else</span> <span class="num">0</span>        <span class="cm"># ReLU activation</span>

<span class="cm"># Immediately Invoked Lambda Expression (IILE)</span>
<span class="var">result</span>  = (<span class="kw">lambda</span> <span class="var">x</span>, <span class="var">y</span>: <span class="var">x</span> + <span class="var">y</span>)(<span class="num">3</span>, <span class="num">4</span>)         <span class="cm"># → 7</span>

<span class="cm"># Lambda returning a lambda (currying)</span>
<span class="var">multiply_by</span> = <span class="kw">lambda</span> <span class="var">k</span>: (<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span> * <span class="var">k</span>)
<span class="var">double</span>      = <span class="var">multiply_by</span>(<span class="num">2</span>)
<span class="var">triple</span>      = <span class="var">multiply_by</span>(<span class="num">3</span>)
<span class="builtin">print</span>(<span class="var">double</span>(<span class="num">10</span>))                             <span class="cm"># → 20</span>
<span class="builtin">print</span>(<span class="var">triple</span>(<span class="num">10</span>))                             <span class="cm"># → 30</span></pre>
      </div>
      <h3>Lambda as a First-Class Object</h3>
      <p>In Python, functions are first-class citizens. A lambda can be stored in a variable, in a list or dictionary,
        passed as an argument, or returned from another function.</p>
      <div>
        <p><strong>File: lambda_first_class.py</strong></p>
        <pre><span class="cm"># ─── Lambda stored in data structures ──────────────────────────</span>

<span class="cm"># Dictionary of operations (dispatch table)</span>
<span class="var">ops</span> = {
    <span class="str">"add"</span>  : <span class="kw">lambda</span> <span class="var">a</span>, <span class="var">b</span>: <span class="var">a</span> + <span class="var">b</span>,
    <span class="str">"sub"</span>  : <span class="kw">lambda</span> <span class="var">a</span>, <span class="var">b</span>: <span class="var">a</span> - <span class="var">b</span>,
    <span class="str">"mul"</span>  : <span class="kw">lambda</span> <span class="var">a</span>, <span class="var">b</span>: <span class="var">a</span> * <span class="var">b</span>,
    <span class="str">"dot"</span>  : <span class="kw">lambda</span> <span class="var">u</span>, <span class="var">v</span>: <span class="builtin">sum</span>(<span class="var">x</span>*<span class="var">y</span> <span class="kw">for</span> <span class="var">x</span>,<span class="var">y</span> <span class="kw">in</span> <span class="builtin">zip</span>(<span class="var">u</span>,<span class="var">v</span>)),
    <span class="str">"config"</span>: <span class="kw">lambda</span> **<span class="var">kw</span>: {<span class="str">'defaults'</span>: {<span class="str">'lr'</span>:<span class="num">0.001</span>}, **<span class="var">kw</span>},
}
<span class="var">ops</span>[<span class="str">"dot"</span>]([<span class="num">1</span>,<span class="num">2</span>,<span class="num">3</span>], [<span class="num">4</span>,<span class="num">5</span>,<span class="num">6</span>])               <span class="cm"># → 32  (1×4 + 2×5 + 3×6)</span>
<span class="var">ops</span>[<span class="str">"config"</span>](<span class="var">lr</span>=<span class="num">0.01</span>, <span class="var">epochs</span>=<span class="num">10</span>)           <span class="cm"># → {'defaults':..., 'lr':0.01, ...}</span>

<span class="cm"># List of activation functions (ML pattern)</span>
<span class="var">activations</span> = [
    <span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>,                             <span class="cm"># identity</span>
    <span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span> <span class="kw">if</span> <span class="var">x</span> &gt; <span class="num">0</span> <span class="kw">else</span> <span class="num">0</span>,               <span class="cm"># relu</span>
    <span class="kw">lambda</span> <span class="var">x</span>: <span class="num">1</span> / (<span class="num">1</span> + <span class="num">2.718</span>**(-<span class="var">x</span>)),          <span class="cm"># sigmoid ≈</span>
    <span class="kw">lambda</span> <span class="var">x</span>: (<span class="num">2.718</span>**<span class="var">x</span> - <span class="num">2.718</span>**(-<span class="var">x</span>)) /
               (<span class="num">2.718</span>**<span class="var">x</span> + <span class="num">2.718</span>**(-<span class="var">x</span>)),     <span class="cm"># tanh ≈</span>
]
<span class="builtin">print</span>(<span class="var">activations</span>[<span class="num">1</span>](-<span class="num">3</span>))                    <span class="cm"># → 0  (relu)</span>
<span class="builtin">print</span>(<span class="var">activations</span>[<span class="num">1</span>](<span class="num">7</span>))                     <span class="cm"># → 7</span></pre>
      </div>
      <h3>Closures &amp; Scope (LEGB)</h3>
      <p>A lambda <em>closes over</em> variables from its enclosing scope. This is powerful for creating parameterised
        functions on the fly — critical in linear algebra where you want to generate scaled or shifted transforms.</p>
      <div>
        <p><strong>File: lambda_closure.py</strong></p>
        <pre><span class="cm"># ─── Closure: lambda captures the outer scope ──────────────────</span>

<span class="kw">def</span> <span class="fn">make_scaler</span>(<span class="var">k</span>):
    <span class="cm">"""Returns a function that multiplies a vector by scalar k."""</span>
    <span class="kw">return</span> <span class="kw">lambda</span> <span class="var">vec</span>: [<span class="var">k</span> * <span class="var">x</span> <span class="kw">for</span> <span class="var">x</span> <span class="kw">in</span> <span class="var">vec</span>]

<span class="var">scale_by_3</span> = <span class="fn">make_scaler</span>(<span class="num">3</span>)
<span class="var">scale_by_π</span> = <span class="fn">make_scaler</span>(<span class="num">3.14159</span>)
<span class="builtin">print</span>(<span class="var">scale_by_3</span>([<span class="num">1</span>, <span class="num">2</span>, <span class="num">3</span>]))              <span class="cm"># → [3, 6, 9]</span>
<span class="builtin">print</span>(<span class="var">scale_by_π</span>([<span class="num">1</span>, <span class="num">0</span>, <span class="num">0</span>]))              <span class="cm"># → [3.14159, 0.0, 0.0]</span>

<span class="cm"># ─── Classic late-binding trap ─────────────────────────────────</span>
<span class="cm"># WRONG: all lambdas capture the same `i` at loop end</span>
<span class="var">bad</span>  = [<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span> * <span class="var">i</span>       <span class="kw">for</span> <span class="var">i</span> <span class="kw">in</span> <span class="builtin">range</span>(<span class="num">3</span>)]
<span class="var">good</span> = [<span class="kw">lambda</span> <span class="var">x</span>, <span class="var">i</span>=<span class="var">i</span>: <span class="var">x</span> * <span class="var">i</span> <span class="kw">for</span> <span class="var">i</span> <span class="kw">in</span> <span class="builtin">range</span>(<span class="num">3</span>)]  <span class="cm"># fix: capture now</span>
<span class="builtin">print</span>([<span class="var">f</span>(<span class="num">5</span>) <span class="kw">for</span> <span class="var">f</span> <span class="kw">in</span> <span class="var">bad</span>])               <span class="cm"># → [10, 10, 10]  ← wrong</span>
<span class="builtin">print</span>([<span class="var">f</span>(<span class="num">5</span>) <span class="kw">for</span> <span class="var">f</span> <span class="kw">in</span> <span class="var">good</span>])              <span class="cm"># → [0,  5, 10]  ← correct</span></pre>
      </div>
      <div class="callout callout-amber">

        <p><strong>Late Binding Trap:</strong> Lambdas inside loops don't capture the value of the loop variable — they
          capture the variable itself. Always use a default argument <code>i=i</code> to snapshot the current value.</p>
      </div>
      <h3>Sorting with lambda</h3>
      <p>One of the most practical uses of lambda is as the <code>key</code> argument to <code>sorted()</code>. Return a
        <strong>tuple</strong> to sort by multiple criteria — Python compares tuples lexicographically, breaking ties on
        subsequent elements.</p>
      <div>
        <p><strong>File: lambda_sorting.py</strong></p>
        <pre><span class="kw">import</span> <span class="var">math</span>

<span class="var">vectors</span> = [[<span class="num">3</span>,<span class="num">4</span>], [<span class="num">1</span>,<span class="num">1</span>], [<span class="num">5</span>,<span class="num">12</span>], [<span class="num">0</span>,<span class="num">7</span>]]

<span class="cm"># Sort by Euclidean magnitude (L2 norm)</span>
<span class="var">by_magnitude</span> = <span class="builtin">sorted</span>(<span class="var">vectors</span>,
    <span class="var">key</span>=<span class="kw">lambda</span> <span class="var">v</span>: <span class="var">math</span>.<span class="fn">sqrt</span>(<span class="builtin">sum</span>(<span class="var">x</span>**<span class="num">2</span> <span class="kw">for</span> <span class="var">x</span> <span class="kw">in</span> <span class="var">v</span>)))
<span class="cm"># → [[1,1], [0,7], [3,4], [5,12]]</span>

<span class="cm"># ─── Multi-key tuple sort ───────────────────────────────────────</span>
<span class="cm"># Primary: GPA descending (-s[1])  Secondary: name ascending (s[0])</span>
<span class="var">students</span> = [(<span class="str">'Bob'</span>,<span class="num">3.5</span>), (<span class="str">'Alice'</span>,<span class="num">3.5</span>), (<span class="str">'Carol'</span>,<span class="num">3.9</span>), (<span class="str">'Dave'</span>,<span class="num">3.9</span>)]
<span class="var">ranked</span> = <span class="builtin">sorted</span>(<span class="var">students</span>, <span class="var">key</span>=<span class="kw">lambda</span> <span class="var">s</span>: (-<span class="var">s</span>[<span class="num">1</span>], <span class="var">s</span>[<span class="num">0</span>]))
<span class="cm"># → [('Carol',3.9), ('Dave',3.9), ('Alice',3.5), ('Bob',3.5)]</span>

<span class="cm"># Applied to matrix rows: sort by row sum desc, then first element asc</span>
<span class="var">M</span> = [[<span class="num">3</span>,<span class="num">1</span>,<span class="num">4</span>], [<span class="num">1</span>,<span class="num">5</span>,<span class="num">9</span>], [<span class="num">2</span>,<span class="num">6</span>,<span class="num">5</span>], [<span class="num">1</span>,<span class="num">4</span>,<span class="num">9</span>]]
<span class="var">sorted_M</span> = <span class="builtin">sorted</span>(<span class="var">M</span>, <span class="var">key</span>=<span class="kw">lambda</span> <span class="var">row</span>: (-<span class="builtin">sum</span>(<span class="var">row</span>), <span class="var">row</span>[<span class="num">0</span>]))
<span class="cm"># → [[1,5,9], [1,4,9], [2,6,5], [3,1,4]]</span>

<span class="cm"># Multi-key: length first, then first element</span>
<span class="var">data</span> = [(<span class="str">"v2"</span>,[<span class="num">1</span>,<span class="num">2</span>]), (<span class="str">"v3"</span>,[<span class="num">1</span>,<span class="num">2</span>,<span class="num">3</span>]), (<span class="str">"v1"</span>,[<span class="num">3</span>])]
<span class="builtin">sorted</span>(<span class="var">data</span>, <span class="var">key</span>=<span class="kw">lambda</span> <span class="var">t</span>: (<span class="builtin">len</span>(<span class="var">t</span>[<span class="num">1</span>]), <span class="var">t</span>[<span class="num">1</span>][<span class="num">0</span>]))
<span class="cm"># → [('v1',[3]), ('v2',[1,2]), ('v3',[1,2,3])]</span></pre>
      </div>

    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-blue">02</span>
        <h2 style="margin:0;font-size:24px;">map() — Complete Reference</h2>
      </div>

      <div class="callout callout-purple">
        <p><strong>The Functor Pattern:</strong> In category theory, a <em>functor</em> is a mapping between categories.
          In Python, <code>map()</code> acts as a functor by safely applying a transformation (your lambda) to every
          element inside a container without altering the structure of the container itself.</p>
      </div>
      <p><code>map(function, iterable, ...)</code> applies <code>function</code> to every element of one or more
        iterables, returning a <em>lazy iterator</em>. It embodies the <em>functor</em> pattern: a structure-preserving
        transformation over a container.</p>
      <div class="formula">
        <span class="sym">map</span>(<span class="eq">f</span>, <span class="sym">[</span><span class="eq">x₀, x₁, ...,
          xₙ</span><span class="sym">]</span>) <span class="op">→</span> <span class="sym">[</span><span
          class="eq">f(x₀), f(x₁), ..., f(xₙ)</span><span class="sym">]</span>
      </div>
      <div class="callout callout-blue">

        <p><strong>Lazy Evaluation:</strong> <code>map()</code> returns a <em>map object</em> (an iterator), not a list.
          It computes values on demand. Wrap with <code>list()</code>, <code>tuple()</code>, or iterate with a loop to
          materialise results — memory-efficient for large matrices.</p>
      </div>
      <h3>Basic Usage Patterns</h3>
      <div>
        <p><strong>File: map_basics.py</strong></p>
        <pre><span class="var">v</span> = [<span class="num">1</span>, <span class="num">2</span>, <span class="num">3</span>, <span class="num">4</span>, <span class="num">5</span>]

<span class="cm"># 1. With a named function</span>
<span class="kw">def</span> <span class="fn">square</span>(<span class="var">x</span>): <span class="kw">return</span> <span class="var">x</span> ** <span class="num">2</span>
<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="fn">square</span>, <span class="var">v</span>))              <span class="cm"># → [1, 4, 9, 16, 25]</span>

<span class="cm"># 2. With a lambda  (most common)</span>
<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>**<span class="num">2</span>, <span class="var">v</span>))      <span class="cm"># → [1, 4, 9, 16, 25]</span>

<span class="cm"># 3. With a built-in function</span>
<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="builtin">str</span>, <span class="var">v</span>))                 <span class="cm"># → ['1','2','3','4','5']</span>
<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="builtin">abs</span>, [-<span class="num">3</span>, <span class="num">1</span>, -<span class="num">4</span>]))       <span class="cm"># → [3, 1, 4]</span>

<span class="cm"># 4. Multiple iterables — function accepts n args</span>
<span class="var">u</span> = [<span class="num">1</span>, <span class="num">2</span>, <span class="num">3</span>]; <span class="var">w</span> = [<span class="num">4</span>, <span class="num">5</span>, <span class="num">6</span>]
<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">a</span>, <span class="var">b</span>: <span class="var">a</span>+<span class="var">b</span>, <span class="var">u</span>, <span class="var">w</span>)) <span class="cm"># → [5, 7, 9]   (vector addition)</span>
<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">a</span>, <span class="var">b</span>: <span class="var">a</span>*<span class="var">b</span>, <span class="var">u</span>, <span class="var">w</span>)) <span class="cm"># → [4, 10, 18] (Hadamard product)</span>

<span class="cm"># 5. Chained maps (function composition)</span>
<span class="var">result</span> = <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>*<span class="num">2</span>,   <span class="cm"># step 2: scale</span>
               <span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>+<span class="num">1</span>,   <span class="cm"># step 1: shift</span>
                   <span class="var">v</span>)))
<span class="cm"># → [4, 6, 8, 10, 12]   i.e., (v+1)*2</span>

<span class="cm"># 6. map is lazy — materialise when needed</span>
<span class="var">m</span> = <span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>**<span class="num">3</span>, <span class="var">v</span>)       <span class="cm"># no computation yet</span>
<span class="builtin">next</span>(<span class="var">m</span>)                           <span class="cm"># → 1  (computes only first)</span>
<span class="builtin">list</span>(<span class="var">m</span>)                           <span class="cm"># → [8, 27, 64, 125]  (rest)</span>

<span class="cm"># 7. Stopping at shortest iterable</span>
<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">a</span>, <span class="var">b</span>: <span class="var">a</span>*<span class="var">b</span>, [<span class="num">1</span>,<span class="num">2</span>,<span class="num">3</span>], [<span class="num">10</span>,<span class="num">20</span>]))
<span class="cm"># → [10, 40]  — stops at length of shortest</span></pre>
      </div>

      <h3>map() vs filter() vs reduce()</h3>
      <div class="grid-2">
        <div class="">
          <h3>map(f, iter)</h3>
          <ul class="">
            <li>Transforms every element</li>
            <li>Output length = Input length</li>
            <li>Returns an iterator</li>
            <li>Example: <code>map(lambda x: x**2, v)</code></li>
          </ul>
        </div>
        <div class="">
          <h3>filter(pred, iter)</h3>
          <ul class="">
            <li>Selects elements where pred is True</li>
            <li>Output length ≤ Input length</li>
            <li>Returns an iterator</li>
            <li>Example: <code>filter(lambda x: x&gt;0, v)</code></li>
          </ul>
        </div>
      </div>
      <div>
        <p><strong>File: map_filter_reduce.py</strong></p>
        <pre><span class="kw">from</span> <span class="var">functools</span> <span class="kw">import</span> <span class="fn">reduce</span>

<span class="var">v</span> = [-<span class="num">3</span>, -<span class="num">1</span>, <span class="num">0</span>, <span class="num">2</span>, <span class="num">4</span>]

<span class="cm"># map   → transform all</span>
<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>**<span class="num">2</span>, <span class="var">v</span>))          <span class="cm"># → [9, 1, 0, 4, 16]</span>

<span class="cm"># filter → keep where condition is True</span>
<span class="builtin">list</span>(<span class="builtin">filter</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span> &gt; <span class="num">0</span>, <span class="var">v</span>))       <span class="cm"># → [2, 4]</span>

<span class="cm"># reduce → fold/accumulate to a single value</span>
<span class="fn">reduce</span>(<span class="kw">lambda</span> <span class="var">acc</span>, <span class="var">x</span>: <span class="var">acc</span> + <span class="var">x</span>, <span class="var">v</span>)       <span class="cm"># → 2  (sum)</span>
<span class="fn">reduce</span>(<span class="kw">lambda</span> <span class="var">acc</span>, <span class="var">x</span>: <span class="var">acc</span> * <span class="var">x</span>, <span class="var">v</span>)       <span class="cm"># → 0  (product)</span>

<span class="cm"># Chained pipeline: square positives then sum</span>
<span class="fn">reduce</span>(<span class="kw">lambda</span> <span class="var">a</span>, <span class="var">b</span>: <span class="var">a</span>+<span class="var">b</span>,
       <span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>**<span class="num">2</span>,
           <span class="builtin">filter</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span> &gt; <span class="num">0</span>, <span class="var">v</span>)))  <span class="cm"># → 20  (4 + 16)</span></pre>
      </div>

          function is short and disposable.</p>

        <span class="badge badge-purple">04</span>
        <h2 style="margin:0;font-size:24px;">Linear Algebra Foundations & Complete Norm Reference</h2>
      </div>
      <p>Before connecting <code>map</code> and <code>lambda</code> to matrix operations, we need a precise mental model
        of data structures and the mathematical objects they represent.</p>
      <h3>Hierarchy of Mathematical Objects in Python</h3>
      <div class="grid-2">
        <div class="">
          <h3>Scalars &amp; Vectors</h3>
          <ul class="">
            <li>Scalar: <code>x = 3.14</code> — a single number, rank-0 tensor</li>
            <li>Vector: <code>[1, 2, 3]</code> — 1D list, rank-1 tensor</li>
            <li>Column vs Row: convention — list = row vector</li>
            <li>Dot product: <code>sum(a*b for a,b in zip(u,v))</code></li>
          </ul>
        </div>
        <div class="">
          <h3>Matrices &amp; Tensors</h3>
          <ul class="">
            <li>2D matrix: list of lists <code>[[row0], [row1]]</code></li>
            <li>3D tensor: list of 2D matrices (batch/depth)</li>
            <li>Shape: <code>(rows, cols)</code> or <code>(depth, rows, cols)</code></li>
            <li>Element: <code>M[i][j]</code> for row i, col j</li>
          </ul>
        </div>
      </div>
      <div class="callout callout-purple">

        <p>
          <strong>Mathematical Correspondence:</strong><br />
          Scalar multiplication: <em>k·v = [k·v₀, k·v₁, ..., k·vₙ]</em> → <code>map(lambda x: k*x, v)</code><br />
          Vector addition: <em>u + v = [u₀+v₀, u₁+v₁, ...]</em> → <code>map(lambda a,b: a+b, u, v)</code><br />
          Matrix scalar op: <em>k·A</em> → <code>map(lambda row: map(lambda x: k*x, row), A)</code><br />
          Function application: <em>f(A)</em> → <code>map(lambda row: map(f, row), A)</code>
        </p>
      </div>
      <h3>Core Building Blocks</h3>
      <div>
        <p><strong>File: la_building_blocks.py</strong></p>
        <pre><span class="cm"># ─── Core vector operations via map + lambda ───────────────────</span>

<span class="builtin">print</span>(<span class="var">v_unit</span>(<span class="var">u</span>))              <span class="cm"># → [0.6, 0.8]</span>
<span class="builtin">print</span>(<span class="var">v_dot</span>([<span class="num">1</span>,<span class="num">0</span>],[<span class="num">0</span>,<span class="num">1</span>]))     <span class="cm"># → 0   (perpendicular)</span>

<span class="cm"># Utility: materialise a nested map result into a list-of-lists</span>
<span class="var">mat</span>   = <span class="kw">lambda</span> <span class="var">m</span>: [<span class="builtin">list</span>(<span class="var">row</span>) <span class="kw">for</span> <span class="var">row</span> <span class="kw">in</span> <span class="var">m</span>]
<span class="var">shape</span> = <span class="kw">lambda</span> <span class="var">m</span>: (<span class="builtin">len</span>(<span class="var">m</span>), <span class="builtin">len</span>(<span class="var">m</span>[<span class="num">0</span>]))</pre>
      </div>
      <h3>Complete Norm Reference (L1, L2, Lp, L∞)</h3>
      <div class="callout callout-purple">

        <p>
          <strong>Norm Family:</strong><br />
          L1 (Manhattan): <em>‖v‖₁ = Σ|vᵢ|</em><br />
          L2 (Euclidean): <em>‖v‖₂ = √(Σvᵢ²)</em><br />
          Lp (General): <em>‖v‖ₚ = (Σ|vᵢ|ᵖ)^(1/p)</em><br />
          L∞ (Chebyshev): <em>‖v‖∞ = max(|vᵢ|)</em> — the limiting case of Lp as p→∞
        </p>
      </div>
      <div>
        <p><strong>File: norms_complete.py</strong></p>
        <pre><span class="kw">import</span> <span class="var">math</span>

<span class="cm"># ─── Complete norm suite ────────────────────────────────────────</span>

<span class="builtin">print</span>(<span class="var">frob</span>([[<span class="num">1</span>,<span class="num">2</span>],[<span class="num">3</span>,<span class="num">4</span>]]))     <span class="cm"># → √30 ≈ 5.477</span>

<span class="cm"># ─── Recursive n×n Determinant (Laplace cofactor expansion) ────</span>
<span class="cm"># minor: delete row i and column j</span>
<span class="var">minor</span> = <span class="kw">lambda</span> <span class="var">M</span>, <span class="var">i</span>, <span class="var">j</span>: [
    <span class="var">row</span>[:<span class="var">j</span>] + <span class="var">row</span>[<span class="var">j</span>+<span class="num">1</span>:]
    <span class="kw">for</span> <span class="var">row</span> <span class="kw">in</span> (<span class="var">M</span>[:<span class="var">i</span>] + <span class="var">M</span>[<span class="var">i</span>+<span class="num">1</span>:])
]

<span class="kw">def</span> <span class="fn">det</span>(<span class="var">M</span>):
    <span class="cm">"""Determinant of n×n matrix — Laplace expansion, O(n!)."""</span>
    <span class="kw">if</span> <span class="builtin">len</span>(<span class="var">M</span>) == <span class="num">1</span>: <span class="kw">return</span> <span class="var">M</span>[<span class="num">0</span>][<span class="num">0</span>]
    <span class="kw">return</span> <span class="builtin">sum</span>(<span class="builtin">map</span>(
        <span class="kw">lambda</span> <span class="var">j</span>: ((-<span class="num">1</span>)**<span class="var">j</span>) * <span class="var">M</span>[<span class="num">0</span>][<span class="var">j</span>] * <span class="fn">det</span>(<span class="var">minor</span>(<span class="var">M</span>, <span class="num">0</span>, <span class="var">j</span>)),
        <span class="builtin">range</span>(<span class="builtin">len</span>(<span class="var">M</span>[<span class="num">0</span>]))
    ))

<span class="builtin">print</span>(<span class="fn">det</span>([[<span class="num">3</span>,<span class="num">8</span>],[<span class="num">4</span>,<span class="num">6</span>]]))                       <span class="cm"># → -14</span>
<span class="builtin">print</span>(<span class="fn">det</span>([[<span class="num">1</span>,<span class="num">2</span>,<span class="num">3</span>],[<span class="num">4</span>,<span class="num">5</span>,<span class="num">6</span>],[<span class="num">7</span>,<span class="num">8</span>,<span class="num">10</span>]]))          <span class="cm"># → -3</span>
<span class="builtin">print</span>(<span class="fn">det</span>([[<span class="num">1</span>,<span class="num">2</span>,<span class="num">3</span>],[<span class="num">4</span>,<span class="num">5</span>,<span class="num">6</span>],[<span class="num">7</span>,<span class="num">8</span>,<span class="num">9</span>]]))          <span class="cm"># → 0  (singular)</span></pre>
      </div>
      <div class="callout callout-amber">

        <p><strong>Complexity:</strong> Laplace expansion is O(n!) — never use it for matrices larger than ~8×8. In
          production, use <code>numpy.linalg.det()</code> which runs O(n³) via LU decomposition.</p>
      </div>

        <h2 style="margin:0;font-size:24px;">3D Tensor Operations</h2>
      </div>
      <p>A 3D tensor is a list of 2D matrices: <code>T[d][i][j]</code> where <em>d</em> is the depth/batch index,
        <em>i</em> the row, <em>j</em> the column. This is the natural shape for batched ML data (batch_size × rows ×
        cols) or volumetric data (depth × height × width).</p>
      <div class="callout callout-blue">

        <p><strong>Notation:</strong> A 3D tensor of shape (D, R, C) has D slices, each an R×C matrix. Accessing
          element: <code>T[d][r][c]</code>. In NumPy this is <code>T[d, r, c]</code>.</p>
      </div>
      <div>
        <p><strong>File: tensor_3d.py</strong></p>
        <pre><span class="cm"># Shape (D, R, C) = (3, 2, 4)</span>

    [[<span class="num">9</span>,<span class="num">10</span>,<span class="num">11</span>,<span class="num">12</span>],  [<span class="num">13</span>,<span class="num">14</span>,<span class="num">15</span>,<span class="num">16</span>]],   <span class="cm"># slice 1</span>

      <p>Modern deep learning is built on element-wise nonlinear functions applied to matrices and tensors. Every

<span class="var">z</span> = [[-<span class="num">2</span>,-<span class="num">1</span>,<span class="num">0</span>,<span class="num">1</span>,<span class="num">2</span>]]
<span class="builtin">print</span>(<span class="var">apply_m</span>(<span class="var">relu</span>,    <span class="var">z</span>))   <span class="cm"># → [[0, 0, 0, 1, 2]]</span>

      <h3>Softmax &amp; Layer Normalisation</h3>
      <div class="formula">
        <span class="sym">softmax</span>(<span class="eq">z</span>)<span class="idx">i</span> <span class="eq">= eᶻⁱ /
          Σⱼ eᶻʲ</span>
      </div>
      <div>
        <p><strong>File: softmax_layernorm.py</strong></p>
        <pre><span class="kw">import</span> <span class="var">math</span>

<span class="cm"># ─── Numerically stable softmax ────────────────────────────────</span>
<span class="kw">def</span> <span class="fn">softmax</span>(<span class="var">v</span>):
    <span class="var">m</span>    = <span class="builtin">max</span>(<span class="var">v</span>)                             <span class="cm"># stability: subtract max</span>
    <span class="var">exps</span> = <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">math</span>.<span class="fn">exp</span>(<span class="var">x</span>-<span class="var">m</span>), <span class="var">v</span>))
    <span class="var">s</span>    = <span class="builtin">sum</span>(<span class="var">exps</span>)
    <span class="kw">return</span> <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">e</span>: <span class="var">e</span>/<span class="var">s</span>, <span class="var">exps</span>))

<span class="cm"># Batch softmax: row-wise over a matrix</span>
<span class="var">batch_softmax</span> = <span class="kw">lambda</span> <span class="var">M</span>: <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="fn">softmax</span>, <span class="var">M</span>))

<span class="var">logits</span> = [[<span class="num">1.0</span>,<span class="num">2.0</span>,<span class="num">3.0</span>],[<span class="num">0.5</span>,<span class="num">0.5</span>,<span class="num">0.5</span>],[<span class="num">10</span>,<span class="num">0</span>,-<span class="num">10</span>]]
<span class="var">probs</span>  = <span class="var">batch_softmax</span>(<span class="var">logits</span>)
<span class="cm"># → [[0.090,0.245,0.665], [0.333,0.333,0.333], [1.000,0.000,0.000]]</span>

<span class="cm"># ─── Layer Normalisation (simplified) ──────────────────────────</span>
<span class="kw">def</span> <span class="fn">layer_norm</span>(<span class="var">v</span>, <span class="var">eps</span>=<span class="num">1e-8</span>):
    <span class="var">mean</span> = <span class="builtin">sum</span>(<span class="var">v</span>) / <span class="builtin">len</span>(<span class="var">v</span>)
    <span class="var">var</span>  = <span class="builtin">sum</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: (<span class="var">x</span>-<span class="var">mean</span>)**<span class="num">2</span>, <span class="var">v</span>)) / <span class="builtin">len</span>(<span class="var">v</span>)
    <span class="var">std</span>  = (<span class="var">var</span> + <span class="var">eps</span>)**<span class="num">0.5</span>
    <span class="kw">return</span> <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: (<span class="var">x</span>-<span class="var">mean</span>)/<span class="var">std</span>, <span class="var">v</span>))

<span class="fn">layer_norm</span>([<span class="num">2</span>,<span class="num">4</span>,<span class="num">4</span>,<span class="num">4</span>,<span class="num">5</span>,<span class="num">5</span>,<span class="num">7</span>,<span class="num">9</span>])
<span class="cm"># → [-1.5, -0.5, -0.5, -0.5,  0.0,  0.0,  1.0,  2.0]</span></pre>
      </div>

    <span class="var">XW</span> = <span class="fn">matmul</span>(<span class="var">X</span>, <span class="var">W</span>)                         <span class="cm"># (batch, out)</span>
    <span class="kw">return</span> <span class="builtin">list</span>(<span class="builtin">map</span>(

      <h3>Scaled Dot-Product Attention</h3>
      <div>
        <p><strong>File: attention.py</strong></p>
        <pre><span class="kw">import</span> <span class="var">math</span>

<span class="kw">def</span> <span class="fn">attention</span>(<span class="var">Q</span>, <span class="var">K</span>, <span class="var">V</span>):
    <span class="cm">"""Scaled dot-product attention. Q, K, V are (seq_len × d_k) matrices."""</span>
    <span class="var">d_k</span>   = <span class="builtin">len</span>(<span class="var">Q</span>[<span class="num">0</span>]); <span class="var">scale</span> = <span class="var">d_k</span>**<span class="num">0.5</span>
    <span class="var">Kt</span>    = <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="builtin">list</span>, <span class="builtin">zip</span>(*<span class="var">K</span>)))
    <span class="var">d</span>     = <span class="kw">lambda</span> <span class="var">r</span>, <span class="var">c</span>: <span class="builtin">sum</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">a</span>,<span class="var">b</span>: <span class="var">a</span>*<span class="var">b</span>, <span class="var">r</span>, <span class="var">c</span>))
    <span class="var">scores</span>= [[<span class="var">d</span>(<span class="var">q</span>,<span class="var">k</span>)/<span class="var">scale</span> <span class="kw">for</span> <span class="var">k</span> <span class="kw">in</span> <span class="var">Kt</span>] <span class="kw">for</span> <span class="var">q</span> <span class="kw">in</span> <span class="var">Q</span>]
    <span class="var">sm</span>    = <span class="kw">lambda</span> <span class="var">v</span>: (
        <span class="kw">lambda</span> <span class="var">e</span>: <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>/<span class="builtin">sum</span>(<span class="var">e</span>), <span class="var">e</span>))
    )(<span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">math</span>.<span class="fn">exp</span>(<span class="var">x</span>-<span class="builtin">max</span>(<span class="var">v</span>)), <span class="var">v</span>)))
    <span class="var">attn</span>  = <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="var">sm</span>, <span class="var">scores</span>))
    <span class="var">Vt</span>    = <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="builtin">list</span>, <span class="builtin">zip</span>(*<span class="var">V</span>)))
    <span class="kw">return</span> [[<span class="var">d</span>(<span class="var">a</span>,<span class="var">v</span>) <span class="kw">for</span> <span class="var">v</span> <span class="kw">in</span> <span class="var">Vt</span>] <span class="kw">for</span> <span class="var">a</span> <span class="kw">in</span> <span class="var">attn</span>]</pre>
      </div>

      <h3>Linear Transformation Composition</h3>
      <p>Multiple linear transforms can be <em>composed</em> into a single matrix using <code>reduce + matmul</code>.
        The composed matrix applies all transforms in one shot, eliminating repeated multiplications at inference time.
      </p>
      <div>
        <p><strong>File: transform_composition.py</strong></p>
        <pre><span class="kw">from</span> <span class="var">functools</span> <span class="kw">import</span> <span class="fn">reduce</span>

<span class="cm"># 2D linear transforms</span>
<span class="var">T_scale</span> = [[<span class="num">2</span>,<span class="num">0</span>],[<span class="num">0</span>,<span class="num">2</span>]]    <span class="cm"># scale by 2</span>
<span class="var">T_rot90</span> = [[<span class="num">0</span>,-<span class="num">1</span>],[<span class="num">1</span>,<span class="num">0</span>]]   <span class="cm"># rotate 90° CCW</span>

<span class="cm"># Compose: ((T_scale @ T_rot90) @ T_shear)</span>
<span class="var">composed</span> = <span class="fn">reduce</span>(<span class="fn">matmul</span>, [<span class="var">T_scale</span>, <span class="var">T_rot90</span>, <span class="var">T_shear</span>])
<span class="cm"># → [[-2,-1],[2,0]]</span>

<span class="cm"># Apply to a batch of points — one map call instead of 3× matvec</span>
<span class="var">matvec</span>    = <span class="kw">lambda</span> <span class="var">M</span>, <span class="var">v</span>: <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">row</span>: <span class="var">dot</span>(<span class="var">row</span>,<span class="var">v</span>), <span class="var">M</span>))

<span class="var">mapped_pts</span> = <span class="builtin">list</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">p</span>: <span class="var">matvec</span>(<span class="var">composed</span>, <span class="var">p</span>), <span class="var">points</span>))
<span class="cm"># All four points transformed in one map call</span></pre>
      </div>

      <h3>Lambda Pipeline with reduce</h3>
      <p>Define a list of lambdas representing processing stages, then fold them sequentially using <code>reduce</code>.
        This makes it easy to add, remove, or reorder preprocessing steps without rewriting nested calls.</p>
      <div>
        <p><strong>File: numpy_pipeline.py</strong></p>
        <pre><span class="kw">import</span> <span class="var">numpy</span> <span class="kw">as</span> <span class="var">np</span>
<span class="kw">from</span> <span class="var">functools</span> <span class="kw">import</span> <span class="fn">reduce</span>

<span class="cm"># ─── Define pipeline as a list of lambdas ──────────────────────</span>
<span class="var">pipeline</span> = [
    <span class="kw">lambda</span> <span class="var">M</span>: <span class="var">M</span> - <span class="var">M</span>.<span class="fn">mean</span>(<span class="var">axis</span>=<span class="num">0</span>),           <span class="cm"># 1. centre columns</span>
    <span class="kw">lambda</span> <span class="var">M</span>: <span class="var">M</span> / (<span class="var">M</span>.<span class="fn">std</span>(<span class="var">axis</span>=<span class="num">0</span>) + <span class="num">1e-8</span>),   <span class="cm"># 2. scale columns</span>
    <span class="kw">lambda</span> <span class="var">M</span>: <span class="var">np</span>.<span class="fn">clip</span>(<span class="var">M</span>, -<span class="num">3</span>, <span class="num">3</span>),               <span class="cm"># 3. clip outliers</span>
    <span class="kw">lambda</span> <span class="var">M</span>: <span class="var">np</span>.<span class="fn">round</span>(<span class="var">M</span>, <span class="num">4</span>),                 <span class="cm"># 4. round for display</span>
]

<span class="cm"># ─── Run with reduce: each step's output feeds the next ────────</span>
<span class="var">apply_pipeline</span> = <span class="kw">lambda</span> <span class="var">data</span>, <span class="var">steps</span>: <span class="fn">reduce</span>(<span class="kw">lambda</span> <span class="var">M</span>,<span class="var">f</span>: <span class="var">f</span>(<span class="var">M</span>), <span class="var">steps</span>, <span class="var">data</span>)

<span class="var">raw</span>    = <span class="var">np</span>.<span class="fn">array</span>([[<span class="num">1.0</span>,<span class="num">200.0</span>],[<span class="num">2.0</span>,<span class="num">400.0</span>],[<span class="num">3.0</span>,<span class="num">600.0</span>],[<span class="num">4.0</span>,<span class="num">800.0</span>]])
<span class="var">result</span> = <span class="var">apply_pipeline</span>(<span class="var">raw</span>, <span class="var">pipeline</span>)

<span class="cm"># Extend at runtime — just append</span>
<span class="var">pipeline</span>.<span class="fn">append</span>(<span class="kw">lambda</span> <span class="var">M</span>: <span class="var">M</span> * <span class="num">100</span>)            <span class="cm"># 5. scale to percentages</span>
<span class="var">result2</span> = <span class="var">apply_pipeline</span>(<span class="var">raw</span>, <span class="var">pipeline</span>)</pre>
      </div>
      <div class="callout callout-amber">

        <p><strong>Production Note:</strong> For real matrix operations in data engineering or ML, always use NumPy,
          PyTorch, or TensorFlow. Pure Python <code>map</code>/<code>lambda</code> is for learning, scripting, and
          small-scale transforms — NumPy vectorisation is 10–100× faster because it operates on contiguous C arrays.</p>
      </div>

      <h3>Key Pitfalls to Avoid</h3>
      <div>
        <p><strong>File: pitfalls.py</strong></p>
        <pre><span class="cm"># ─── Pitfall 1: Consuming an iterator twice ─────────────────────</span>
<span class="var">m</span> = <span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>**<span class="num">2</span>, [<span class="num">1</span>,<span class="num">2</span>,<span class="num">3</span>])
<span class="builtin">list</span>(<span class="var">m</span>)  <span class="cm"># → [1, 4, 9]</span>
<span class="builtin">list</span>(<span class="var">m</span>)  <span class="cm"># → []  ← iterator exhausted! Store result immediately.</span>

<span class="cm"># ─── Pitfall 2: Side effects in lambda ─────────────────────────</span>
<span class="cm"># f = lambda x: print(x); x**2  # SyntaxError — no statements!</span>
<span class="cm"># Fix: use def for functions with side effects</span>

<span class="cm"># ─── Pitfall 3: Nested map readability collapse ─────────────────</span>
<span class="cm"># Hard to read — name your lambdas:</span>

<span class="cm"># ─── Pitfall 4: Late binding in loops (see §1) ─────────────────</span>
<span class="var">fns</span> = [<span class="kw">lambda</span> <span class="var">x</span>, <span class="var">i</span>=<span class="var">i</span>: <span class="var">x</span>*<span class="var">i</span> <span class="kw">for</span> <span class="var">i</span> <span class="kw">in</span> <span class="builtin">range</span>(<span class="num">3</span>)]  <span class="cm"># ← correct</span>

<span class="cm"># ─── Pitfall 5: Lambda when def is cleaner ─────────────────────</span>
<span class="cm"># WRONG: long, unreadable</span>
<span class="var">f</span> = <span class="kw">lambda</span> <span class="var">m</span>: <span class="builtin">sum</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">r</span>: <span class="builtin">sum</span>(<span class="builtin">map</span>(<span class="kw">lambda</span> <span class="var">x</span>: <span class="var">x</span>**<span class="num">2</span>, <span class="var">r</span>)), <span class="var">m</span>))**<span class="num">0.5</span>

<span class="cm"># RIGHT: named function with docstring</span>
<span class="kw">def</span> <span class="fn">frobenius_norm</span>(<span class="var">M</span>):
    <span class="cm">"""Frobenius (L2) norm of a matrix."""</span>
    <span class="kw">return</span> <span class="builtin">sum</span>(<span class="var">x</span>**<span class="num">2</span> <span class="kw">for</span> <span class="var">row</span> <span class="kw">in</span> <span class="var">M</span> <span class="kw">for</span> <span class="var">x</span> <span class="kw">in</span> <span class="var">row</span>)**<span class="num">0.5</span></pre>
      </div>
      <div class="callout callout-green">

        <p>
          <strong>Final Rule Set:</strong><br />
          1. Use <code>lambda</code> for short, disposable, single-expression functions.<br />
          2. Use <code>map()</code> when applying a function to an iterable — especially lazy or over multiple
          iterables.<br />
          3. Use list comprehensions for readable single-pass transforms.<br />
          4. Name your lambdas when used more than once — it makes them debuggable and faster.<br />
          5. Graduate to NumPy the moment performance matters for matrix/tensor work.
        </p>
