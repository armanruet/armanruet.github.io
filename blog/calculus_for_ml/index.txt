      <!-- 01 WHAT IS CALCULUS? -->
      <a class="section-anchor" id="what-is-calculus"></a>
      <div class="section">
        <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
          <span class="badge badge-green">01</span>
          <h3>What is Calculus? <span class="difficulty-pill diff-beginner">Beginner</span></h3>
        </div>

        <div class="concept-intro">
          <strong>Calculus is the mathematics of change.</strong> It gives us two superpowers: (1) measuring how fast
          things change (<em>differentiation</em>) and (2) accumulating tiny changes into totals (<em>integration</em>).
          In ML, we use differentiation to figure out <em>which direction to adjust model parameters</em> to reduce
          error.
          <div class="mental-model">🧠 Mental model: Imagine driving a car. Your speedometer shows the
            <em>derivative</em> — the rate of change of position. Your odometer shows the <em>integral</em> — the total
            distance accumulated. Training a neural network is like adjusting the steering wheel (weights) based on how
            fast the error is changing (gradient).
          </div>
        </div>

        <div class="section-flow">
          <span class="flow-step">📖 Concept</span><span class="flow-arrow">›</span>
          <span class="flow-step">📐 Visual</span><span class="flow-arrow">›</span>
          <span class="flow-step">🎯 ML Apply</span>
        </div>

        <p>Calculus has two fundamental branches:</p>

        <div class="grid-2" style="margin:1rem 0;">
          <div style="background:var(--accent-bg);border-radius:var(--radius);padding:1rem;">
            <div style="font-size:13px;font-weight:600;color:var(--accent);margin-bottom:6px;">Differential Calculus
            </div>
            <p style="font-size:13px;line-height:1.6;margin:0;">Studies <strong>rates of change</strong>. Given a
              function f(x), the derivative f'(x) tells us how fast f is changing at each point x. This is how we
              compute <strong>gradients</strong> to train models.</p>
          </div>
          <div style="background:var(--green-bg);border-radius:var(--radius);padding:1rem;">
            <div style="font-size:13px;font-weight:600;color:var(--green);margin-bottom:6px;">Integral Calculus</div>
            <p style="font-size:13px;line-height:1.6;margin:0;">Studies <strong>accumulation</strong>. Given a rate of
              change, integration recovers the total quantity. In ML, integrals appear in probability distributions,
              expected values, and KL divergence.</p>
          </div>
        </div>

        <div class="by-hand-note">Why ML needs calculus</div>
        <div class="mermaid">
          graph LR
          subgraph Training ["How Neural Networks Learn"]
          A["Model makes<br />prediction ŷ"] --> B{"Compare with<br />true label y"}
          B --> C["Compute Loss<br />L(ŷ, y)"]
          C --> D["Take Derivative<br />∂L/∂w (Calculus!)"]
          D --> E["Update Weights<br />w ← w − α·∂L/∂w"]
          E --> A
          end
        </div>

        <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Every time you call
          <code>loss.backward()</code> in PyTorch, you are using differential calculus. The framework computes ∂Loss/∂w
          for every weight w in your network, then gradient descent uses those derivatives to update each weight in the
          direction that reduces the loss.
        </div>
      </div>

      <!-- 03 LIMITS & CONTINUITY -->
      <a class="section-anchor" id="limits"></a>
      <div class="section">
        <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
          <span class="badge badge-green">03</span>
          <h3>Limits & Continuity <span class="difficulty-pill diff-beginner">Beginner</span></h3>
        </div>

        <div class="concept-intro">
          <strong>A limit describes the value a function approaches</strong> as its input gets closer and closer to some
          point. Limits are the foundation of derivatives — a derivative <em>is</em> a limit. Continuity means a
          function has no jumps or holes.
          <div class="mental-model">🧠 Mental model: Walk toward a cliff edge. The <em>limit</em> is the edge — you can
            get arbitrarily close without stepping off. The function's value at that point might differ from the limit
            (a "hole"), or it might match perfectly (continuous).</div>
        </div>

        <div class="section-flow">
          <span class="flow-step">📖 Concept</span><span class="flow-arrow">›</span>
          <span class="flow-step">📐 Math</span><span class="flow-arrow">›</span>
          <span class="flow-step">💻 NumPy</span><span class="flow-arrow">›</span>
          <span class="flow-step">🎯 ML Apply</span>
        </div>

        <div class="formula-block">
          <div class="formula-main">lim<sub>x→a</sub> f(x) = L</div>
          <div class="formula-sub">"As x approaches a, f(x) approaches L"</div>
        </div>

        <p>Three key properties of limits that matter for ML:</p>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
          <div style="font-family:var(--mono);font-size:13px;line-height:2;">
            <strong>1. Sum rule:</strong> lim[f(x) + g(x)] = lim f(x) + lim g(x)<br>
            <strong>2. Product rule:</strong> lim[f(x) · g(x)] = lim f(x) · lim g(x)<br>
            <strong>3. Chain rule:</strong> lim f(g(x)) = f(lim g(x)) <em style="color:var(--text3);">(if f is
              continuous)</em>
          </div>
        </div>

        <div class="grid-2" style="margin:1rem 0;">
          <div>
            <div class="code-heading numpy-heading">Numerical Limit with NumPy</div>
            <pre style="margin:0;"><span class="cm"># lim x→0 sin(x)/x = 1</span>

<span class="cm"># Approach from right</span>

<span class="kw">for</span> h <span class="kw">in</span> h_values:
    <span class="kw">print</span>(f<span class="st">"h={h}: {np.sin(h)/h:.8f}"</span>)

<span class="out"># h=0.1:    0.99833417</span>
<span class="out"># h=0.01:   0.99998333</span>
<span class="out"># h=0.001:  0.99999983</span>
<span class="out"># h=0.0001: 1.00000000</span>
<span class="cm"># → Approaches 1!</span></pre>
          </div>
          <div>
            <div class="code-heading python-heading">Pure Python</div>
            <pre style="margin:0;"><span class="kw">import</span> math

    <span class="st">"""Estimate lim x→a f(x)"""</span>
    <span class="kw">if</span> h_values <span class="kw">is None</span>:
        h_values = [<span class="nm">10</span>**-i
                    <span class="kw">for</span> i <span class="kw">in</span> range(<span class="nm">1</span>,<span class="nm">9</span>)]
    results = []
    <span class="kw">for</span> h <span class="kw">in</span> h_values:
        results.append(f(a + h))
    <span class="kw">return</span> results[-<span class="nm">1</span>]

<span class="cm"># lim x→0 sin(x)/x</span>

    <span class="kw">lambda</span> x: math.sin(x)/x, <span class="nm">0</span>

          </div>
        </div>

        <div class="callout callout-amber">⚠️ <strong>Why continuity matters:</strong> ReLU is <strong>not
            differentiable at x=0</strong> (it has a kink). In practice, we define the derivative at 0 as either 0 or 1
          — this works because we almost never land <em>exactly</em> at 0 during training. But it shows why
          understanding limits is important: the derivative at a kink is a limit that doesn't exist.</div>

        <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Gradient-based optimization assumes the
          loss landscape is <strong>smooth</strong> (differentiable almost everywhere). When we use ReLU or other
          piecewise functions, we rely on the fact that the set of non-differentiable points has measure zero — limits
          and continuity formalize this.</div>
      </div>

      <!-- 04 THE DERIVATIVE (GEOMETRIC VIEW) -->
      <a class="section-anchor" id="derivative-geometric"></a>
      <div class="section">
        <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
          <span class="badge badge-green">04</span>
          <h3>The Derivative (Geometric View) <span class="difficulty-pill diff-beginner">Beginner</span></h3>
        </div>

        <div class="concept-intro">
          <strong>The derivative f'(x) is the slope of the tangent line</strong> to f(x) at point x. It tells you: if
          you nudge x by a tiny amount, how much does f(x) change? Positive derivative = function going up. Negative =
          going down. Zero = flat (potential minimum or maximum).
          <div class="mental-model">🧠 Mental model: Place a ruler tangent to a hill at your feet. The angle of the
            ruler is the derivative. Steep uphill = large positive derivative. Flat plateau = zero derivative. Steep
            downhill = large negative derivative.</div>
        </div>

        <div class="section-flow">
          <span class="flow-step">📖 Concept</span><span class="flow-arrow">›</span>
          <span class="flow-step">📐 Visual</span><span class="flow-arrow">›</span>
          <span class="flow-step">💻 NumPy</span><span class="flow-arrow">›</span>
          <span class="flow-step">🐍 Python</span><span class="flow-arrow">›</span>
          <span class="flow-step">🎯 ML Apply</span>
        </div>

        <div class="formula-block">
          <div class="formula-main">f'(x) = slope of tangent line at x</div>
          <div class="formula-sub">f'(x) > 0 → increasing · f'(x) < 0 → decreasing · f'(x)=0 → critical point</div>
          </div>

          <div class="callout callout-bg2" style="margin-top: 1rem;">
            <strong>Mathematical Explanation & Example:</strong>
            <p style="margin-top: 0.5rem; margin-bottom: 0.5rem;">A derivative measures how a function changes as its
              input changes. Mathematically, it's defined as the limit of the average rate of change as the interval
              approaches zero:</p>
            <p
              style="text-align:center; font-family: var(--mono); margin-bottom: 0.5rem; background: var(--surface); padding: 0.5rem; border-radius: var(--radius);">
              <code>f'(x) = lim(h→0) [f(x+h) - f(x)] / h</code>
            </p>
            <p style="margin-bottom: 0.5rem;"><strong>Example:</strong> Let <code>f(x) = x²</code>. We want to find the
              derivative at <code>x = 3</code>.</p>
            <ul style="margin-left: 1.5rem; margin-bottom: 0;">
              <li>Using the definition, the rate of change from <code>x</code> to <code>x+h</code> is
                <code>((x+h)² - x²) / h</code>.
              </li>
              <li>Expanding the numerator: <code>x² + 2xh + h² - x² = 2xh + h²</code>.</li>
              <li>Dividing by <code>h</code> gives: <code>2x + h</code>.</li>
              <li>As <code>h → 0</code>, this expression becomes <code>2x</code>.</li>
              <li>So, the derivative is <code>f'(x) = 2x</code>. At <code>x = 3</code>, the slope is
                <code>2(3) = 6</code>.
              </li>
            </ul>
            <div id="secant-plot"
              style="width:100%; height:300px; margin-top: 1rem; border: 1px solid var(--border); border-radius: var(--radius); overflow: hidden;">
            </div>
          </div>

          <div class="by-hand-note">The derivative tells gradient descent which way to go!</div>
          <div class="mermaid">
            graph TD
            subgraph Slope ["Derivative as Slope"]
            POS["f'(x) > 0<br />Going uphill ↗"] --> ACT1["Move LEFT<br />(Decrease x)"]
            NEG["f'(x)
            < 0<br />Going downhill ↘"] --> ACT2["Move RIGHT<br />(Increase x)"]
            ZERO["f'(x) = 0<br />Flat — at minimum? ✓"] --> ACT3["Stop!<br />(Converged)"]
            end
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="cm"># f(x) = x² → f'(x) = 2x</span>
<span class="kw">def</span> f(x): <span class="kw">return</span> x**<span class="nm">2</span>
<span class="kw">def</span> f_prime(x): <span class="kw">return</span> <span class="nm">2</span>*x

x = np.array([-<span class="nm">2</span>, -<span class="nm">1</span>, <span class="nm">0</span>, <span class="nm">1</span>, <span class="nm">2</span>])
<span class="kw">print</span>(f<span class="st">"f(x)  = {f(x)}"</span>)
<span class="out"># [4, 1, 0, 1, 4]</span>
<span class="kw">print</span>(f<span class="st">"f'(x) = {f_prime(x)}"</span>)
<span class="out"># [-4, -2, 0, 2, 4]</span>

<span class="cm"># At x=-2: slope is -4 (steeply downhill)</span>
<span class="cm"># At x=0:  slope is  0 (minimum!)</span>
<span class="cm"># At x=2:  slope is  4 (steeply uphill)</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="cm"># Numerical derivative approximation</span>
<span class="kw">def</span> numerical_derivative(f, x, h=<span class="nm">1e-7</span>):
    <span class="st">"""Central difference formula"""</span>
    <span class="kw">return</span> (f(x + h) - f(x - h)) / (<span class="nm">2</span> * h)

<span class="kw">def</span> f(x): <span class="kw">return</span> x ** <span class="nm">2</span>

<span class="cm"># Test at several points</span>
<span class="kw">for</span> x <span class="kw">in</span> [-<span class="nm">2</span>, -<span class="nm">1</span>, <span class="nm">0</span>, <span class="nm">1</span>, <span class="nm">2</span>]:
    slope = numerical_derivative(f, x)
    <span class="kw">print</span>(f<span class="st">"x={x}: f'={slope:.4f}"</span>)

<span class="out"># x=-2: f'=-4.0000</span>
<span class="out"># x=-1: f'=-2.0000</span>
<span class="out"># x= 0: f'= 0.0000  ← minimum!</span>
<span class="out"># x= 1: f'= 2.0000</span>
<span class="out"># x= 2: f'= 4.0000</span></pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> When training a model, the loss function
            L(w) depends on weights w. The derivative ∂L/∂w tells us the slope of the loss landscape at our current
            position. <strong>Gradient descent</strong> simply moves in the opposite direction of the slope: w ← w − α ·
            ∂L/∂w. That's it — that's the entire algorithm.</div>
        </div>

        <!-- 05 DERIVATIVE FROM FIRST PRINCIPLES -->
        <a class="section-anchor" id="first-principles"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-green">05</span>
            <h3>Derivative from First Principles <span class="difficulty-pill diff-beginner">Beginner</span></h3>
          </div>

          <div class="concept-intro">
            <strong>The derivative is formally defined as a limit:</strong> f'(x) = lim<sub>h→0</sub> [f(x+h) − f(x)] /
            h. This is called the "first principles" or "limit definition" of the derivative. It measures the slope of a
            secant line as the two points merge into one.
            <div class="mental-model">🧠 Mental model: Draw a secant line between two points on a curve. Now slide the
              second point closer and closer to the first. The secant line rotates and becomes the tangent line in the
              limit. Its slope is the derivative.</div>
          </div>

          <div class="section-flow">
            <span class="flow-step">📖 Concept</span><span class="flow-arrow">›</span>
            <span class="flow-step">📐 Math</span><span class="flow-arrow">›</span>
            <span class="flow-step">💻 NumPy</span><span class="flow-arrow">›</span>
            <span class="flow-step">🐍 Python</span><span class="flow-arrow">›</span>
            <span class="flow-step">🎯 ML Apply</span>
          </div>

          <div class="formula-block">
            <div class="formula-main">f'(x) = lim<sub>h→0</sub> [f(x + h) − f(x)] / h</div>
            <div class="formula-sub">The limit of the difference quotient — slope of the secant as h shrinks to zero
            </div>
          </div>

          <p><strong>Worked example:</strong> Derive f'(x) for f(x) = x² from first principles:</p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2.2;color:var(--text2);">
              f'(x) = lim<sub>h→0</sub> [(x+h)² − x²] / h<br>
              &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;= lim<sub>h→0</sub> [x² + 2xh + h² − x²] / h<br>
              &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;= lim<sub>h→0</sub> [2xh + h²] / h<br>
              &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;= lim<sub>h→0</sub> (2x + h)<br>
              &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;= <strong style="color:var(--accent);">2x</strong> ✓
            </div>
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">Three Numerical Approximations</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> f(x): <span class="kw">return</span> x**<span class="nm">2</span>

x = <span class="nm">3.0</span>; h = <span class="nm">1e-7</span>

<span class="cm"># Forward difference</span>
fwd = (f(x+h) - f(x)) / h
<span class="cm"># Backward difference</span>
bwd = (f(x) - f(x-h)) / h
<span class="cm"># Central difference (best!)</span>
ctr = (f(x+h) - f(x-h)) / (<span class="nm">2</span>*h)

<span class="kw">print</span>(f<span class="st">"Forward:  {fwd:.10f}"</span>)
<span class="kw">print</span>(f<span class="st">"Backward: {bwd:.10f}"</span>)
<span class="kw">print</span>(f<span class="st">"Central:  {ctr:.10f}"</span>)
<span class="kw">print</span>(f<span class="st">"Exact:    {2*x:.10f}"</span>)
<span class="out"># Central is most accurate!</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Full Numerical Gradient</div>
              <pre style="margin:0;"><span class="kw">def</span> numerical_gradient(f, params, h=<span class="nm">1e-7</span>):
    <span class="st">"""Gradient for a list of params"""</span>
    grad = []
    <span class="kw">for</span> i <span class="kw">in</span> range(len(params)):
        <span class="cm"># Perturb param i</span>
        params_plus = params[:]
        params_minus = params[:]
        params_plus[i] += h
        params_minus[i] -= h
        <span class="cm"># Central difference</span>
        g = (f(params_plus) - f(params_minus))
        g /= (<span class="nm">2</span> * h)
        grad.append(g)
    <span class="kw">return</span> grad

<span class="cm"># f(w0, w1) = w0² + 3*w1²</span>
<span class="kw">def</span> loss(w):
    <span class="kw">return</span> w[<span class="nm">0</span>]**<span class="nm">2</span> + <span class="nm">3</span>*w[<span class="nm">1</span>]**<span class="nm">2</span>

g = numerical_gradient(loss, [<span class="nm">2.0</span>, <span class="nm">1.0</span>])
<span class="kw">print</span>(g)  <span class="out"># [4.0, 6.0]</span></pre>
            </div>
          </div>

          <div class="callout callout-green">💡 <strong>Gradient checking:</strong> Before trusting your backprop
            implementation, compare its gradients against the numerical gradient from first principles. If they differ
            by more than ~1e-5, you have a bug. This is called <strong>gradient checking</strong> and is a standard
            debugging technique.</div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> The <code>numerical_gradient</code>
            function above is exactly what <strong>gradient checking</strong> does in practice. Frameworks like PyTorch
            provide <code>torch.autograd.gradcheck()</code> which computes both the analytical gradient (via backprop)
            and the numerical gradient, then asserts they match.</div>
        </div>

        <!-- 06 POWER RULE -->
        <a class="section-anchor" id="power-rule"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-blue">06</span>
            <h3>Power Rule <span class="difficulty-pill diff-beginner">Beginner</span></h3>
          </div>

          <div class="concept-intro">
            <strong>The power rule is the most used differentiation rule:</strong> bring down the exponent as a
            coefficient, then subtract one from the exponent. It handles all polynomials.
            <div class="mental-model">🧠 Mental model: x³ → bring down the 3, reduce to x²: 3x². Simple as moving the
              exponent to the front and decrementing.</div>
          </div>

          <div class="formula-block">
            <div class="formula-main">d/dx [xⁿ] = n · x<sup>n−1</sup></div>
            <div class="formula-sub">Works for any real n — integers, fractions, negatives</div>
          </div>

          <p><strong>Examples:</strong></p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2.2;color:var(--text2);">
              d/dx [x³] = 3x²<br>
              d/dx [x⁻¹] = −x⁻² = −1/x²<br>
              d/dx [√x] = d/dx [x<sup>1/2</sup>] = (1/2)x<sup>−1/2</sup> = 1/(2√x)<br>
              d/dx [5] = 0 &nbsp;<em style="color:var(--text3);">(constant → derivative is 0)</em>
            </div>
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="cm"># f(x) = 3x⁴ + 2x² − 7x + 5</span>
<span class="cm"># f'(x) = 12x³ + 4x − 7</span>

<span class="kw">def</span> f(x):
    <span class="kw">return</span> <span class="nm">3</span>*x**<span class="nm">4</span> + <span class="nm">2</span>*x**<span class="nm">2</span> - <span class="nm">7</span>*x + <span class="nm">5</span>

<span class="kw">def</span> f_prime(x):
    <span class="kw">return</span> <span class="nm">12</span>*x**<span class="nm">3</span> + <span class="nm">4</span>*x - <span class="nm">7</span>

x = np.array([<span class="nm">0</span>, <span class="nm">1</span>, <span class="nm">2</span>])
<span class="kw">print</span>(f_prime(x))  <span class="out"># [-7, 9, 97]</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">def</span> power_rule(coeff, exp):
    <span class="st">"""Apply power rule: d/dx [c·x^n]"""</span>
    <span class="kw">if</span> exp == <span class="nm">0</span>:
        <span class="kw">return</span> (<span class="nm">0</span>, <span class="nm">0</span>)  <span class="cm"># constant</span>
    new_coeff = coeff * exp
    new_exp = exp - <span class="nm">1</span>
    <span class="kw">return</span> (new_coeff, new_exp)

<span class="cm"># Differentiate 3x⁴</span>
c, e = power_rule(<span class="nm">3</span>, <span class="nm">4</span>)
<span class="kw">print</span>(f<span class="st">"{c}x^{e}"</span>)
<span class="out"># 12x^3</span>

<span class="cm"># Differentiate √x = x^0.5</span>
c, e = power_rule(<span class="nm">1</span>, <span class="nm">0.5</span>)
<span class="kw">print</span>(f<span class="st">"{c}x^{e}"</span>)
<span class="out"># 0.5x^-0.5</span></pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> L2 regularization adds λ||w||² to the
            loss. Its derivative is 2λw — a direct application of the power rule. This is why L2 regularization is also
            called <strong>weight decay</strong>: it adds a gradient that pushes weights toward zero.</div>
        </div>

        <!-- 07 SUM, DIFFERENCE & CONSTANT RULES -->
        <a class="section-anchor" id="sum-const-rule"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-blue">07</span>
            <h3>Sum, Difference & Constant Rules <span class="difficulty-pill diff-beginner">Beginner</span></h3>
          </div>

          <div class="concept-intro">
            <strong>Differentiation is linear:</strong> the derivative of a sum is the sum of derivatives, and you can
            pull constants out front. This lets you differentiate any polynomial term by term.
          </div>

          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.2;">
              d/dx [f(x) + g(x)] = f'(x) + g'(x) &nbsp;<em style="font-size:12px;color:var(--text3);">Sum rule</em><br>
              d/dx [f(x) − g(x)] = f'(x) − g'(x) &nbsp;<em style="font-size:12px;color:var(--text3);">Difference
                rule</em><br>
              d/dx [c · f(x)] = c · f'(x) &nbsp;<em style="font-size:12px;color:var(--text3);">Constant multiple
                rule</em><br>
              d/dx [c] = 0 &nbsp;<em style="font-size:12px;color:var(--text3);">Constant rule</em>
            </div>
          </div>

          <p><strong>Example:</strong> Differentiate f(x) = 5x³ − 2x² + 7x − 3</p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2;color:var(--text2);">
              f'(x) = 5·3x² − 2·2x + 7·1 − 0 = <strong style="color:var(--accent);">15x² − 4x + 7</strong>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> A loss function is often a <strong>sum
              over all training examples</strong>: L = (1/N)Σ Lᵢ. The linearity of differentiation means ∂L/∂w = (1/N)Σ
            ∂Lᵢ/∂w — we can compute gradients for each example independently and average them. This is why
            <strong>mini-batch gradient descent</strong> works.
          </div>
        </div>

        <!-- 08 PRODUCT & QUOTIENT RULES -->
        <a class="section-anchor" id="product-quotient"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-blue">08</span>
            <h3>Product & Quotient Rules <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>

          <div class="concept-intro">
            <strong>When two functions are multiplied or divided, their derivatives interact.</strong> The product rule
            says both functions contribute to the rate of change. The quotient rule is the product rule applied to
            f(x)/g(x).
          </div>

          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.4;">
              <strong>Product rule:</strong> d/dx [f·g] = f'·g + f·g'<br>
              <strong>Quotient rule:</strong> d/dx [f/g] = (f'·g − f·g') / g²
            </div>
          </div>

          <p><strong>Product rule example:</strong> d/dx [x² · eˣ]</p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2;color:var(--text2);">
              f(x) = x², &nbsp;f'(x) = 2x<br>
              g(x) = eˣ, &nbsp;g'(x) = eˣ<br>
              (fg)' = 2x·eˣ + x²·eˣ = <strong style="color:var(--accent);">eˣ(2x + x²)</strong>
            </div>
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="cm"># f(x) = x² · eˣ</span>
<span class="cm"># f'(x) = eˣ(2x + x²)</span>

<span class="kw">def</span> f(x):
    <span class="kw">return</span> x**<span class="nm">2</span> * np.exp(x)

<span class="kw">def</span> f_prime(x):
    <span class="kw">return</span> np.exp(x) * (<span class="nm">2</span>*x + x**<span class="nm">2</span>)

x = np.array([<span class="nm">0</span>, <span class="nm">1</span>, <span class="nm">2</span>])
<span class="kw">print</span>(f_prime(x))
<span class="out"># [0.0, 8.154, 43.510]</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">import</span> math

<span class="kw">def</span> product_rule(f, f_p, g, g_p, x):
    <span class="st">"""(fg)' = f'g + fg'"""</span>
    <span class="kw">return</span> f_p(x)*g(x) + f(x)*g_p(x)

<span class="cm"># f=x², f'=2x, g=eˣ, g'=eˣ</span>
result = product_rule(
    <span class="kw">lambda</span> x: x**<span class="nm">2</span>,
    <span class="kw">lambda</span> x: <span class="nm">2</span>*x,
    <span class="kw">lambda</span> x: math.exp(x),
    <span class="kw">lambda</span> x: math.exp(x),
    <span class="nm">1.0</span>
)
<span class="kw">print</span>(result)  <span class="out"># 8.1548</span></pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> The product rule appears when
            differentiating attention scores in Transformers (Q·Kᵀ), gated mechanisms in LSTMs (forget gate × cell
            state), and any layer that multiplies two learned quantities.</div>
        </div>

        <!-- 09 CHAIN RULE -->
        <a class="section-anchor" id="chain-rule"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">09</span>
            <h3>Chain Rule <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>

          <div class="concept-intro">
            <strong>The chain rule is THE most important rule in all of machine learning.</strong> It tells us how to
            differentiate <em>composed functions</em> — functions inside functions. Since a neural network is just a
            chain of composed functions, backpropagation is literally the chain rule applied recursively.
            <div class="mental-model">🧠 Mental model: Imagine a production line in a factory. If the raw material
              speeds up by 2×, the first machine produces 3× faster, and the second machine produces 4× faster, the
              total speedup is 2 × 3 × 4 = 24×. You multiply the rates through each stage — that's the chain rule.</div>
          </div>

          <div class="section-flow">
            <span class="flow-step">📖 Concept</span><span class="flow-arrow">›</span>
            <span class="flow-step">📐 Visual</span><span class="flow-arrow">›</span>
            <span class="flow-step">💻 NumPy</span><span class="flow-arrow">›</span>
            <span class="flow-step">🐍 Python</span><span class="flow-arrow">›</span>
            <span class="flow-step">🎯 ML Apply</span>
          </div>

          <div class="formula-block">
            <div class="formula-main">d/dx [f(g(x))] = f'(g(x)) · g'(x)</div>
            <div class="formula-sub">"Derivative of the outer function (evaluated at the inner) × derivative of the
              inner function"</div>
          </div>

          <div class="by-hand-note">The chain rule powers ALL of backpropagation!</div>
          <div class="mermaid">

            graph LR
            subgraph Chain
            X[Input x] -->|g| G[Inner Function]
            G -->|f| F[Outer Function]
            F -->|dy/dx| O[Final Derivative]
            end
          </div>

          <p><strong>Worked example:</strong> d/dx [sin(x²)]</p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2;color:var(--text2);">
              outer: f(u) = sin(u) → f'(u) = cos(u)<br>
              inner: g(x) = x² → g'(x) = 2x<br>
              chain rule: f'(g(x)) · g'(x) = cos(x²) · 2x = <strong style="color:var(--accent);">2x·cos(x²)</strong>
            </div>
          </div>

          <p><strong>Multi-layer chain:</strong> d/dx [f(g(h(x)))] = f'(g(h(x))) · g'(h(x)) · h'(x)</p>
          <div style="background:var(--amber-bg);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-size:13px;color:var(--amber);line-height:1.7;">
              This is <strong>exactly</strong> how backpropagation works through a neural network with 3 layers. Each
              layer is a function, and the chain rule multiplies the local derivatives through all layers.
            </div>
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="cm"># f(x) = sigmoid(3x + 2)</span>
<span class="cm"># outer: sigmoid(u), inner: 3x+2</span>

<span class="kw">def</span> sigmoid(x):
    <span class="kw">return</span> <span class="nm">1</span> / (<span class="nm">1</span> + np.exp(-x))

<span class="kw">def</span> sigmoid_prime(x):
    s = sigmoid(x)
    <span class="kw">return</span> s * (<span class="nm">1</span> - s)

<span class="cm"># Chain rule: σ'(3x+2) · 3</span>
x = np.array([<span class="nm">0</span>, <span class="nm">1</span>, <span class="nm">2</span>])
inner = <span class="nm">3</span>*x + <span class="nm">2</span>
deriv = sigmoid_prime(inner) * <span class="nm">3</span>
<span class="kw">print</span>(deriv)
<span class="out"># [0.315, 0.044, 0.005]</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python — General Chain Rule</div>
              <pre style="margin:0;"><span class="kw">def</span> chain_rule(outer_deriv, inner_deriv,
              inner_fn, x):
    <span class="st">"""d/dx f(g(x)) = f'(g(x))·g'(x)"""</span>
    u = inner_fn(x)        <span class="cm"># g(x)</span>
    df_du = outer_deriv(u)  <span class="cm"># f'(g(x))</span>
    du_dx = inner_deriv(x)  <span class="cm"># g'(x)</span>
    <span class="kw">return</span> df_du * du_dx

<span class="kw">import</span> math
<span class="cm"># d/dx sin(x²) = cos(x²)·2x</span>
result = chain_rule(
    <span class="kw">lambda</span> u: math.cos(u),  <span class="cm"># f'</span>
    <span class="kw">lambda</span> x: <span class="nm">2</span>*x,          <span class="cm"># g'</span>
    <span class="kw">lambda</span> x: x**<span class="nm">2</span>,         <span class="cm"># g</span>
    <span class="nm">1.0</span>
)
<span class="kw">print</span>(result)  <span class="out"># ≈ 1.0806</span></pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> A 3-layer neural network computes y =
            σ(W₃ · σ(W₂ · σ(W₁ · x + b₁) + b₂) + b₃). To find ∂Loss/∂W₁, we apply the chain rule through all 3 layers.
            This recursive application IS backpropagation — invented by Rumelhart, Hinton & Williams in 1986.
            <strong>The chain rule is all you need.</strong>
          </div>
        </div>

                <td>1/(x·ln(a))</td>
                <td>Information theory</td>
              </tr>
            </tbody>
          </table>

          <div class="callout callout-green">💡 <strong>Pro tip:</strong> You only need to truly memorize 5 derivatives
            for ML: <code>xⁿ → nxⁿ⁻¹</code>, <code>eˣ → eˣ</code>, <code>ln(x) → 1/x</code>, <code>σ(x) → σ(1−σ)</code>,
            and <code>ReLU → step function</code>. Everything else can be derived from these using the chain rule.</div>
        </div>

        <!-- 11 SIGMOID -->
        <a class="section-anchor" id="sigmoid"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-amber">11</span>
            <h3>Sigmoid & Its Derivative <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>

          <div class="concept-intro">
            <strong>The sigmoid function squashes any input into (0, 1).</strong> Its derivative has a beautiful
            property: σ'(x) = σ(x)(1 − σ(x)). The derivative is maximized at x=0 (where σ=0.5) and vanishes for large
            |x|. This "saturation" causes the <strong>vanishing gradient problem</strong>.
            <div class="mental-model">🧠 Mental model: Sigmoid is like a dimmer switch. At x=0 it's at 50% brightness,
              and the switch is most sensitive (high derivative). At extreme values, it's either fully off or fully on,
              and turning the knob barely changes anything (near-zero derivative).</div>
          </div>

          <div class="section-flow">
            <span class="flow-step">📖 Concept</span><span class="flow-arrow">›</span>
            <span class="flow-step">📐 Visual</span><span class="flow-arrow">›</span>
            <span class="flow-step">💻 NumPy</span><span class="flow-arrow">›</span>
            <span class="flow-step">🐍 Python</span><span class="flow-arrow">›</span>
            <span class="flow-step">🎯 ML Apply</span>
          </div>

          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.2;">
              σ(x) = 1 / (1 + e⁻ˣ)<br>
              σ'(x) = σ(x) · (1 − σ(x))<br>
              <span style="font-size:12px;color:var(--text3);">Max derivative: σ'(0) = 0.25</span>
            </div>
          </div>

          <p><strong>Derivation using chain rule:</strong></p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2;color:var(--text2);">
              σ(x) = (1 + e⁻ˣ)⁻¹<br>
              σ'(x) = −1·(1 + e⁻ˣ)⁻² · (−e⁻ˣ) &nbsp;<em style="color:var(--text3);">chain rule</em><br>
              &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;= e⁻ˣ / (1 + e⁻ˣ)²<br>
              &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;= [1/(1 + e⁻ˣ)] · [e⁻ˣ/(1 + e⁻ˣ)]<br>
              &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;= σ(x) · [1 − σ(x)] &nbsp;✓
            </div>
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> sigmoid(x):
    <span class="kw">return</span> <span class="nm">1</span> / (<span class="nm">1</span> + np.exp(-x))

<span class="kw">def</span> sigmoid_deriv(x):
    s = sigmoid(x)
    <span class="kw">return</span> s * (<span class="nm">1</span> - s)

x = np.linspace(-<span class="nm">6</span>, <span class="nm">6</span>, <span class="nm">5</span>)
<span class="kw">print</span>(<span class="st">"σ(x):  "</span>, np.round(sigmoid(x), <span class="nm">4</span>))
<span class="kw">print</span>(<span class="st">"σ'(x): "</span>, np.round(sigmoid_deriv(x), <span class="nm">4</span>))
<span class="out"># σ(x):   [0.0025, 0.0474, 0.5, 0.9526, 0.9975]</span>
<span class="out"># σ'(x):  [0.0025, 0.0452, 0.25, 0.0452, 0.0025]</span>
<span class="cm"># ↑ Derivative peaks at 0.25 (x=0) and vanishes at extremes!</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">import</span> math

    <span class="kw">if</span> x >= <span class="nm">0</span>:

        <span class="kw">return</span> <span class="nm">1</span> / (<span class="nm">1</span> + z)
    <span class="kw">else</span>:
        z = math.exp(x)
        <span class="kw">return</span> z / (<span class="nm">1</span> + z)

    <span class="kw">return</span> s * (<span class="nm">1</span> - s)

<span class="kw">print</span>(sigmoid_deriv(<span class="nm">0</span>))   <span class="out"># 0.25</span>

<span class="kw">print</span>(sigmoid_deriv(-<span class="nm">5</span>))  <span class="out"># 0.0066</span></pre>
            </div>
          </div>

          <div class="callout callout-amber">⚠️ <strong>Vanishing gradient problem:</strong> The maximum value of σ'(x)
            is only <strong>0.25</strong>. In a 10-layer network, gradients get multiplied through each layer: 0.25¹⁰ =
            0.00000095. The gradient essentially <strong>disappears</strong> — early layers barely learn. This is why

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Sigmoid is still used in (1)

        <!-- 12 TANH -->
        <a class="section-anchor" id="tanh"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-amber">12</span>
            <h3>Tanh & Its Derivative <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>

          <div class="concept-intro">
            <strong>Tanh is a "centered" sigmoid:</strong> it maps inputs to (−1, 1) instead of (0, 1). This
            zero-centering helps gradient descent converge faster. Its derivative: tanh'(x) = 1 − tanh²(x). Maximum
            derivative is 1.0 (at x=0) — better than sigmoid's 0.25.
          </div>

          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.2;">
              tanh(x) = (eˣ − e⁻ˣ) / (eˣ + e⁻ˣ)<br>
              tanh'(x) = 1 − tanh²(x)<br>
              <span style="font-size:12px;color:var(--text3);">Note: tanh(x) = 2σ(2x) − 1 (scaled sigmoid!)</span>
            </div>
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> tanh(x):
    <span class="kw">return</span> np.tanh(x)

<span class="kw">def</span> tanh_deriv(x):
    t = np.tanh(x)
    <span class="kw">return</span> <span class="nm">1</span> - t**<span class="nm">2</span>

x = np.array([-<span class="nm">2</span>, -<span class="nm">1</span>, <span class="nm">0</span>, <span class="nm">1</span>, <span class="nm">2</span>])
<span class="kw">print</span>(<span class="st">"tanh:  "</span>, np.round(tanh(x), <span class="nm">4</span>))
<span class="kw">print</span>(<span class="st">"tanh': "</span>, np.round(tanh_deriv(x), <span class="nm">4</span>))
<span class="out"># tanh:   [-0.9640, -0.7616, 0.0, 0.7616, 0.9640]</span>
<span class="out"># tanh':  [0.0707, 0.4200, 1.0, 0.4200, 0.0707]</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">import</span> math

<span class="kw">def</span> tanh(x):
    ep = math.exp(x)
    em = math.exp(-x)
    <span class="kw">return</span> (ep - em) / (ep + em)

<span class="kw">def</span> tanh_deriv(x):
    t = tanh(x)
    <span class="kw">return</span> <span class="nm">1</span> - t**<span class="nm">2</span>

<span class="cm"># Verify: max deriv at x=0 is 1.0</span>
<span class="kw">print</span>(tanh_deriv(<span class="nm">0</span>))  <span class="out"># 1.0</span>
<span class="kw">print</span>(tanh_deriv(<span class="nm">3</span>))  <span class="out"># 0.0099</span></pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Tanh is preferred over sigmoid in
            <strong>RNN/LSTM hidden states</strong> because its output is zero-centered. In Transformers, tanh appears
            in <strong>GELU</strong> (the default activation): GELU(x) ≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³))).
          </div>
        </div>

        <!-- 13 ReLU FAMILY -->
        <a class="section-anchor" id="relu"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-amber">13</span>
            <h3>ReLU Family <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>

          <div class="concept-intro">
            <strong>ReLU (Rectified Linear Unit) solved the vanishing gradient problem.</strong> Its derivative is
            either 0 or 1 — no shrinking! This lets deep networks train effectively. Variants like Leaky ReLU, ELU, and
            GELU address ReLU's weaknesses.
            <div class="mental-model">🧠 Mental model: ReLU is a gate that blocks negative signals and passes positive
              ones unchanged. If a neuron's input is negative, it outputs 0 (dead neuron). If positive, it passes the
              signal straight through with no distortion.</div>
          </div>

          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:13px;line-height:2.4;">
              <strong>ReLU:</strong> f(x) = max(0, x) &nbsp;→&nbsp; f'(x) = 0 if x<0, 1 if x>0<br>
                <strong>Leaky ReLU:</strong> f(x) = max(αx, x) &nbsp;→&nbsp; f'(x) = α if x<0, 1 if x>0 &nbsp;<em
                    style="color:var(--text3);">(α≈0.01)</em><br>
                  <strong>ELU:</strong> f(x) = x if x>0, α(eˣ−1) if x≤0 &nbsp;→&nbsp; f'(x) = 1 if x>0, f(x)+α if
                  x≤0<br>
                  <strong>GELU:</strong> f(x) = x · Φ(x) &nbsp;<em style="color:var(--text3);">(Φ = standard normal CDF,
                    used in GPT/BERT)</em>
            </div>
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy — All Variants</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> relu(x): <span class="kw">return</span> np.maximum(<span class="nm">0</span>, x)
<span class="kw">def</span> relu_deriv(x): <span class="kw">return</span> (x > <span class="nm">0</span>).astype(float)

<span class="kw">def</span> leaky_relu(x, a=<span class="nm">0.01</span>):
    <span class="kw">return</span> np.where(x > <span class="nm">0</span>, x, a*x)
<span class="kw">def</span> leaky_relu_deriv(x, a=<span class="nm">0.01</span>):
    <span class="kw">return</span> np.where(x > <span class="nm">0</span>, <span class="nm">1</span>, a)

<span class="kw">def</span> gelu(x):
    <span class="kw">return</span> <span class="nm">0.5</span>*x*(<span class="nm">1</span>+np.tanh(
        np.sqrt(<span class="nm">2</span>/np.pi)*(x+<span class="nm">0.044715</span>*x**<span class="nm">3</span>)))

x = np.array([-<span class="nm">2</span>, -<span class="nm">1</span>, <span class="nm">0</span>, <span class="nm">1</span>, <span class="nm">2</span>])
<span class="kw">print</span>(<span class="st">"ReLU:  "</span>, relu(x))
<span class="out"># [0, 0, 0, 1, 2]</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">def</span> relu(x):
    <span class="kw">return</span> max(<span class="nm">0</span>, x)

<span class="kw">def</span> relu_deriv(x):
    <span class="kw">return</span> <span class="nm">1</span> <span class="kw">if</span> x > <span class="nm">0</span> <span class="kw">else</span> <span class="nm">0</span>

<span class="kw">def</span> leaky_relu(x, alpha=<span class="nm">0.01</span>):
    <span class="kw">return</span> x <span class="kw">if</span> x > <span class="nm">0</span> <span class="kw">else</span> alpha * x

<span class="kw">def</span> leaky_relu_deriv(x, alpha=<span class="nm">0.01</span>):
    <span class="kw">return</span> <span class="nm">1</span> <span class="kw">if</span> x > <span class="nm">0</span> <span class="kw">else</span> alpha

<span class="cm"># The "dying ReLU" problem:</span>
<span class="cm"># If a neuron always receives</span>
<span class="cm"># negative input, its gradient</span>
<span class="cm"># is always 0 → it never learns!</span>
<span class="cm"># Leaky ReLU fixes this by</span>
<span class="cm"># allowing a small gradient (α)</span>
<span class="cm"># for negative inputs.</span></pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> <strong>ReLU</strong> is the default for
            hidden layers in CNNs and MLPs. <strong>GELU</strong> is the default for Transformers (GPT, BERT).
            <strong>Leaky ReLU</strong> is preferred when you suspect dying neurons. <strong>ELU</strong> sometimes
            gives faster convergence by producing negative outputs.
          </div>
        </div>

        <!-- 14 SOFTMAX & CROSS-ENTROPY -->
        <a class="section-anchor" id="softmax"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-amber">14</span>
            <h3>Softmax & Cross-Entropy <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>

          <div class="concept-intro">
            <strong>Softmax converts a vector of raw scores (logits) into probabilities that sum to 1.</strong> Combined
            with cross-entropy loss, the gradient simplifies beautifully to ŷ − y (predicted minus true). This elegance
            is why softmax + cross-entropy is the standard classification pipeline.
          </div>

          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.2;">
              softmax(zᵢ) = eᶻⁱ / Σⱼ eᶻʲ<br>
              cross-entropy: L = −Σᵢ yᵢ · log(ŷᵢ)<br>
              <span style="font-size:12px;color:var(--text3);">Combined gradient: ∂L/∂zᵢ = ŷᵢ − yᵢ (beautifully
                simple!)</span>
            </div>
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> softmax(z):
    <span class="cm"># Numerically stable</span>
    e = np.exp(z - np.max(z))
    <span class="kw">return</span> e / e.sum()

<span class="kw">def</span> cross_entropy(y_true, y_pred):
    <span class="kw">return</span> -np.sum(y_true * np.log(y_pred + <span class="nm">1e-15</span>))

<span class="cm"># Example: 3-class classification</span>
logits = np.array([<span class="nm">2.0</span>, <span class="nm">1.0</span>, <span class="nm">0.1</span>])
probs = softmax(logits)
<span class="kw">print</span>(probs)
<span class="out"># [0.659, 0.242, 0.099]</span>
<span class="kw">print</span>(probs.sum())  <span class="out"># 1.0</span>

<span class="cm"># Gradient: simply ŷ − y!</span>
y_true = np.array([<span class="nm">1</span>, <span class="nm">0</span>, <span class="nm">0</span>])  <span class="cm"># one-hot</span>
grad = probs - y_true
<span class="kw">print</span>(grad)
<span class="out"># [-0.341, 0.242, 0.099]</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">import</span> math

<span class="kw">def</span> softmax(z):
    <span class="cm"># Subtract max for stability</span>
    m = max(z)
    exps = [math.exp(zi - m) <span class="kw">for</span> zi <span class="kw">in</span> z]
    total = sum(exps)
    <span class="kw">return</span> [e / total <span class="kw">for</span> e <span class="kw">in</span> exps]

<span class="kw">def</span> cross_entropy(y_true, y_pred):
    <span class="kw">return</span> -sum(
        yt * math.log(yp + <span class="nm">1e-15</span>)
        <span class="kw">for</span> yt, yp <span class="kw">in</span> zip(y_true, y_pred)
    )

logits = [<span class="nm">2.0</span>, <span class="nm">1.0</span>, <span class="nm">0.1</span>]
probs = softmax(logits)
<span class="kw">print</span>([round(p, <span class="nm">3</span>) <span class="kw">for</span> p <span class="kw">in</span> probs])
<span class="out"># [0.659, 0.242, 0.099]</span></pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Softmax + cross-entropy is the
            <strong>output layer of every classification network</strong>. The fact that the gradient simplifies to ŷ −
            y means backprop through the last layer is trivially simple and numerically stable. PyTorch's
            <code>nn.CrossEntropyLoss</code> combines both into one fused operation for efficiency.
          </div>
        </div>

        <!-- 15 LOSS FUNCTIONS & THEIR GRADIENTS -->
        <a class="section-anchor" id="loss-functions"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-amber">15</span>
            <h3>Loss Functions & Their Gradients <span class="difficulty-pill diff-intermediate">Intermediate</span>
            </h3>
          </div>

          <div class="concept-intro">
            <strong>The loss function measures how wrong the model is.</strong> Its gradient tells us how to adjust each
            parameter to reduce that wrongness. Different tasks need different loss functions.
          </div>

          <div class="by-hand-note">Loss → Gradient → Update → Repeat!</div>
          <div class="mermaid">
            graph LR
            subgraph Losses ["Loss Functions by Task"]
            REG["Regression<br />(continuous output)"] --> MSE["MSE Loss<br />L = (1/n)Σ(ŷ−y)²"]
            REG --> MAE["MAE Loss<br />L = (1/n)Σ|ŷ−y|"]
            CLS["Classification<br />(categorical)"] --> BCE["Binary CE<br />L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]"]
            CLS --> CCE["Categorical CE<br />L = −Σ yᵢ·log(ŷᵢ)"]
            end
          </div>

          <table class="deriv-table">
            <thead>
              <tr>
                <th>Loss</th>
                <th>Formula</th>
                <th>Gradient ∂L/∂ŷ</th>
              </tr>
            </thead>
            <tbody>
              <tr>
                <td>MSE</td>
                <td>(ŷ − y)²</td>
                <td>2(ŷ − y)</td>
              </tr>
              <tr>
                <td>MAE</td>
                <td>|ŷ − y|</td>
                <td>sign(ŷ − y)</td>
              </tr>
              <tr>
                <td>Binary CE</td>
                <td>−y·log(ŷ) − (1−y)·log(1−ŷ)</td>
                <td>(ŷ − y) / [ŷ(1 − ŷ)]</td>
              </tr>
              <tr>
                <td>Categorical CE</td>
                <td>−Σ yᵢ·log(ŷᵢ)</td>
                <td>−y/ŷ (then softmax combines to ŷ−y)</td>
              </tr>
              <tr>
                <td>Huber</td>
                <td>½(ŷ−y)² if |ŷ−y|≤δ, else δ|ŷ−y|−½δ²</td>
                <td>(ŷ−y) if |ŷ−y|≤δ, else δ·sign(ŷ−y)</td>
              </tr>
            </tbody>
          </table>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> mse(y_true, y_pred):
    <span class="kw">return</span> np.mean((y_pred - y_true)**<span class="nm">2</span>)

<span class="kw">def</span> mse_grad(y_true, y_pred):
    n = len(y_true)
    <span class="kw">return</span> <span class="nm">2</span> * (y_pred - y_true) / n

<span class="kw">def</span> binary_ce(y_true, y_pred):
    eps = <span class="nm">1e-15</span>
    y_pred = np.clip(y_pred, eps, <span class="nm">1</span>-eps)
    <span class="kw">return</span> -np.mean(
        y_true*np.log(y_pred) +
        (<span class="nm">1</span>-y_true)*np.log(<span class="nm">1</span>-y_pred)
    )

y = np.array([<span class="nm">1</span>, <span class="nm">0</span>, <span class="nm">1</span>])
yh = np.array([<span class="nm">0.9</span>, <span class="nm">0.1</span>, <span class="nm">0.8</span>])
<span class="kw">print</span>(f<span class="st">"MSE:  {mse(y, yh):.4f}"</span>)
<span class="kw">print</span>(f<span class="st">"BCE:  {binary_ce(y, yh):.4f}"</span>)</pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">import</span> math

<span class="kw">def</span> mse(y_true, y_pred):
    n = len(y_true)
    <span class="kw">return</span> sum((yp-yt)**<span class="nm">2</span>
        <span class="kw">for</span> yt,yp <span class="kw">in</span> zip(y_true,y_pred)) / n

<span class="kw">def</span> mse_grad(y_true, y_pred):
    n = len(y_true)
    <span class="kw">return</span> [<span class="nm">2</span>*(yp-yt)/n
        <span class="kw">for</span> yt,yp <span class="kw">in</span> zip(y_true,y_pred)]

<span class="kw">def</span> binary_ce(y_true, y_pred):
    eps = <span class="nm">1e-15</span>
    n = len(y_true)
    total = <span class="nm">0</span>
    <span class="kw">for</span> yt, yp <span class="kw">in</span> zip(y_true, y_pred):
        yp = max(eps, min(<span class="nm">1</span>-eps, yp))
        total -= (yt*math.log(yp) +
                  (<span class="nm">1</span>-yt)*math.log(<span class="nm">1</span>-yp))
    <span class="kw">return</span> total / n</pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Choosing the right loss function is one
            of the most important design decisions: <strong>MSE</strong> for regression, <strong>Cross-Entropy</strong>
            for classification, <strong>Huber</strong> for robust regression. The gradient of the loss function is what
            flows backward through the network during training.</div>
        </div>

        <!-- 16 THE INTEGRAL -->
        <a class="section-anchor" id="integral"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-purple">16</span>
            <h3>The Integral (Area Under the Curve) <span class="difficulty-pill diff-intermediate">Intermediate</span>
            </h3>
          </div>

          <div class="concept-intro">
            <strong>If the derivative measures the rate of change, the integral measures the total accumulated
              change.</strong> Geometrically, the definite integral ∫ₐᵇ f(x)dx is the signed area between f(x) and the
            x-axis from a to b.
            <div class="mental-model">🧠 Mental model: If your speedometer (derivative) shows 60 mph for 2 hours, the
              integral tells you the total distance: 120 miles. Integration sums up infinitely many tiny slices of
              "speed × time".</div>
          </div>

          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.2;">
              <strong>Definite integral:</strong> ∫ₐᵇ f(x) dx = F(b) − F(a) &nbsp;<em
                style="font-size:12px;color:var(--text3);">where F'(x) = f(x)</em><br>
              <strong>Indefinite integral:</strong> ∫ f(x) dx = F(x) + C
            </div>
          </div>

          <div class="callout callout-bg2" style="margin-top: 1rem;">
            <strong>Mathematical Explanation & Example:</strong>
            <p style="margin-top: 0.5rem; margin-bottom: 0.5rem;">An integral computes the total accumulation of a
              quantity, visualized as the area under a curve. If a function <code>f(x)</code> describes the rate of
              change, its integral <code>F(x)</code> (the antiderivative) gives the total amount.</p>
            <p style="margin-bottom: 0.5rem;"><strong>Example:</strong> Calculate the definite integral of
              <code>f(x) = 2x</code> from <code>x = 0</code> to <code>x = 3</code>.
            </p>
            <ul style="margin-left: 1.5rem; margin-bottom: 0;">
              <li>First, find the indefinite integral. What function's derivative is <code>2x</code>? The power rule in
                reverse tells us <code>F(x) = x²</code>.</li>
              <li>Using the Fundamental Theorem of Calculus: <code>∫₀³ 2x dx = F(3) - F(0)</code>.</li>
              <li>Evaluate at the bounds: <code>(3)² - (0)² = 9 - 0 = 9</code>.</li>
              <li>The total area under the curve <code>y = 2x</code> between <code>x = 0</code> and <code>x = 3</code>
                is exactly 9.</li>
            </ul>
            <div id="integral-area-plot"
              style="width:100%; height:300px; margin-top: 1rem; border: 1px solid var(--border); border-radius: var(--radius); overflow: hidden;">
            </div>
          </div>

          <div class="by-hand-note">Integration undoes differentiation!</div>
          <div class="mermaid">
            graph LR
            subgraph IntDiff ["The Fundamental Connection"]
            F["F(x)<br />Antiderivative"] -->|"Differentiate<br />d/dx"| f["f(x)<br />Function"]
            f -->|"Integrate<br />∫ dx"| F
            end
          </div>

          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy/SciPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="cm"># ∫₀² x² dx = [x³/3]₀² = 8/3 ≈ 2.667</span>
result, error = integrate.quad(
    <span class="kw">lambda</span> x: x**<span class="nm">2</span>, <span class="nm">0</span>, <span class="nm">2</span>
)
<span class="kw">print</span>(f<span class="st">"{result:.4f}"</span>)  <span class="out"># 2.6667</span>

            <div>
              <div class="code-heading python-heading">Pure Python — Riemann Sums</div>
              <pre style="margin:0;"><span class="kw">def</span> riemann_sum(f, a, b, n=<span class="nm">1000</span>):
    <span class="st">"""Left Riemann sum approximation"""</span>
    dx = (b - a) / n
    total = <span class="nm">0</span>
    <span class="kw">for</span> i <span class="kw">in</span> range(n):
        x = a + i * dx
        total += f(x) * dx
    <span class="kw">return</span> total

<span class="kw">def</span> trapezoidal(f, a, b, n=<span class="nm">1000</span>):
    <span class="st">"""More accurate than Riemann"""</span>
    dx = (b - a) / n
    total = (f(a) + f(b)) / <span class="nm">2</span>
    <span class="kw">for</span> i <span class="kw">in</span> range(<span class="nm">1</span>, n):
        total += f(a + i * dx)
    <span class="kw">return</span> total * dx

<span class="cm"># ∫₀² x² dx = 8/3</span>
result = trapezoidal(
    <span class="kw">lambda</span> x: x**<span class="nm">2</span>, <span class="nm">0</span>, <span class="nm">2</span>
)
<span class="kw">print</span>(f<span class="st">"{result:.6f}"</span>)
<span class="out"># 2.666667</span></pre>
            </div>
          </div>

          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Integrals are everywhere in probability:
            the probability of an event is the integral of a PDF over a region. <strong>Expected values</strong> E[f(X)]
            = ∫f(x)p(x)dx are integrals. <strong>KL divergence</strong> between two distributions is an integral. Monte
            Carlo sampling is how we approximate integrals that have no closed form.</div>

        <!-- 17 FUNDAMENTAL THEOREM OF CALCULUS -->
        <a class="section-anchor" id="ftc"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-purple">17</span>
            <h3>Fundamental Theorem of Calculus <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>
          <div class="concept-intro">
            <strong>The FTC bridges differentiation and integration:</strong> if F'(x) = f(x), then ∫ₐᵇ f(x)dx = F(b) −
            F(a). Differentiation and integration are inverse operations.
            <div class="mental-model">🧠 Mental model: If you know the speed at every moment (derivative), integrating
              gives total distance (antiderivative). If you know total distance at every moment, differentiating gives
              speed.</div>
          </div>
          <div class="formula-block">
            <div class="formula-main">d/dx [∫ₐˣ f(t) dt] = f(x)</div>
            <div class="formula-sub">Part 1: The derivative of an integral is the original function</div>
          </div>
          <div class="formula-block">
            <div class="formula-main">∫ₐᵇ f(x) dx = F(b) − F(a)</div>
            <div class="formula-sub">Part 2: Evaluate a definite integral using any antiderivative F</div>
          </div>
          <p><strong>Common antiderivatives:</strong></p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2.2;color:var(--text2);">
              ∫ xⁿ dx = x<sup>n+1</sup>/(n+1) + C &nbsp;<em style="color:var(--text3);">(n ≠ −1)</em><br>
              ∫ eˣ dx = eˣ + C<br>
              ∫ 1/x dx = ln|x| + C<br>
              ∫ cos(x) dx = sin(x) + C<br>
              ∫ sin(x) dx = −cos(x) + C
            </div>
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="cm"># Verify FTC: ∫₀³ 2x dx = [x²]₀³ = 9</span>
<span class="cm"># Antiderivative F(x) = x²</span>
F = <span class="kw">lambda</span> x: x**<span class="nm">2</span>
exact = F(<span class="nm">3</span>) - F(<span class="nm">0</span>)
<span class="kw">print</span>(exact)  <span class="out"># 9</span>

<span class="cm"># Numerical check</span>
x = np.linspace(<span class="nm">0</span>, <span class="nm">3</span>, <span class="nm">10000</span>)
numerical = np.trapz(<span class="nm">2</span>*x, x)
<span class="kw">print</span>(f<span class="st">"{numerical:.4f}"</span>)  <span class="out"># 9.0000</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">def</span> integrate_poly(coeffs, a, b):
    <span class="st">"""Exact integral of polynomial
    coeffs: [c0, c1, c2, ...]
    p(x) = c0 + c1*x + c2*x² + ...
    """</span>
    <span class="cm"># Antiderivative coeffs</span>
    F_coeffs = [<span class="nm">0</span>]  <span class="cm"># constant term</span>
    <span class="kw">for</span> i, c <span class="kw">in</span> enumerate(coeffs):
        F_coeffs.append(c / (i + <span class="nm">1</span>))

    <span class="kw">def</span> F(x):
        <span class="kw">return</span> sum(c * x**(i)
            <span class="kw">for</span> i, c <span class="kw">in</span> enumerate(F_coeffs))

    <span class="kw">return</span> F(b) - F(a)

<span class="cm"># ∫₀³ 2x dx (coeffs = [0, 2])</span>
<span class="kw">print</span>(integrate_poly([<span class="nm">0</span>, <span class="nm">2</span>], <span class="nm">0</span>, <span class="nm">3</span>))
<span class="out"># 9.0</span></pre>
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> The FTC guarantees that if a loss
            function has a known antiderivative, we can compute exact expectations. In <strong>variational autoencoders
              (VAEs)</strong>, the ELBO involves integrals over latent distributions — we use the FTC when possible and
            Monte Carlo when not.</div>
        </div>

          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-purple">18</span>
            <h3>Integration Techniques <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Two essential techniques:</strong> <em>substitution</em> (the chain rule in reverse) and
            <em>integration by parts</em> (the product rule in reverse). Most ML integrals are solved numerically, but
            these techniques help build intuition.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.4;">
              <strong>Substitution:</strong> ∫ f(g(x))·g'(x) dx = ∫ f(u) du &nbsp;<em
                style="font-size:12px;color:var(--text3);">where u = g(x)</em><br>
              <strong>By Parts:</strong> ∫ u·dv = u·v − ∫ v·du
            </div>
          </div>
          <p><strong>Substitution example:</strong> ∫ 2x·eˣ² dx</p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2;color:var(--text2);">
              Let u = x², &nbsp;du = 2x dx<br>
              ∫ eᵘ du = eᵘ + C = <strong style="color:var(--accent);">eˣ² + C</strong> ✓
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Substitution appears when computing the
            <strong>reparameterization trick</strong> in VAEs: we substitute z = μ + σε to move the gradient inside an
            expectation. Integration by parts is used in deriving the <strong>REINFORCE</strong> gradient estimator in
            reinforcement learning.
          </div>
        </div>

            <h3>Numerical Integration <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Most ML integrals have no closed-form solution.</strong> We approximate them numerically using
            Riemann sums, the trapezoidal rule, or Simpson's rule — or statistically using Monte Carlo sampling.
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy — Simpson's Rule</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> simpsons(f, a, b, n=<span class="nm">1000</span>):
    <span class="st">"""Simpson's 1/3 rule (most accurate)"""</span>
    <span class="kw">assert</span> n % <span class="nm">2</span> == <span class="nm">0</span>
    x = np.linspace(a, b, n+<span class="nm">1</span>)
    y = f(x)
    h = (b - a) / n
    <span class="kw">return</span> h/<span class="nm">3</span> * (y[<span class="nm">0</span>] + y[-<span class="nm">1</span>]
        + <span class="nm">4</span>*np.sum(y[<span class="nm">1</span>:-<span class="nm">1</span>:<span class="nm">2</span>])
        + <span class="nm">2</span>*np.sum(y[<span class="nm">2</span>:-<span class="nm">2</span>:<span class="nm">2</span>]))

<span class="cm"># ∫₀π sin(x) dx = 2</span>
result = simpsons(np.sin, <span class="nm">0</span>, np.pi)
<span class="kw">print</span>(f<span class="st">"{result:.10f}"</span>)
<span class="out"># 2.0000000000</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python — Monte Carlo</div>
              <pre style="margin:0;"><span class="kw">import</span> random, math

<span class="kw">def</span> monte_carlo_integrate(f, a, b, n=<span class="nm">100000</span>):
    <span class="st">"""Approximate ∫ₐᵇ f(x) dx
    by sampling random points"""</span>
    total = <span class="nm">0</span>
    <span class="kw">for</span> _ <span class="kw">in</span> range(n):
        x = random.uniform(a, b)
        total += f(x)
    <span class="kw">return</span> (b - a) * total / n

<span class="cm"># ∫₀π sin(x) dx = 2</span>
result = monte_carlo_integrate(
    math.sin, <span class="nm">0</span>, math.pi
)
<span class="kw">print</span>(f<span class="st">"{result:.4f}"</span>)
<span class="out"># ≈ 2.0000 (stochastic!)</span></pre>
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> <strong>Monte Carlo sampling</strong> is
            the backbone of modern ML: training GANs, estimating expectations in VAEs, computing policy gradients in RL,
            and MCMC for Bayesian inference. When you can't compute an integral exactly, you sample.</div>

        <!-- 20 INTEGRALS IN ML -->
        <a class="section-anchor" id="integrals-ml"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-purple">20</span>
            <h3>Integrals in Machine Learning <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Integrals show up in three critical ML contexts:</strong> (1) computing expected values, (2)
            measuring distances between probability distributions (KL divergence), and (3) normalizing probability
            distributions.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.4;">
              <strong>Expected value:</strong> E[f(X)] = ∫ f(x)·p(x) dx<br>
              <strong>KL divergence:</strong> D<sub>KL</sub>(P||Q) = ∫ p(x)·log(p(x)/q(x)) dx<br>
              <strong>Normalization:</strong> ∫ p(x) dx = 1
            </div>
          </div>
          <div class="by-hand-note">Where integrals hide in ML</div>
          <div class="mermaid">
            graph TD
            subgraph IntML ["Integrals in Machine Learning"]
            EV["Expected Value<br />E[X] = ∫x·p(x)dx"] --> LOSS["Expected Loss<br />Risk minimization"]
            KL["KL Divergence<br />∫p·log(p/q)dx"] --> VAE["VAE Training<br />(ELBO objective)"]
            NORM["Normalization<br />∫p(x)dx = 1"] --> SOFT["Softmax<br />Σ eᶻⁱ = partition fn"]
            end
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">KL Divergence — NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> kl_divergence(p, q):
    <span class="st">"""KL(P||Q) for discrete dists"""</span>
    p = np.asarray(p, dtype=float)
    q = np.asarray(q, dtype=float)
    <span class="cm"># Avoid log(0)</span>
    mask = p > <span class="nm">0</span>
    <span class="kw">return</span> np.sum(p[mask] * np.log(p[mask] / q[mask]))

p = [<span class="nm">0.4</span>, <span class="nm">0.3</span>, <span class="nm">0.3</span>]
q = [<span class="nm">0.33</span>, <span class="nm">0.33</span>, <span class="nm">0.34</span>]
<span class="kw">print</span>(f<span class="st">"KL(P||Q) = {kl_divergence(p, q):.4f}"</span>)
<span class="out"># KL(P||Q) = 0.0133</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Expected Value — Pure Python</div>
              <pre style="margin:0;"><span class="kw">import</span> math

<span class="kw">def</span> expected_value(values, probs):
    <span class="st">"""E[X] = Σ xᵢ · p(xᵢ)"""</span>
    <span class="kw">return</span> sum(x*p <span class="kw">for</span> x,p
               <span class="kw">in</span> zip(values, probs))

<span class="cm"># Fair die: E[X] = 3.5</span>
vals = [<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>,<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]
probs = [<span class="nm">1</span>/<span class="nm">6</span>]*<span class="nm">6</span>
<span class="kw">print</span>(expected_value(vals, probs))
<span class="out"># 3.5</span>

<span class="cm"># Variance: E[X²] − E[X]²</span>
ex2 = expected_value([x**<span class="nm">2</span> <span class="kw">for</span> x <span class="kw">in</span> vals], probs)
ex = expected_value(vals, probs)
<span class="kw">print</span>(f<span class="st">"Var = {ex2 - ex**2:.4f}"</span>)
<span class="out"># Var = 2.9167</span></pre>
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> The <strong>ELBO</strong> in VAEs =
            E<sub>q</sub>[log p(x|z)] − D<sub>KL</sub>(q(z|x) || p(z)). Both terms are integrals — the first is
            estimated via Monte Carlo sampling, the second has a closed form for Gaussians. Understanding integrals is
            essential for generative models.</div>
        </div>

        <a class="section-anchor" id="double-integrals"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-purple">20.5</span>
            <h3>Double Integrals <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>A double integral integrates a function of two variables over a 2D region.</strong> While a single
            integral computes the area under a 1D curve, a double integral computes the <em>volume</em> under a 2D
            surface <code>z = f(x, y)</code>.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.4;">
              <strong>Double Integral:</strong> ∬<sub>R</sub> f(x, y) dA = ∫<sub>a</sub><sup>b</sup>
              ∫<sub>c</sub><sup>d</sup> f(x, y) dy dx
            </div>
          </div>
          <div class="callout callout-bg2" style="margin-top: 1rem;">
            <strong>Mathematical Explanation & Example:</strong>
            <p style="margin-top: 0.5rem; margin-bottom: 0.5rem;">To compute a double integral over a rectangular
              region, we perform <strong>iterated integration</strong>: integrate with respect to one variable first
              (treating the other as a constant), then integrate the result with respect to the second variable.</p>
            <p style="margin-bottom: 0.5rem;"><strong>Example:</strong> Calculate the volume under
              <code>f(x, y) = xy</code> for <code>x</code> from 0 to 2, and <code>y</code> from 0 to 1.
            </p>
            <ul style="margin-left: 1.5rem; margin-bottom: 0;">
              <li>Set up the integral: <code>∫₀² ( ∫₀¹ xy dy ) dx</code></li>
              <li><strong>Inner integral (w.r.t y):</strong> Treat <code>x</code> as a constant.
                <ul>
                  <li>The integral of <code>y</code> is <code>y²/2</code>. So,
                    <code>∫₀¹ xy dy = x * [y²/2]₀¹ = x * (1/2 - 0) = x/2</code>.
                  </li>
                </ul>
              </li>
              <li style="margin-top: 0.5rem;"><strong>Outer integral (w.r.t x):</strong> Now integrate the result w.r.t
                <code>x</code>.
                <ul>
                  <li><code>∫₀² (x/2) dx = (1/2) * [x²/2]₀² = (1/2) * (2 - 0) = 1</code>.</li>
                </ul>
              </li>
              <li>The total volume under the surface is exactly 1.</li>
            </ul>
            <div class="mermaid" style="margin-top: 1rem;">
              graph LR
              START["∬ xy dy dx"] --> INNER["Inner Integral<br>∫₀¹ xy dy"]
              INNER -->|Treat x as constant| INNER_EVAL["[x · y²/2]₀¹<br>= x/2"]
              INNER_EVAL --> OUTER["Outer Integral<br>∫₀² (x/2) dx"]
              OUTER -->|Standard 1D integral| OUTER_EVAL["[x²/4]₀²<br>= 4/4 = 1"]
              OUTER_EVAL --> VOL["Volume = 1"]

              style START fill:var(--surface),stroke:var(--border),color:var(--text)
              style VOL fill:var(--purple-bg),stroke:var(--purple),color:var(--purple)
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Double integrals (and multiple integrals
            in general) appear frequently in continuous probability. To find the probability that two continuous random
            variables fall within a certain region, you compute the double integral of their joint probability density
            function <code>p(x, y)</code>. They are also used in marginalization (e.g., integrating out a variable
            <code>p(x) = ∫ p(x,y) dy</code>).
          </div>
        </div>

        <a class="section-anchor" id="partial-derivatives"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-teal">21</span>
            <h3>Partial Derivatives <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>
          <div class="concept-intro">
            <strong>A partial derivative measures how a function changes when you vary ONE variable while holding all
              others constant.</strong> For f(x, y), ∂f/∂x treats y as a constant and differentiates with respect to x.
            <div class="mental-model">🧠 Mental model: Imagine standing on a hilly landscape (z = f(x,y)). ∂f/∂x is the
              slope if you walk purely East. ∂f/∂y is the slope if you walk purely North. Together they tell you the
              steepest uphill direction.</div>
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.2;">
              ∂f/∂x = lim<sub>h→0</sub> [f(x+h, y) − f(x, y)] / h<br>
              <span style="font-size:12px;color:var(--text3);">Treat y as a constant, differentiate with respect to x
                only</span>
            </div>
          </div>
          <div class="callout callout-bg2" style="margin-top: 1rem;">
            <strong>Mathematical Explanation & Example:</strong>
            <p style="margin-top: 0.5rem; margin-bottom: 0.5rem;">A partial derivative is used when a function depends
              on multiple variables, but we only want to see how it changes with respect to <em>one</em> variable,
              holding all others constant.</p>
            <p style="margin-bottom: 0.5rem;"><strong>Example:</strong> Let <code>f(x, y) = 3x²y + 2xy³ - 5</code></p>
            <ul style="margin-left: 1.5rem; margin-bottom: 0;">
              <li><strong>To find ∂f/∂x (Partial w.r.t x):</strong> Treat <code>y</code> as a constant number (like 4 or
                10).
                <ul>
                  <li>The derivative of <code>3x²y</code> w.r.t x is <code>6xy</code> (the <code>3y</code> is just a
                    constant multiplier of <code>x²</code>).</li>
                  <li>The derivative of <code>2xy³</code> w.r.t x is <code>2y³</code> (the <code>2y³</code> is a
                    constant multiplier of <code>x</code>).</li>
                  <li>The derivative of <code>-5</code> is 0.</li>
                  <li>So, <code>∂f/∂x = 6xy + 2y³</code>.</li>
                </ul>
              </li>
              <li style="margin-top: 0.5rem;"><strong>To find ∂f/∂y (Partial w.r.t y):</strong> Treat <code>x</code> as
                a constant number.
                <ul>
                  <li>The derivative of <code>3x²y</code> w.r.t y is <code>3x²</code> (the <code>3x²</code> is a
                    constant multiplier of <code>y</code>).</li>
                  <li>The derivative of <code>2xy³</code> w.r.t y is <code>2x * 3y² = 6xy²</code>.</li>
                  <li>So, <code>∂f/∂y = 3x² + 6xy²</code>.</li>
                </ul>
              </li>
            </ul>
            <div class="mermaid" style="margin-top: 1rem;">
              graph TD
              F["f(x, y) = 3x²y + 2xy³ - 5"]
              F -->|∂/∂x| X["Hold y constant"]
              F -->|∂/∂y| Y["Hold x constant"]
              X --> X1["d(3x²y)/dx = 6xy
              d(2xy³)/dx = 2y³
              d(-5)/dx = 0"]
              Y --> Y1["d(3x²y)/dy = 3x²
              d(2xy³)/dy = 6xy²
              d(-5)/dy = 0"]
              X1 --> RES_X["∂f/∂x = 6xy + 2y³"]
              Y1 --> RES_Y["∂f/∂y = 3x² + 6xy²"]
            </div>
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre
                style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> f(x, y):
    <span class="kw">return</span> <span class="nm">3</span>*x**<span class="nm">2</span>*y + <span class="nm">2</span>*x*y**<span class="nm">3</span> - <span class="nm">5</span>

<span class="kw">def</span> partial_x(f, x, y, h=<span class="nm">1e-7</span>):
    <span class="kw">return</span> (f(x+h, y) - f(x-h, y)) / (<span class="nm">2</span>*h)

<span class="kw">def</span> partial_y(f, x, y, h=<span class="nm">1e-7</span>):
    <span class="kw">return</span> (f(x, y+h) - f(x, y-h)) / (<span class="nm">2</span>*h)

<span class="kw">print</span>(partial_x(f, <span class="nm">1</span>, <span class="nm">2</span>))  <span class="out"># 28.0 (6·1·2 + 2·8)</span>
<span class="kw">print</span>(partial_y(f, <span class="nm">1</span>, <span class="nm">2</span>))  <span class="out"># 27.0 (3·1 + 6·1·4)</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">def</span> partial_derivative(f, args, idx, h=<span class="nm">1e-7</span>):
    <span class="st">"""Partial deriv w.r.t. args[idx]"""</span>
    args_p = list(args)
    args_m = list(args)
    args_p[idx] += h
    args_m[idx] -= h
    <span class="kw">return</span> (f(*args_p) - f(*args_m)) / (<span class="nm">2</span>*h)

<span class="kw">def</span> f(x, y):
    <span class="kw">return</span> <span class="nm">3</span>*x**<span class="nm">2</span>*y + <span class="nm">2</span>*x*y**<span class="nm">3</span> - <span class="nm">5</span>

<span class="cm"># ∂f/∂x at (1,2)</span>
<span class="kw">print</span>(partial_derivative(f, [<span class="nm">1</span>,<span class="nm">2</span>], <span class="nm">0</span>))
<span class="out"># 28.0</span>
<span class="cm"># ∂f/∂y at (1,2)</span>
<span class="kw">print</span>(partial_derivative(f, [<span class="nm">1</span>,<span class="nm">2</span>], <span class="nm">1</span>))
<span class="out"># 27.0</span></pre>
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Each weight wᵢⱼ in a neural network gets
            its own partial derivative ∂L/∂wᵢⱼ. With millions of weights, backpropagation efficiently computes ALL
            partial derivatives in a single backward pass — much faster than computing each one individually.</div>
        </div>

        <!-- 22 GRADIENT VECTOR -->
        <a class="section-anchor" id="gradient-vector"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-teal">22</span>
            <h3>The Gradient Vector <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>
          <div class="concept-intro">
            <strong>The gradient ∇f bundles ALL partial derivatives into a single vector.</strong> It points in the
            direction of steepest ascent. Its negative, −∇f, points in the direction of steepest descent — which is
            exactly where gradient descent goes.
          </div>
          <div class="formula-block">
            <div class="formula-main">∇f = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]</div>
            <div class="formula-sub">The gradient is a vector of all partial derivatives — it points "uphill"</div>
          </div>
          <div class="by-hand-note">∇f always points uphill — negate it to go downhill!</div>
          <div class="mermaid">
            graph TD
            subgraph Grad ["Gradient Properties"]
            DIR["Direction<br />Points toward steepest ascent"] --> NEG["Negate it<br />−∇f = steepest DESCENT"]
            MAG["Magnitude<br />||∇f|| = steepness of slope"] --> FLAT["||∇f|| ≈ 0<br />means flat (near minimum)"]
            end
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> gradient(f, params, h=<span class="nm">1e-7</span>):
    <span class="st">"""Compute ∇f numerically"""</span>
    grad = np.zeros_like(params, dtype=float)
    <span class="kw">for</span> i <span class="kw">in</span> range(len(params)):
        p_plus = params.copy(); p_plus[i] += h
        p_minus = params.copy(); p_minus[i] -= h
        grad[i] = (f(p_plus) - f(p_minus)) / (<span class="nm">2</span>*h)
    <span class="kw">return</span> grad

<span class="cm"># f(w) = w₀² + 3w₁² (bowl)</span>
f = <span class="kw">lambda</span> w: w[<span class="nm">0</span>]**<span class="nm">2</span> + <span class="nm">3</span>*w[<span class="nm">1</span>]**<span class="nm">2</span>
w = np.array([<span class="nm">4.0</span>, <span class="nm">2.0</span>])
<span class="kw">print</span>(gradient(f, w))  <span class="out"># [8.0, 12.0]</span>
<span class="cm"># Exact: [2w₀, 6w₁] = [8, 12] ✓</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">def</span> gradient(f, params, h=<span class="nm">1e-7</span>):
    <span class="st">"""Compute ∇f for list of params"""</span>
    grad = []
    <span class="kw">for</span> i <span class="kw">in</span> range(len(params)):
        p_plus = params[:]
        p_minus = params[:]
        p_plus[i] += h
        p_minus[i] -= h
        g = (f(p_plus) - f(p_minus)) / (<span class="nm">2</span>*h)
        grad.append(g)
    <span class="kw">return</span> grad

<span class="kw">def</span> f(w):
    <span class="kw">return</span> w[<span class="nm">0</span>]**<span class="nm">2</span> + <span class="nm">3</span>*w[<span class="nm">1</span>]**<span class="nm">2</span>

<span class="kw">print</span>(gradient(f, [<span class="nm">4.0</span>, <span class="nm">2.0</span>]))
<span class="out"># [8.0, 12.0]</span></pre>
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> When you call
            <code>loss.backward()</code>, PyTorch computes ∇L with respect to every parameter in the model. The
            optimizer then uses this gradient to update each parameter. The gradient vector is THE central object in all
            of machine learning optimization.
          </div>
        </div>

        <!-- 23 GRADIENT DESCENT -->
        <a class="section-anchor" id="gradient-descent"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">23</span>
            <h3>Gradient Descent <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Gradient descent is THE optimization algorithm of deep learning.</strong> It iteratively adjusts
            parameters in the direction opposite to the gradient. Three words: compute gradient → step downhill →
            repeat.
            <div class="mental-model">🧠 Mental model: You're blindfolded on a mountain. To find the valley, feel the
              slope under your feet (gradient) and take a step downhill (negative gradient). Repeat until flat
              (minimum).</div>
          </div>
          <div class="section-flow">
            <span class="flow-step">📖 Concept</span><span class="flow-arrow">›</span>
            <span class="flow-step">📐 Visual</span><span class="flow-arrow">›</span>
            <span class="flow-step">💻 NumPy</span><span class="flow-arrow">›</span>
            <span class="flow-step">🐍 Python</span><span class="flow-arrow">›</span>
            <span class="flow-step">🎯 ML Apply</span>
          </div>
          <div class="formula-block">
            <div class="formula-main">w ← w − α · ∇L(w)</div>
            <div class="formula-sub">α = learning rate · ∇L = gradient of loss · w = parameters</div>
          </div>
          <div class="callout callout-bg2" style="margin-top: 1rem;">
            <strong>Mathematical Explanation & Example:</strong>
            <p style="margin-top: 0.5rem; margin-bottom: 0.5rem;">Let's trace one step of Gradient Descent to minimize
              <code>f(w₀, w₁) = w₀² + 3w₁²</code>.
            </p>
            <ul style="margin-left: 1.5rem; margin-bottom: 0;">
              <li><strong>1. Compute the Gradient:</strong> The partial derivatives are <code>∂f/∂w₀ = 2w₀</code> and
                <code>∂f/∂w₁ = 6w₁</code>. So, <code>∇f(w) = [2w₀, 6w₁]</code>.
              </li>
              <li><strong>2. Pick a Starting Point & Learning Rate:</strong> Let's start at <code>w = [4, 2]</code> with
                learning rate <code>α = 0.1</code>.</li>
              <li><strong>3. Evaluate Gradient at Start:</strong> <code>∇f([4, 2]) = [2(4), 6(2)] = [8, 12]</code>.</li>
              <li><strong>4. Take a Step:</strong> <code>w_new = w - α · ∇f(w)</code><br>
                <code>w_new = [4, 2] - 0.1 · [8, 12]</code><br>
                <code>w_new = [4, 2] - [0.8, 1.2] = [3.2, 0.8]</code>
              </li>
              <li><strong>Result:</strong> We moved from <code>[4, 2]</code> to <code>[3.2, 0.8]</code>, getting much
                closer to the true minimum at <code>[0, 0]</code>!</li>
            </ul>
          </div>
          <div class="by-hand-note">The ENTIRE training loop in one line!</div>
          <div class="mermaid">
            graph TD
            subgraph GD ["Gradient Descent Loop"]
            INIT["Initialize weights w<br />randomly"] --> FWD["Forward Pass<br />ŷ = model(x)"]
            FWD --> LOSS["Compute Loss<br />L = loss(ŷ, y)"]
            LOSS --> GRAD["Compute Gradient<br />∇L = ∂L/∂w"]
            GRAD --> UPDATE["Update Weights<br />w ← w − α·∇L"]
            UPDATE -->|Repeat| FWD
            end
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy — Full GD</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> gradient_descent(f, grad_f, w0,
                     lr=<span class="nm">0.01</span>, epochs=<span class="nm">100</span>):
    w = w0.copy()
    history = [f(w)]
    <span class="kw">for</span> _ <span class="kw">in</span> range(epochs):
        g = grad_f(w)
        w = w - lr * g
        history.append(f(w))
    <span class="kw">return</span> w, history

<span class="cm"># Minimize f(w) = w₀² + 3w₁²</span>
f = <span class="kw">lambda</span> w: w[<span class="nm">0</span>]**<span class="nm">2</span> + <span class="nm">3</span>*w[<span class="nm">1</span>]**<span class="nm">2</span>
grad = <span class="kw">lambda</span> w: np.array([<span class="nm">2</span>*w[<span class="nm">0</span>], <span class="nm">6</span>*w[<span class="nm">1</span>]])

w_opt, hist = gradient_descent(
    f, grad, np.array([<span class="nm">4.0</span>, <span class="nm">2.0</span>]),
    lr=<span class="nm">0.1</span>, epochs=<span class="nm">50</span>)
<span class="kw">print</span>(f<span class="st">"w = {w_opt}"</span>)
<span class="kw">print</span>(f<span class="st">"f(w) = {f(w_opt):.6f}"</span>)
<span class="out"># w ≈ [0.0, 0.0]  f(w) ≈ 0.0</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">def</span> gradient_descent(f, grad_f, w0,
                     lr=<span class="nm">0.01</span>, epochs=<span class="nm">100</span>):

    <span class="kw">for</span> _ <span class="kw">in</span> range(epochs):

        w = [wi - lr*gi
             <span class="kw">for</span> wi, gi <span class="kw">in</span> zip(w, g)]

        <!-- 24 LEARNING RATE & CONVERGENCE -->
        <a class="section-anchor" id="learning-rate"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-teal">24</span>
            <h3>Learning Rate & Convergence <span class="difficulty-pill diff-intermediate">Intermediate</span></h3>
          </div>
          <div class="concept-intro">
            <strong>The learning rate α controls how big each step is.</strong> Too large → overshoot and diverge. Too
            small → converge painfully slowly. Finding the right learning rate is one of the most important
            hyperparameter choices in ML.
          </div>
          <div class="grid-3" style="margin:1rem 0;">
            <div style="background:var(--red-bg);border-radius:var(--radius);padding:1rem;text-align:center;">
              <div style="font-size:13px;font-weight:600;color:var(--red);margin-bottom:4px;">α too large</div>
              <div style="font-size:12px;color:var(--text2);">Overshoots minimum, loss oscillates or diverges to
                infinity</div>
            </div>
            <div style="background:var(--green-bg);border-radius:var(--radius);padding:1rem;text-align:center;">
              <div style="font-size:13px;font-weight:600;color:var(--green);margin-bottom:4px;">α just right</div>
              <div style="font-size:12px;color:var(--text2);">Smooth convergence to minimum in reasonable time</div>
            </div>
            <div style="background:var(--amber-bg);border-radius:var(--radius);padding:1rem;text-align:center;">
              <div style="font-size:13px;font-weight:600;color:var(--amber);margin-bottom:4px;">α too small</div>
              <div style="font-size:12px;color:var(--text2);">Converges but takes forever; may get stuck in local
                minimum</div>
            </div>
          </div>
          <p><strong>Common schedules:</strong></p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-family:var(--mono);font-size:13px;line-height:2.2;color:var(--text2);">
              <strong>Step decay:</strong> α = α₀ × 0.1 every 30 epochs<br>
              <strong>Exponential:</strong> α = α₀ × e<sup>−kt</sup><br>
              <strong>Cosine annealing:</strong> α = α<sub>min</sub> + ½(α<sub>max</sub>−α<sub>min</sub>)(1 +
              cos(πt/T))<br>
              <strong>Warmup:</strong> linearly increase α from 0 to α₀ over first N steps
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Modern practice: start with lr=3e-4 for
            Adam, lr=0.1 for SGD. Use <strong>cosine annealing with warmup</strong> (the default in most Transformer
            training). The <code>torch.optim.lr_scheduler</code> module provides all common schedules.</div>
        </div>

        <a class="section-anchor" id="sgd-momentum"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">25</span>
            <h3>SGD, Mini-Batch & Momentum <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Full-batch GD is slow.</strong> Stochastic GD (SGD) uses one random sample per step — fast but
            noisy. <strong>Mini-batch GD</strong> (the standard) uses a batch of 32-512 samples — balancing speed and
            stability. <strong>Momentum</strong> adds a velocity term that accelerates through ravines.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2.4;">
              <strong>SGD with Momentum:</strong><br>
              v ← β·v + ∇L(w) &nbsp;<em style="font-size:12px;color:var(--text3);">accumulate velocity (β≈0.9)</em><br>
              w ← w − α·v &nbsp;<em style="font-size:12px;color:var(--text3);">step in velocity direction</em>
            </div>
          </div>
          <div class="callout callout-bg2" style="margin-top: 1rem;">
            <strong>Mathematical Explanation & Example:</strong>
            <p style="margin-top: 0.5rem; margin-bottom: 0.5rem;">Let's trace one step of Momentum with
              <code>β = 0.9</code> and <code>α = 0.1</code>. Assume our current velocity is
              <code>v = [1.0, -0.5]</code>, and the current gradient is <code>g = [0.2, 2.0]</code>.
            </p>
            <ul style="margin-left: 1.5rem; margin-bottom: 0;">
              <li><strong>1. Update Velocity:</strong> <code>v_new = β·v + g</code><br>
                <code>v_new = 0.9 · [1.0, -0.5] + [0.2, 2.0]</code><br>
                <code>v_new = [0.9, -0.45] + [0.2, 2.0] = [1.1, 1.55]</code><br>
                <em>Notice how the velocity "remembers" the past direction but adds the new gradient's push.</em>
              </li>
              <li><strong>2. Update Weights:</strong> <code>w_new = w - α·v_new</code><br>
                <code>w_new = w - 0.1 · [1.1, 1.55] = w - [0.11, 0.155]</code>
              </li>
            </ul>
            <div class="mermaid" style="margin-top: 1rem;">

        <!-- 26 ADAM OPTIMIZER -->
        <a class="section-anchor" id="adam"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">26</span>
            <h3>Adam Optimizer <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Adam = Adaptive Moment Estimation.</strong> It combines momentum (first moment) with RMSProp (second
            moment) to give each parameter its own adaptive learning rate. Adam is the default optimizer for most deep
            learning tasks.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:13px;line-height:2.2;">
              m ← β₁·m + (1−β₁)·g &nbsp;<em style="color:var(--text3);">(first moment — mean of gradients)</em><br>
              v ← β₂·v + (1−β₂)·g² &nbsp;<em style="color:var(--text3);">(second moment — variance of
                gradients)</em><br>
              m̂ = m / (1 − β₁ᵗ) &nbsp;<em style="color:var(--text3);">(bias correction)</em><br>
              v̂ = v / (1 − β₂ᵗ)<br>
              w ← w − α · m̂ / (√v̂ + ε) &nbsp;<em style="color:var(--text3);">(update)</em>
            </div>
          </div>
          <div class="by-hand-note">Adam adapts the learning rate per-parameter!</div>
          <div class="mermaid">
            graph LR
            subgraph Opt ["Optimizer Evolution"]
            SGD["SGD<br />w−α·g"] -->|"Add velocity"| MOM["Momentum<br />v=βv+g, w−αv"]
            MOM -->|"Add per-param lr"| ADAM["Adam<br />Adaptive lr per param"]
            end
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy — Full Adam</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> adam(grad_f, w0, lr=<span class="nm">0.001</span>,
        b1=<span class="nm">0.9</span>, b2=<span class="nm">0.999</span>, eps=<span class="nm">1e-8</span>,
        epochs=<span class="nm">200</span>):
    w = w0.copy()
    m = np.zeros_like(w)
    v = np.zeros_like(w)
    <span class="kw">for</span> t <span class="kw">in</span> range(<span class="nm">1</span>, epochs+<span class="nm">1</span>):
        g = grad_f(w)
        m = b1*m + (<span class="nm">1</span>-b1)*g
        v = b2*v + (<span class="nm">1</span>-b2)*g**<span class="nm">2</span>
        m_hat = m / (<span class="nm">1</span> - b1**t)
        v_hat = v / (<span class="nm">1</span> - b2**t)
        w = w - lr * m_hat / (np.sqrt(v_hat) + eps)
    <span class="kw">return</span> w

grad = <span class="kw">lambda</span> w: np.array([<span class="nm">2</span>*w[<span class="nm">0</span>], <span class="nm">6</span>*w[<span class="nm">1</span>]])
w = adam(grad, np.array([<span class="nm">4.0</span>, <span class="nm">2.0</span>]))
<span class="kw">print</span>(np.round(w, <span class="nm">4</span>))  <span class="out"># [0, 0]</span></pre>
            </div>

        <a class="section-anchor" id="jacobian"></a>

          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">27</span>
            <h3>Jacobian Matrix <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>The Jacobian is the matrix of ALL first-order partial derivatives</strong> for a vector-valued
            function. If f: ℝⁿ → ℝᵐ, the Jacobian J is an m×n matrix where J<sub>ij</sub> = ∂fᵢ/∂xⱼ.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2;">
              J = [∂f₁/∂x₁ ∂f₁/∂x₂ ··· ∂f₁/∂xₙ]<br>
              &nbsp;&nbsp;&nbsp;&nbsp;[∂f₂/∂x₁ ∂f₂/∂x₂ ··· ∂f₂/∂xₙ]<br>
              &nbsp;&nbsp;&nbsp;&nbsp;[&nbsp;&nbsp;⋮&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;⋮&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;⋮&nbsp;&nbsp;]<br>
              &nbsp;&nbsp;&nbsp;&nbsp;[∂fₘ/∂x₁ ∂fₘ/∂x₂ ··· ∂fₘ/∂xₙ]<br>
              <span style="font-size:12px;color:var(--text3);">Shape: (outputs × inputs) = (m × n)</span>
            </div>
          </div>
          <div class="callout callout-bg2" style="margin-top: 1rem;">
            <strong>Mathematical Explanation & Example:</strong>
            <p style="margin-top: 0.5rem; margin-bottom: 0.5rem;">Let's find the Jacobian for the function
              <code>f(x, y) = [x² + y, xy]</code>. Here, we have two inputs <code>(x, y)</code> and two outputs
              <code>(f₁, f₂)</code>.
            </p>
            <ul style="margin-left: 1.5rem; margin-bottom: 0;">
              <li><strong>Output 1:</strong> <code>f₁(x, y) = x² + y</code>.
                <br>Its partial derivatives are <code>∂f₁/∂x = 2x</code> and <code>∂f₁/∂y = 1</code>. This forms the
                first row.
              </li>
              <li><strong>Output 2:</strong> <code>f₂(x, y) = xy</code>.
                <br>Its partial derivatives are <code>∂f₂/∂x = y</code> and <code>∂f₂/∂y = x</code>. This forms the
                second row.
              </li>
              <li><strong>The Jacobian Matrix J is:</strong><br>
                <code>[[ 2x,  1 ]]<br>[[  y,  x ]]</code>
              </li>
              <li><strong>Evaluate at (3, 2):</strong> Plugging in <code>x=3, y=2</code> gives:<br>
                <code>[[ 2(3), 1 ]]<br>[[  2,   3 ]]</code> = <code>[[ 6, 1 ], [ 2, 3 ]]</code>
              </li>
            </ul>
            <div class="mermaid" style="margin-top: 1rem;">
              graph LR

              J1["Row 1: ∇f₁"] -.->|"∂f₁/∂x, ∂f₁/∂y"| F
              J2["Row 2: ∇f₂"] -.->|"∂f₂/∂x, ∂f₂/∂y"| F

              style X fill:#f8f9fa,stroke:#999999
              style F fill:#f8f9fa,stroke:#999999
            </div>
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> jacobian(f, x, h=<span class="nm">1e-7</span>):
    <span class="st">"""Numerical Jacobian matrix"""</span>
    x = np.asarray(x, dtype=float)
    f0 = np.asarray(f(x))
    n = len(x); m = len(f0)
    J = np.zeros((m, n))
    <span class="kw">for</span> j <span class="kw">in</span> range(n):
        xp = x.copy(); xp[j] += h
        xm = x.copy(); xm[j] -= h
        J[:, j] = (f(xp) - f(xm)) / (<span class="nm">2</span>*h)
    <span class="kw">return</span> J

<span class="cm"># f(x,y) = [x²+y, xy]</span>
f = <span class="kw">lambda</span> x: np.array([x[<span class="nm">0</span>]**<span class="nm">2</span>+x[<span class="nm">1</span>], x[<span class="nm">0</span>]*x[<span class="nm">1</span>]])
J = jacobian(f, [<span class="nm">3.0</span>, <span class="nm">2.0</span>])
<span class="kw">print</span>(J)
<span class="out"># [[6. 1.]    ← [2x, 1]</span>
<span class="out">#  [2. 3.]]  ← [y, x]</span></pre>

    f0 = f(x)
    m = len(f0)
    J = [[<span class="nm">0.0</span>]*n <span class="kw">for</span> _ <span class="kw">in</span> range(m)]

              networks (PINNs)</strong>, the Jacobian encodes how outputs change with respect to inputs — essential for
            enforcing differential equations.</div>

        <!-- 28 HESSIAN MATRIX -->
        <a class="section-anchor" id="hessian"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">28</span>
            <h3>Hessian Matrix <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>The Hessian is the matrix of second-order partial derivatives.</strong> It captures the
            <em>curvature</em> of a function. If the Hessian is positive definite at a critical point, that point is a
            minimum. The Hessian also tells us about the conditioning of the optimization landscape.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2;">
              H<sub>ij</sub> = ∂²f / ∂xᵢ∂xⱼ<br>
              <span style="font-size:12px;color:var(--text3);">Hessian is symmetric (H = Hᵀ) for smooth
                functions</span><br>
              <span style="font-size:12px;color:var(--text3);">Newton's method: w ← w − H⁻¹ · ∇f (quadratic
                convergence)</span>
            </div>
          </div>
          <div class="callout callout-bg2" style="margin-top: 1rem;">
            <strong>Mathematical Explanation & Example:</strong>
            <p style="margin-top: 0.5rem; margin-bottom: 0.5rem;">Let's calculate the Hessian for
              <code>f(x, y) = x² + 3y²</code>.
            </p>
            <ul style="margin-left: 1.5rem; margin-bottom: 0;">
              <li><strong>1. First Derivatives (Gradient):</strong><br>
                <code>∂f/∂x = 2x</code><br>
                <code>∂f/∂y = 6y</code>
              </li>
              <li><strong>2. Second Derivatives (Hessian entries):</strong>
                <ul>
                  <li>Differentiate <code>2x</code> wrt <code>x</code>: <code>∂²f/∂x² = 2</code></li>
                  <li>Differentiate <code>2x</code> wrt <code>y</code>: <code>∂²f/∂x∂y = 0</code></li>
                  <li>Differentiate <code>6y</code> wrt <code>x</code>: <code>∂²f/∂y∂x = 0</code></li>
                  <li>Differentiate <code>6y</code> wrt <code>y</code>: <code>∂²f/∂y² = 6</code></li>
                </ul>
              </li>
              <li><strong>The Hessian Matrix H is:</strong><br>
                <code>[[ 2, 0 ]]<br>[[ 0, 6 ]]</code>
              </li>
            </ul>
            <div class="mermaid" style="margin-top: 1rem;">
              graph TD
              subgraph Landscape ["Curvature from Hessian"]
              POS["Hessian is Positive Definite<br>(Eigenvalues > 0)"] --> MIN["Local Minimum<br>Looks like a Bowl"]
              NEG["Hessian is Negative Definite<br>(Eigenvalues < 0)"] --> MAX["Local Maximum<br>Looks like a Hill"]
                MIX["Hessian is Indefinite<br>(Mixed Eigenvalues)"] --> SAD["Saddle Point<br>Looks like a Pringles
                chip"]
                end
            </div>
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> hessian(f, x, h=<span class="nm">1e-5</span>):
    <span class="st">"""Numerical Hessian matrix"""</span>
    n = len(x)
    H = np.zeros((n, n))
    <span class="kw">for</span> i <span class="kw">in</span> range(n):
        <span class="kw">for</span> j <span class="kw">in</span> range(n):
            xpp=x.copy(); xpp[i]+=h; xpp[j]+=h
            xpm=x.copy(); xpm[i]+=h; xpm[j]-=h
            xmp=x.copy(); xmp[i]-=h; xmp[j]+=h
            xmm=x.copy(); xmm[i]-=h; xmm[j]-=h
            H[i,j] = (f(xpp)-f(xpm)-f(xmp)+f(xmm))
            H[i,j] /= (<span class="nm">4</span>*h*h)
    <span class="kw">return</span> H

<span class="cm"># f(x,y) = x² + 3y² → H = [[2,0],[0,6]]</span>
f = <span class="kw">lambda</span> x: x[<span class="nm">0</span>]**<span class="nm">2</span> + <span class="nm">3</span>*x[<span class="nm">1</span>]**<span class="nm">2</span>
<span class="kw">print</span>(hessian(f, np.array([<span class="nm">1.0</span>,<span class="nm">1.0</span>])))
<span class="out"># [[2. 0.]</span>
<span class="out">#  [0. 6.]]</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Hessian Eigenvalues</div>
              <pre style="margin:0;"><span class="cm"># Hessian eigenvalues tell us
# about the loss landscape shape:
#
# All positive → local minimum
# All negative → local maximum
# Mixed → saddle point
#
# Condition number (λ_max / λ_min)
# tells us how "elongated" the
# bowl is. High condition number
# → GD oscillates → slow.
# This is why Adam helps: it
# scales each dimension by 1/√v,
# effectively preconditioning.</span>

<span class="kw">import</span> numpy <span class="kw">as</span> np
H = np.array([[<span class="nm">2</span>, <span class="nm">0</span>], [<span class="nm">0</span>, <span class="nm">6</span>]])
eigvals = np.linalg.eigvalsh(H)
<span class="kw">print</span>(f<span class="st">"Eigenvalues: {eigvals}"</span>)
<span class="out"># [2. 6.]</span>
<span class="kw">print</span>(f<span class="st">"Condition: {max(eigvals)/min(eigvals)}"</span>)
<span class="out"># 3.0</span></pre>
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> Second-order optimizers (L-BFGS, natural
            gradient) use Hessian information for faster convergence. In practice, the full Hessian is too expensive
            (O(n²) storage), so methods like <strong>Hessian-free optimization</strong> and <strong>Fisher
              information</strong> approximations are used.</div>
        </div>

          <h2 class="chapter-title">Backpropagation</h2>
        </div>
        <p style="margin-bottom:1.25rem;color:var(--text2);">Backpropagation is the chain rule applied systematically
          through a computational graph. It is the algorithm that makes training deep neural networks possible.</p>

        <div class="learn-objectives">
          <strong>After Part VI you will be able to:</strong>
          <ul>
            <li>Draw a computational graph for any neural network</li>
            <li>Trace the forward and backward passes step by step</li>

        <!-- 29 COMPUTATIONAL GRAPHS -->
        <a class="section-anchor" id="comp-graphs"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">29</span>
            <h3>Computational Graphs <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>A computational graph is a DAG (directed acyclic graph) where nodes are operations and edges carry
              data.</strong> The forward pass flows data left-to-right to compute the output. The backward pass flows
            gradients right-to-left using the chain rule.
          </div>
          <div class="by-hand-note">Forward pass builds the graph; backward pass computes gradients!</div>
          <div class="mermaid">
            graph LR
            subgraph CompGraph ["Simple Neuron: y = σ(wx + b)"]
            X["x (input)"] --> MUL["× (multiply)"]
            W["w (weight)"] --> MUL
            MUL --> ADD["+ (add)"]
            B["b (bias)"] --> ADD
            ADD --> SIG["σ (sigmoid)"]
            SIG --> Y["y (output)"]
            end
          </div>
          <p>Each node in the graph stores two things during training:</p>
          <div class="grid-2" style="margin:1rem 0;">
            <div style="background:var(--accent-bg);border-radius:var(--radius);padding:1rem;">
              <div style="font-size:13px;font-weight:600;color:var(--accent);margin-bottom:4px;">Forward Pass →</div>
              <p style="font-size:12px;color:var(--text2);margin:0;">Each node computes its output from its inputs and
                caches intermediate values (needed for backward pass).</p>
            </div>
            <div style="background:var(--red-bg);border-radius:var(--radius);padding:1rem;">
              <div style="font-size:13px;font-weight:600;color:var(--red);margin-bottom:4px;">← Backward Pass</div>
              <p style="font-size:12px;color:var(--text2);margin:0;">Each node receives ∂L/∂output, computes ∂L/∂input
                using its local derivative, and passes it upstream.</p>
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> PyTorch and TensorFlow build a
            computational graph <strong>dynamically</strong> during the forward pass. When you call
            <code>.backward()</code>, they traverse this graph in reverse to compute all gradients. This is why tensors
            track <code>requires_grad=True</code>.
          </div>
        </div>

        <!-- 30 FORWARD PASS -->
        <a class="section-anchor" id="forward-pass"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">30</span>
            <h3>Forward Pass <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>The forward pass computes the output of the network by propagating input through each
              layer.</strong> At each layer: (1) linear transformation z = Wx + b, then (2) activation a = σ(z). Cache z
            and a — you'll need them for backprop.
          </div>
          <pre><span class="cm"># Forward pass for a 2-layer network</span>
<span class="cm"># Layer 1: z1 = W1·x + b1,  a1 = ReLU(z1)</span>
<span class="cm"># Layer 2: z2 = W2·a1 + b2, a2 = sigmoid(z2)</span>

<span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> forward(x, W1, b1, W2, b2):
    <span class="cm"># Layer 1</span>
    z1 = W1 @ x + b1
    a1 = np.maximum(<span class="nm">0</span>, z1)         <span class="cm"># ReLU</span>

    <span class="cm"># Layer 2</span>
    z2 = W2 @ a1 + b2
    a2 = <span class="nm">1</span> / (<span class="nm">1</span> + np.exp(-z2))     <span class="cm"># Sigmoid</span>

    cache = (x, z1, a1, z2, a2, W1, W2)
    <span class="kw">return</span> a2, cache</pre>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> During inference (prediction), you only
            need the forward pass. During training, you need both forward AND backward. The cache stores everything the
            backward pass needs, which is why training uses ~2-3× more memory than inference.</div>
        </div>

        <!-- 31 BACKWARD PASS -->
        <a class="section-anchor" id="backward-pass"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">31</span>
            <h3>Backward Pass (Backpropagation) <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Backpropagation = chain rule applied in reverse through the computational graph.</strong> Starting
            from ∂L/∂output, each layer computes its local gradients and passes ∂L/∂input to the previous layer.
          </div>
          <div class="by-hand-note">Each node: receive upstream gradient, multiply by local gradient, pass downstream
          </div>
          <div class="mermaid">
            graph RL
            subgraph Backprop ["Backward Pass"]
            dL["∂L/∂ŷ<br />(from loss)"] --> dSIG["σ' node<br />×σ(1−σ)"]
            dSIG --> dADD["+ node<br />pass through"]
            dADD --> dW["∂L/∂w<br />(save for update)"]
            dADD --> dB["∂L/∂b<br />(save for update)"]
            dADD --> dX["∂L/∂x<br />(to prev layer)"]
            end
          </div>
          <pre><span class="cm"># Backward pass — compute all gradients</span>
<span class="kw">def</span> backward(y_true, cache):
    x, z1, a1, z2, a2, W1, W2 = cache
    m = y_true.shape[<span class="nm">0</span>]  <span class="cm"># batch size</span>

    <span class="cm"># Output layer gradient (BCE loss + sigmoid)</span>
    dz2 = a2 - y_true             <span class="cm"># ∂L/∂z2 = ŷ - y</span>
    dW2 = (<span class="nm">1</span>/m) * dz2 @ a1.T       <span class="cm"># ∂L/∂W2</span>
    db2 = (<span class="nm">1</span>/m) * np.sum(dz2, axis=<span class="nm">1</span>, keepdims=<span class="kw">True</span>)

    <span class="cm"># Hidden layer gradient</span>
    da1 = W2.T @ dz2              <span class="cm"># chain rule</span>
    dz1 = da1 * (z1 > <span class="nm">0</span>)          <span class="cm"># ReLU derivative</span>
    dW1 = (<span class="nm">1</span>/m) * dz1 @ x.T
    db1 = (<span class="nm">1</span>/m) * np.sum(dz1, axis=<span class="nm">1</span>, keepdims=<span class="kw">True</span>)

    <span class="kw">return</span> {<span class="st">'dW1'</span>: dW1, <span class="st">'db1'</span>: db1,
            <span class="st">'dW2'</span>: dW2, <span class="st">'db2'</span>: db2}</pre>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> This is exactly what PyTorch does when
            you call <code>loss.backward()</code>. The framework traverses the computational graph in reverse, applying
            the chain rule at each node. All parameter gradients are stored in <code>param.grad</code> and then consumed
            by the optimizer.</div>
        </div>

        <!-- 32 BACKPROP THROUGH 2-LAYER NETWORK -->
        <a class="section-anchor" id="backprop-2layer"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">32</span>
            <h3>Backprop Through a 2-Layer Network <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Let's put it all together.</strong> A complete training loop: initialize → forward → loss → backward
            → update. This is the foundation of ALL neural network training.
          </div>
          <pre><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="cm"># === COMPLETE 2-LAYER NETWORK ===</span>
np.random.seed(<span class="nm">42</span>)

<span class="cm"># Data: XOR problem</span>
X = np.array([[<span class="nm">0</span>,<span class="nm">0</span>],[<span class="nm">0</span>,<span class="nm">1</span>],[<span class="nm">1</span>,<span class="nm">0</span>],[<span class="nm">1</span>,<span class="nm">1</span>]]).T  <span class="cm"># (2, 4)</span>
Y = np.array([[<span class="nm">0</span>,<span class="nm">1</span>,<span class="nm">1</span>,<span class="nm">0</span>]])                    <span class="cm"># (1, 4)</span>

<span class="cm"># Initialize weights</span>
W1 = np.random.randn(<span class="nm">4</span>, <span class="nm">2</span>) * <span class="nm">0.5</span>  <span class="cm"># (4, 2)</span>
b1 = np.zeros((<span class="nm">4</span>, <span class="nm">1</span>))
W2 = np.random.randn(<span class="nm">1</span>, <span class="nm">4</span>) * <span class="nm">0.5</span>  <span class="cm"># (1, 4)</span>
b2 = np.zeros((<span class="nm">1</span>, <span class="nm">1</span>))

lr = <span class="nm">1.0</span>
<span class="kw">for</span> epoch <span class="kw">in</span> range(<span class="nm">10000</span>):
    <span class="cm"># Forward</span>
    z1 = W1 @ X + b1
    a1 = np.maximum(<span class="nm">0</span>, z1)
    z2 = W2 @ a1 + b2
    a2 = <span class="nm">1</span> / (<span class="nm">1</span> + np.exp(-z2))

    <span class="cm"># Loss</span>
    loss = -np.mean(Y*np.log(a2+<span class="nm">1e-8</span>) + (<span class="nm">1</span>-Y)*np.log(<span class="nm">1</span>-a2+<span class="nm">1e-8</span>))

    <span class="cm"># Backward</span>
    m = <span class="nm">4</span>
    dz2 = a2 - Y
    dW2 = (<span class="nm">1</span>/m) * dz2 @ a1.T
    db2 = (<span class="nm">1</span>/m) * np.sum(dz2, axis=<span class="nm">1</span>, keepdims=<span class="kw">True</span>)
    dz1 = (W2.T @ dz2) * (z1 > <span class="nm">0</span>)
    dW1 = (<span class="nm">1</span>/m) * dz1 @ X.T
    db1 = (<span class="nm">1</span>/m) * np.sum(dz1, axis=<span class="nm">1</span>, keepdims=<span class="kw">True</span>)

    <span class="cm"># Update</span>
    W1 -= lr * dW1; b1 -= lr * db1
    W2 -= lr * dW2; b2 -= lr * db2

    <span class="kw">if</span> epoch % <span class="nm">2000</span> == <span class="nm">0</span>:
        <span class="kw">print</span>(f<span class="st">"Epoch {epoch}: loss = {loss:.4f}"</span>)

<span class="cm"># Test predictions</span>
<span class="kw">print</span>(np.round(a2, <span class="nm">2</span>))  <span class="out"># ≈ [0, 1, 1, 0] — XOR solved!</span></pre>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> This 40-line script IS a complete neural
            network. Every deep learning framework (PyTorch, TensorFlow, JAX) automates exactly these steps.
            Understanding this from scratch means you truly understand how neural networks learn.</div>
        </div>

        <!-- 33 VANISHING & EXPLODING GRADIENTS -->
        <a class="section-anchor" id="vanishing-gradients"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-red">33</span>
            <h3>Vanishing & Exploding Gradients <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>In deep networks, gradients are multiplied through many layers (chain rule).</strong> If each
            layer's local gradient is < 1, the product shrinks exponentially (vanishing). If> 1, it grows exponentially
              (exploding). Both prevent learning.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:13px;line-height:2;">
              ∂L/∂w₁ = ∂L/∂aₙ · ∂aₙ/∂aₙ₋₁ · ... · ∂a₂/∂a₁ · ∂a₁/∂w₁<br>
              <span style="font-size:12px;color:var(--text3);">n multiplications → (0.25)ⁿ for sigmoid → vanishes for
                large n</span>
            </div>
          </div>
          <p><strong>Solutions:</strong></p>
          <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;">
            <div style="font-size:13px;color:var(--text2);line-height:2;">
              <strong>1. ReLU activation:</strong> derivative is 0 or 1 → no shrinking<br>
              <strong>2. Skip connections (ResNet):</strong> gradient flows through shortcut paths<br>
              <strong>3. Batch normalization:</strong> keeps activations in well-conditioned range<br>
              <strong>4. Gradient clipping:</strong> cap gradient magnitude to prevent explosion<br>
              <strong>5. Careful initialization:</strong> He init (ReLU), Xavier init (tanh/sigmoid)<br>
              <strong>6. LSTM/GRU gates:</strong> control gradient flow in recurrent networks
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> ResNet's skip connections (y = F(x) + x)
            were revolutionary because the gradient for the shortcut path is exactly 1 — no vanishing. This enabled
            training networks with 100+ layers. <code>torch.nn.utils.clip_grad_norm_(params, max_norm=1.0)</code> is
            standard for RNNs and Transformers.</div>
        </div>

        <!-- 34 TAYLOR SERIES -->
        <a class="section-anchor" id="taylor-series"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-purple">34</span>
            <h3>Taylor Series & Approximation <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>A Taylor series approximates any smooth function as an infinite polynomial.</strong> The key
            insight: knowing a function and all its derivatives at one point lets you reconstruct the function
            everywhere nearby. Adam uses first and second moment estimates — essentially a 2nd-order Taylor
            approximation of the loss.
          </div>
          <div class="formula-block">
            <div style="font-family:var(--mono);font-size:14px;line-height:2;">
              f(x) = f(a) + f'(a)(x−a) + f''(a)(x−a)²/2! + f'''(a)(x−a)³/3! + ...<br>
              <span style="font-size:12px;color:var(--text3);">1st order ≈ linear · 2nd order ≈ quadratic (Newton's
                method)</span>
            </div>
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div>
              <div class="code-heading numpy-heading">With NumPy — Taylor of eˣ</div>
              <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

<span class="kw">def</span> taylor_exp(x, n_terms=<span class="nm">10</span>):
    <span class="st">"""eˣ ≈ 1 + x + x²/2! + x³/3! +..."""</span>
    result = np.zeros_like(x, dtype=float)
    <span class="kw">for</span> n <span class="kw">in</span> range(n_terms):
        result += x**n / np.math.factorial(n)
    <span class="kw">return</span> result

x = np.array([<span class="nm">0</span>, <span class="nm">1</span>, <span class="nm">2</span>])
<span class="kw">print</span>(<span class="st">"Taylor:"</span>, taylor_exp(x))
<span class="kw">print</span>(<span class="st">"Exact: "</span>, np.exp(x))
<span class="out"># Taylor: [1.    2.718 7.389]</span>
<span class="out"># Exact:  [1.    2.718 7.389]</span></pre>
            </div>
            <div>
              <div class="code-heading python-heading">Pure Python</div>
              <pre style="margin:0;"><span class="kw">def</span> factorial(n):
    r = <span class="nm">1</span>
    <span class="kw">for</span> i <span class="kw">in</span> range(<span class="nm">2</span>, n+<span class="nm">1</span>): r *= i
    <span class="kw">return</span> r

<span class="kw">def</span> taylor_exp(x, n_terms=<span class="nm">15</span>):
    <span class="kw">return</span> sum(x**n / factorial(n)
               <span class="kw">for</span> n <span class="kw">in</span> range(n_terms))

<span class="kw">def</span> taylor_sin(x, n_terms=<span class="nm">10</span>):
    <span class="kw">return</span> sum(
        (-<span class="nm">1</span>)**n * x**(<span class="nm">2</span>*n+<span class="nm">1</span>) / factorial(<span class="nm">2</span>*n+<span class="nm">1</span>)
        <span class="kw">for</span> n <span class="kw">in</span> range(n_terms))

<span class="kw">print</span>(taylor_exp(<span class="nm">1</span>))    <span class="out"># 2.71828...</span>
<span class="kw">print</span>(taylor_sin(<span class="nm">3.14159</span>/<span class="nm">2</span>))
<span class="out"># ≈ 1.0</span></pre>
            </div>
          </div>
          <div class="callout callout-blue">🎯 <strong>ML Application:</strong> <strong>Newton's method</strong> uses a
            2nd-order Taylor expansion to find minima faster than GD. <strong>Adam</strong>'s adaptive learning rates
            are related to diagonal approximations of the Hessian. The <strong>GELU</strong> activation uses a Taylor
            approximation of the Gaussian CDF.</div>
        </div>

        <!-- 35 AUTOMATIC DIFFERENTIATION -->
        <a class="section-anchor" id="autodiff"></a>
        <div class="section">
          <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
            <span class="badge badge-purple">35</span>
            <h3>Automatic Differentiation <span class="difficulty-pill diff-advanced">Advanced</span></h3>
          </div>
          <div class="concept-intro">
            <strong>Automatic differentiation (AD) computes exact derivatives by decomposing functions into elementary
              operations</strong> and applying the chain rule systematically. It's neither symbolic (like Wolfram Alpha)
            nor numerical (like finite differences) — it's exact AND efficient.
          </div>
          <div class="by-hand-note">How PyTorch computes gradients — not magic, just clever chain rule!</div>
          <div class="mermaid">
            graph TD
            subgraph AD ["Automatic Differentiation Modes"]
            FWD["Forward Mode<br />Compute ∂y/∂x alongside y<br />Good for few inputs, many outputs"] --> DUAL["Uses
            dual numbers<br />(value, derivative) pairs"]
            REV["Reverse Mode (Backprop)<br />Compute all ∂y/∂xᵢ in one pass<br />Good for many inputs, few outputs"]
            --> TAPE["Uses computation tape<br />= what PyTorch does"]
            end
          </div>
          <div class="grid-2" style="margin:1rem 0;">
            <div style="background:var(--accent-bg);border-radius:var(--radius);padding:1rem;">
              <div style="font-size:13px;font-weight:600;color:var(--accent);margin-bottom:6px;">Forward Mode</div>
              <p style="font-size:12px;color:var(--text2);margin:0;">Propagates derivatives alongside values. Cost =
                O(n) passes for n inputs. Good when inputs << outputs (rare in ML).</p>
            </div>
            <div style="background:var(--green-bg);border-radius:var(--radius);padding:1rem;">
              <div style="font-size:13px;font-weight:600;color:var(--green);margin-bottom:6px;">Reverse Mode (Backprop)
              </div>
              <p style="font-size:12px;color:var(--text2);margin:0;">One forward pass + one backward pass computes ALL
                gradients. Cost = O(1) backward passes regardless of # inputs. <strong>This is why backprop is used in
                  ML.</strong></p>
            </div>
          </div>
          <pre><span class="cm"># Tiny autodiff engine (inspired by Karpathy's micrograd)</span>
<span class="kw">class</span> Value:
    <span class="kw">def</span> __init__(self, data, children=(), op=<span class="st">''</span>):
        self.data = data
        self.grad = <span class="nm">0.0</span>
        self._backward = <span class="kw">lambda</span>: <span class="kw">None</span>
        self._prev = set(children)

    <span class="kw">def</span> __add__(self, other):
        out = Value(self.data + other.data, (self, other), <span class="st">'+'</span>)
        <span class="kw">def</span> _backward():
            self.grad += out.grad   <span class="cm"># ∂(a+b)/∂a = 1</span>
            other.grad += out.grad  <span class="cm"># ∂(a+b)/∂b = 1</span>
        out._backward = _backward
        <span class="kw">return</span> out

    <span class="kw">def</span> __mul__(self, other):
        out = Value(self.data * other.data, (self, other), <span class="st">'*'</span>)
        <span class="kw">def</span> _backward():
            self.grad += other.data * out.grad  <span class="cm"># ∂(ab)/∂a = b</span>
            other.grad += self.data * out.grad  <span class="cm"># ∂(ab)/∂b = a</span>
        out._backward = _backward
        <span class="kw">return</span> out

    <span class="kw">def</span> backward(self):
        <span class="cm"># Topological sort + reverse traverse</span>
        topo = []; visited = set()
        <span class="kw">def</span> build(v):
            <span class="kw">if</span> v <span class="kw">not in</span> visited:
                visited.add(v)
                <span class="kw">for</span> c <span class="kw">in</span> v._prev: build(c)
                topo.append(v)
        build(self)
        self.grad = <span class="nm">1.0</span>
        <span class="kw">for</span> v <span class="kw">in</span> reversed(topo): v._backward()

<span class="cm"># Example: f = (a * b) + b</span>
a = Value(<span class="nm">2.0</span>); b = Value(<span class="nm">3.0</span>)
c = a * b      <span class="cm"># c = 6</span>
d = c + b      <span class="cm"># d = 9</span>
d.backward()
<span class="kw">print</span>(f<span class="st">"∂d/∂a = {a.grad}"</span>)  <span class="out"># 3.0 (= b)</span>
<span class="kw">print</span>(f<span class="st">"∂d/∂b = {b.grad}"</span>)  <span class="out"># 3.0 (= a + 1)</span></pre>
