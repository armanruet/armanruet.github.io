    <!-- 1 WHAT IS ZIP -->
    <a class="section-anchor" id="1-what-is-zip"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-green">01</span><h2 style="margin:0;font-size:24px;">What is zip()?</h2>
      </div>
      <p><code>zip()</code> is a Python built-in function that <strong>aggregates elements from multiple iterables</strong> into tuples, pairing them index by index.</p>
      <p>Think of it like a physical zipper on a jacket — it interleaves two sides (iterables) into one unified structure. Rather than writing clunky <code>for i in range(len(list))</code> loops, <code>zip()</code> allows you to iterate over multiple lists in parallel, elegantly and Pythonically.</p>
      <pre>
Iterable A:  [A1,  A2,  A3,  A4]
Iterable B:  [B1,  B2,  B3,  B4]
              &darr;    &darr;    &darr;    &darr;
zip result:  (A1,B1) (A2,B2) (A3,B3) (A4,B4)</pre>
      
      <h3>Why is it powerful?</h3>
      <ul>
        <li>It is <strong>lazy</strong> — produces values on demand, making it incredibly memory efficient when processing millions of records.</li>
        <li>Works seamlessly on <strong>any iterable</strong> — lists, tuples, strings, generators, dictionaries, and ranges.</li>
        <li>Combined with the unpacking operator <code>*</code>, it functions as a highly optimized <strong>matrix transposer</strong>.</li>
        <li>Chains cleanly with functional programming paradigms like <code>map</code>, <code>filter</code>, and list comprehensions.</li>
      </ul>
    </div>

    <!-- 2 SYNTAX -->
    <a class="section-anchor" id="2-syntax--parameters"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-blue">02</span><h2 style="margin:0;font-size:24px;">Syntax &amp; Parameters</h2>
      </div>
      <p>The signature of the function is deceptively simple:</p>
      <pre><span class="kw">zip</span>(*iterables, strict=<span class="kw">False</span>)</pre>
      
      <table>
        <thead>
          <tr>
            <th>Parameter</th>
            <th>Type</th>
            <th>Description</th>
          </tr>
        </thead>
        <tbody>
          <tr>
            <td><code>*iterables</code></td>
            <td>Any iterable</td>
            <td>One or more iterables to zip together. If passing a nested list (matrix), use <code>*matrix</code> to pass inner lists as separate arguments.</td>
          </tr>
          <tr>
            <td><code>strict</code></td>
            <td><code>bool</code></td>
            <td>Introduced in Python 3.10. If <code>True</code>, Python enforces equal lengths and raises a <code>ValueError</code> if any iterable is exhausted before the others.</td>
          </tr>
        </tbody>
      </table>

      <p><strong>Return value:</strong> <code>zip()</code> returns a <code>zip</code> object, which is an iterator of tuples. To view the contents, you must cast it to a list, tuple, or iterate through it.</p>
      <pre>
<span class="cm"># Basic example</span>
names  = [<span class="st">"Alice"</span>, <span class="st">"Bob"</span>, <span class="st">"Carol"</span>]
scores = [<span class="nm">95</span>,      <span class="nm">87</span>,    <span class="nm">92</span>    ]

zipped = <span class="kw">zip</span>(names, scores)
<span class="kw">print</span>(<span class="kw">type</span>(zipped))         <span class="out"># &lt;class 'zip'&gt;</span>
<span class="kw">print</span>(<span class="kw">list</span>(zipped))         <span class="out"># [('Alice', 95), ('Bob', 87), ('Carol', 92)]</span></pre>
    </div>

    <!-- 3 INTERNALLY -->
    <a class="section-anchor" id="3-how-zip-works-internally"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-amber">03</span><h2 style="margin:0;font-size:24px;">How zip() Works Internally</h2>
      </div>
      <p><code>zip()</code> is implemented as a <strong>lazy iterator</strong> in C. It does NOT build the entire paired list in memory. Instead, it maintains internal pointers to each provided iterable and advances them simultaneously by exactly one step every time <code>next()</code> is called.</p>
      
      <pre>
a = [<span class="nm">10</span>, <span class="nm">20</span>, <span class="nm">30</span>]
b = [<span class="st">'x'</span>, <span class="st">'y'</span>, <span class="st">'z'</span>]
z = <span class="kw">zip</span>(a, b)

<span class="cm"># Each call to next() fetches one element from EACH iterable:</span>
<span class="kw">print</span>(<span class="kw">next</span>(z))   <span class="out"># (10, 'x')</span>
<span class="kw">print</span>(<span class="kw">next</span>(z))   <span class="out"># (20, 'y')</span>
<span class="kw">print</span>(<span class="kw">next</span>(z))   <span class="out"># (30, 'z')</span>
<span class="kw">print</span>(<span class="kw">next</span>(z))   <span class="out"># StopIteration — all iterables exhausted</span></pre>

      <h3>The Unpacking Operator (*) with zip</h3>
      <p>The combination of <code>*</code> and <code>zip</code> is arguably the most important pattern to memorize for matrix manipulation. By prefixing a list of lists with <code>*</code>, you "explode" the outer list into individual arguments. <code>zip()</code> then groups the nth element of every argument together, effectively <strong>transposing</strong> the matrix.</p>
      <pre>
matrix = [(<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>),
          (<span class="nm">4</span>, <span class="nm">5</span>, <span class="nm">6</span>)]

<span class="cm"># *matrix unpacks to: zip((1,2,3), (4,5,6))</span>
<span class="cm"># zip pairs by position &rarr; transpose!</span>
transposed = <span class="kw">list</span>(<span class="kw">zip</span>(*matrix))
<span class="out"># [(1, 4), (2, 5), (3, 6)]</span></pre>
    </div>

    <!-- 4 EDGE CASES -->
    <a class="section-anchor" id="4-core-behaviors--edge-cases"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-purple">04</span><h2 style="margin:0;font-size:24px;">Core Behaviors &amp; Edge Cases</h2>
      </div>
      <p>Understanding how <code>zip()</code> behaves at the boundaries is critical to preventing silent data loss in your applications.</p>
      
      <div class="grid-2">
        <div>
          <h3>4.1 Unequal Lengths</h3>
          <p>By default, <code>zip()</code> stops iterating as soon as the <strong>shortest</strong> iterable is exhausted. Remaining elements in longer iterables are silently ignored.</p>
          <pre>
a = [<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>, <span class="nm">4</span>, <span class="nm">5</span>]
b = [<span class="st">'a'</span>, <span class="st">'b'</span>, <span class="st">'c'</span>]

<span class="kw">print</span>(<span class="kw">list</span>(<span class="kw">zip</span>(a, b)))
<span class="out"># [(1, 'a'), (2, 'b'), (3, 'c')]</span></pre>
        </div>
        <div>
          <h3>4.2 Strict Mode (Python 3.10+)</h3>
          <p>If silent dropping is dangerous (e.g., in ML pipelines where features and labels must match), use <code>strict=True</code>.</p>
          <pre>
<span class="kw">list</span>(<span class="kw">zip</span>([<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>], [<span class="st">'a'</span>, <span class="st">'b'</span>], strict=<span class="kw">True</span>))
<span class="out"># ValueError: zip() has arguments with different lengths</span></pre>
        </div>
        <div>
          <h3>4.3 zip_longest</h3>
          <p>If you want to pad the shorter iterables instead of truncating the longer ones, use <code>itertools.zip_longest</code>.</p>
          <pre>
<span class="kw">from</span> itertools <span class="kw">import</span> zip_longest
<span class="kw">list</span>(zip_longest([<span class="nm">1</span>, <span class="nm">2</span>], [<span class="st">'a'</span>], fillvalue=<span class="nm">0</span>))
<span class="out"># [(1, 'a'), (2, 0)]</span></pre>
        </div>
        <div>
          <h3>4.4 Iterator Exhaustion</h3>
          <p>Because <code>zip()</code> returns an iterator, it can only be consumed <strong>once</strong>. Iterating over it a second time yields an empty list.</p>
          <pre>
z = <span class="kw">zip</span>([<span class="nm">1</span>], [<span class="nm">2</span>])
<span class="kw">list</span>(z)  <span class="out"># [(1, 2)]</span>
<span class="kw">list</span>(z)  <span class="out"># []  &larr; Exhausted!</span></pre>
        </div>
      </div>
    </div>

    <!-- 5 1D SEQUENCES -->
    <a class="section-anchor" id="5-zip-with-1d-sequences"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-teal">05</span><h2 style="margin:0;font-size:24px;">zip() with 1D Sequences</h2>
      </div>
      <p>Before moving to matrices, let's explore how <code>zip()</code> simplifies operations on flat lists (vectors).</p>
      
      <div class="grid-2">
        <div>
          <h3>5.1 Parallel Iteration</h3>
          <p>The most common use case: looping through multiple lists cleanly without using a fragile index <code>i</code>.</p>
          <pre>names  = [<span class="st">"Alice"</span>, <span class="st">"Bob"</span>, <span class="st">"Carol"</span>]
ages   = [<span class="nm">30</span>, <span class="nm">25</span>, <span class="nm">35</span>]

<span class="kw">for</span> n, a <span class="kw">in</span> <span class="kw">zip</span>(names, ages):
    <span class="kw">print</span>(<span class="kw">f</span><span class="st">"{n} is {a}"</span>)</pre>
        </div>
        <div>
          <h3>5.2 Building Dictionaries</h3>
          <p>Quickly map a list of keys to a list of values to dynamically generate a dictionary.</p>
          <pre>keys   = [<span class="st">"name"</span>, <span class="st">"age"</span>]
values = [<span class="st">"Alice"</span>, <span class="nm">30</span>]

d = <span class="kw">dict</span>(<span class="kw">zip</span>(keys, values))
<span class="out"># {'name': 'Alice', 'age': 30}</span></pre>
        </div>
        <div>
          <h3>5.3 Pairwise Differences</h3>
          <p>By zipping a list with a sliced version of itself, you can easily calculate differences between adjacent elements (useful for time-series data).</p>
          <pre>data = [<span class="nm">10</span>, <span class="nm">15</span>, <span class="nm">13</span>, <span class="nm">20</span>]
diffs = [b - a <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(data, data[<span class="nm">1</span>:])]
<span class="out"># [5, -2, 7]</span></pre>
        </div>
        <div>
          <h3>5.4 Unzipping</h3>
          <p>You can reverse the process using the unpacking operator to split a list of tuples back into separate, independent tuples.</p>
          <pre>pairs = [(<span class="nm">1</span>, <span class="st">'a'</span>), (<span class="nm">2</span>, <span class="st">'b'</span>), (<span class="nm">3</span>, <span class="st">'c'</span>)]

numbers, letters = <span class="kw">zip</span>(*pairs)
<span class="out"># numbers = (1, 2, 3)</span>
<span class="out"># letters = ('a', 'b', 'c')</span></pre>
        </div>
      </div>
    </div>

    <!-- 6 2D MATRICES -->
    <a class="section-anchor" id="6-zip-with-2d-matrices"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-red">06</span><h2 style="margin:0;font-size:24px;">zip() with 2D Matrices</h2>
      </div>
      <p>A 2D matrix in pure Python is represented as a list of lists. While libraries like NumPy handle these efficiently in C, understanding how to manipulate matrices with <code>zip()</code> is a rite of passage for Python developers and crucial for environments where external dependencies aren't allowed.</p>
      
      <div class="grid-2">
        <div>
          <h3>6.1 Deriving Shape</h3>
          <p>Rows are the length of the outer list. Columns are the length of the transposed list (since transposition groups columns into tuples).</p>
          <pre><span class="kw">def</span> shape_2d(M):
    rows = <span class="kw">len</span>(M)
    cols = <span class="kw">len</span>(<span class="kw">list</span>(<span class="kw">zip</span>(*M)))
    <span class="kw">return</span> (rows, cols)</pre>
        </div>
        <div>
          <h3>6.2 Matrix Transposition</h3>
          <p>The most iconic use of <code>zip()</code>. It swaps rows and columns, turning an M&times;N matrix into an N&times;M matrix in a single line.</p>
          <pre>M = [[<span class="nm">1</span>, <span class="nm">2</span>], [<span class="nm">3</span>, <span class="nm">4</span>]]
MT = [<span class="kw">list</span>(row) <span class="kw">for</span> row <span class="kw">in</span> <span class="kw">zip</span>(*M)]
<span class="out"># [[1, 3], [2, 4]]</span></pre>
        </div>
      </div>
      
      <h3>6.3 Element-wise Matrix Operations</h3>
      <p>To add or multiply matrices element-by-element (the Hadamard product), we use a nested list comprehension. The outer <code>zip(A, B)</code> pairs up the rows, and the inner <code>zip(rowA, rowB)</code> pairs up the individual elements within those rows.</p>
      <pre>
A = [[<span class="nm">1</span>, <span class="nm">2</span>], [<span class="nm">3</span>, <span class="nm">4</span>]]
B = [[<span class="nm">5</span>, <span class="nm">6</span>], [<span class="nm">7</span>, <span class="nm">8</span>]]

<span class="cm"># Element-wise Addition</span>
C = [[a + b <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(rowA, rowB)]
     <span class="kw">for</span> rowA, rowB <span class="kw">in</span> <span class="kw">zip</span>(A, B)]

<span class="cm"># Element-wise Multiply (Hadamard)</span>
H = [[a * b <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(rowA, rowB)]
     <span class="kw">for</span> rowA, rowB <span class="kw">in</span> <span class="kw">zip</span>(A, B)]</pre>
    </div>

    <!-- 7 3D MATRICES -->
    <a class="section-anchor" id="7-zip-with-3d-matrices"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-purple">07</span><h2 style="margin:0;font-size:24px;">zip() with 3D Matrices</h2>
      </div>
      <p>In Machine Learning, 3D matrices (tensors) are incredibly common, representing batches of 2D data (like images or sequences). A 3D matrix in Python is a list containing lists of lists.</p>
      <pre>
M3 = [
    <span class="cm"># Layer 0 (Batch 1)</span>
    [[<span class="nm">1</span>, <span class="nm">2</span>],
     [<span class="nm">3</span>, <span class="nm">4</span>]],
    <span class="cm"># Layer 1 (Batch 2)</span>
    [[<span class="nm">5</span>, <span class="nm">6</span>],
     [<span class="nm">7</span>, <span class="nm">8</span>]]
]</pre>
      
      <div class="grid-2">
        <div>
          <h3>7.1 Recursive Universal Shape</h3>
          <p>We can use <code>zip()</code> iteratively to peel back dimensions, or recursion to dive deep into the list structure to find the shape of a tensor of any dimension.</p>
          <pre>
<span class="kw">def</span> get_shape(matrix):
    <span class="kw">if</span> <span class="kw">not</span> <span class="kw">isinstance</span>(matrix, <span class="kw">list</span>) <span class="kw">or</span> <span class="kw">not</span> matrix:
        <span class="kw">return</span> ()
    <span class="kw">return</span> (<span class="kw">len</span>(matrix),) + get_shape(matrix[<span class="nm">0</span>])

<span class="kw">print</span>(get_shape(M3))   <span class="out"># (2, 2, 2)</span></pre>
        </div>
        <div>
          <h3>7.2 Element-wise 3D Operations</h3>
          <p>By nesting three levels deep, we can perform operations like tensor addition entirely in pure Python.</p>
          <pre>
<span class="cm"># Add two 3D tensors T1 and T2</span>
T_add = [
    [[a + b <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(rA, rB)]
     <span class="kw">for</span> rA, rB <span class="kw">in</span> <span class="kw">zip</span>(layerA, layerB)]
    <span class="kw">for</span> layerA, layerB <span class="kw">in</span> <span class="kw">zip</span>(T1, T2)
]</pre>
        </div>
      </div>
    </div>

    <!-- 8 LINEAR ALGEBRA -->
    <a class="section-anchor" id="8-linear-algebra-basics"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-amber">08</span><h2 style="margin:0;font-size:24px;">Linear Algebra: Core Operations</h2>
      </div>
      <p><code>zip()</code> is the fundamental building block for translating mathematical formulas into clean Python code.</p>
      
      <h3>8.1 The Dot Product</h3>
      <p>The dot product multiplies corresponding elements of two equal-length vectors and sums the result into a single scalar. It evaluates the directional alignment of two vectors.</p>
      <div class="formula">a · b = &Sigma; (a<sub>i</sub> &times; b<sub>i</sub>)</div>
      <pre>
<span class="kw">def</span> dot_product(v1, v2):
    <span class="kw">return</span> <span class="kw">sum</span>(a * b <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(v1, v2))

<span class="kw">print</span>(dot_product([<span class="nm">1</span>, <span class="nm">3</span>, -<span class="nm">5</span>], [<span class="nm">4</span>, -<span class="nm">2</span>, -<span class="nm">1</span>]))  <span class="out"># 3</span></pre>

      <h3>8.2 Matrix Multiplication (matmul)</h3>
      <p>Matrix multiplication is a series of dot products. To multiply matrix A by matrix B, we dot the <em>rows</em> of A against the <em>columns</em> of B. Using <code>zip(*B)</code> we can transpose B ahead of time so its columns are easily iterable as rows.</p>
      <pre>
<span class="kw">def</span> matmul(A, B):
    BT = <span class="kw">list</span>(<span class="kw">zip</span>(*B))   <span class="cm"># Transpose B so columns become rows</span>
    <span class="kw">return</span> [
        [<span class="kw">sum</span>(a * b <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(row_A, col_B)) <span class="kw">for</span> col_B <span class="kw">in</span> BT]
        <span class="kw">for</span> row_A <span class="kw">in</span> A
    ]

A = [[<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>], [<span class="nm">4</span>, <span class="nm">5</span>, <span class="nm">6</span>]]   <span class="cm"># 2×3</span>
B = [[<span class="nm">7</span>, <span class="nm">8</span>], [<span class="nm">9</span>, <span class="nm">10</span>], [<span class="nm">11</span>, <span class="nm">12</span>]]  <span class="cm"># 3×2</span>

C = matmul(A, B)  <span class="out"># [[58, 64], [139, 154]]</span></pre>

      <h3>8.3 Matrix-Vector Multiplication</h3>
      <p>A specialized case of matmul where a matrix transforms a single vector. We dot each row of the matrix with the vector.</p>
      <pre>
<span class="kw">def</span> mat_vec_mul(matrix, vector):
    <span class="kw">return</span> [<span class="kw">sum</span>(a * b <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(row, vector)) <span class="kw">for</span> row <span class="kw">in</span> matrix]

<span class="kw">print</span>(mat_vec_mul([[<span class="nm">1</span>, <span class="nm">2</span>], [<span class="nm">3</span>, <span class="nm">4</span>]], [<span class="nm">5</span>, <span class="nm">6</span>]))  <span class="out"># [17, 39]</span></pre>
    </div>

    <!-- 9 DISTANCE METRICS -->
    <a class="section-anchor" id="9-distance-metrics"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-blue">09</span><h2 style="margin:0;font-size:24px;">Distance Metrics &amp; Similarity</h2>
      </div>
      <p>In Machine Learning (like K-Nearest Neighbors or clustering), measuring the distance or similarity between feature vectors is essential. <code>zip()</code> makes implementing these mathematical definitions trivial.</p>
      
      <div class="grid-2">
        <div>
          <h3>9.1 Euclidean Distance (L2)</h3>
          <p>The straight-line distance between two points in multidimensional space.</p>
          <pre>
<span class="kw">import</span> math
<span class="kw">def</span> euclidean_distance(p, q):
    <span class="kw">return</span> math.sqrt(<span class="kw">sum</span>((a - b)**<span class="nm">2</span> <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(p, q)))</pre>
        </div>
        <div>
          <h3>9.2 Manhattan Distance (L1)</h3>
          <p>The sum of absolute differences across all dimensions (grid-like distance).</p>
          <pre>
<span class="kw">def</span> manhattan_distance(p, q):
    <span class="kw">return</span> <span class="kw">sum</span>(<span class="kw">abs</span>(a - b) <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(p, q))</pre>
        </div>
      </div>
      
      <h3>9.3 Cosine Similarity</h3>
      <p>Cosine similarity measures the angle between two vectors, completely ignoring their magnitude. It's heavily used in NLP to compare document embeddings or word vectors. It is defined as the dot product divided by the product of their magnitudes.</p>
      <pre>
<span class="kw">def</span> cosine_similarity(v1, v2):
    dot_prod = <span class="kw">sum</span>(a * b <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(v1, v2))
    mag1 = math.sqrt(<span class="kw">sum</span>(a**<span class="nm">2</span> <span class="kw">for</span> a <span class="kw">in</span> v1))
    mag2 = math.sqrt(<span class="kw">sum</span>(b**<span class="nm">2</span> <span class="kw">for</span> b <span class="kw">in</span> v2))
    <span class="kw">return</span> dot_prod / (mag1 * mag2) <span class="kw">if</span> mag1 * mag2 <span class="kw">else</span> <span class="nm">0</span></pre>

      <h3>9.4 Vector Projection (Gram-Schmidt Foundation)</h3>
      <p>Projecting vector <em>v</em> onto vector <em>u</em> isolates the portion of <em>v</em> that points in the same direction as <em>u</em>. This is the cornerstone of orthogonalization and Principal Component Analysis (PCA).</p>
      <pre>
<span class="kw">def</span> project(v, u):
    dot_vu = <span class="kw">sum</span>(a * b <span class="kw">for</span> a, b <span class="kw">in</span> <span class="kw">zip</span>(v, u))
    dot_uu = <span class="kw">sum</span>(a * a <span class="kw">for</span> a <span class="kw">in</span> u)
    scalar = dot_vu / dot_uu
    <span class="kw">return</span> [scalar * x <span class="kw">for</span> x <span class="kw">in</span> u]</pre>
    </div>

          <h3>10.1 Covariance</h3>
          <p>Measures how two variables change together. <code>zip()</code> pairs the corresponding observations.</p>
          <pre>
<span class="kw">def</span> mean(v): <span class="kw">return</span> <span class="kw">sum</span>(v) / <span class="kw">len</span>(v)

<span class="kw">def</span> covariance(x, y):
    mx, my = mean(x), mean(y)
    <span class="kw">return</span> <span class="kw">sum</span>((xi - mx) * (yi - my) 
               <span class="kw">for</span> xi, yi <span class="kw">in</span> <span class="kw">zip</span>(x, y)) / <span class="kw">len</span>(x)</pre>
        </div>
        <div>
          <h3>10.2 Pearson Correlation</h3>
          <p>Normalizes covariance to a value between -1 and 1.</p>
          <pre>
<span class="kw">def</span> pearson(x, y):
    mx, my = mean(x), mean(y)
    num = <span class="kw">sum</span>((xi-mx)*(yi-my) <span class="kw">for</span> xi,yi <span class="kw">in</span> <span class="kw">zip</span>(x,y))
    den = math.sqrt(<span class="kw">sum</span>((xi-mx)**<span class="nm">2</span> <span class="kw">for</span> xi <span class="kw">in</span> x) * 
                    <span class="kw">sum</span>((yi-my)**<span class="nm">2</span> <span class="kw">for</span> yi <span class="kw">in</span> y))
    <span class="kw">return</span> num / den <span class="kw">if</span> den <span class="kw">else</span> <span class="nm">0</span></pre>

    <span class="kw">return</span> [<span class="kw">list</span>(row) <span class="kw">for</span> row <span class="kw">in</span> <span class="kw">zip</span>(*matrix[::-<span class="nm">1</span>])]</pre>

    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:1rem;">
        <span class="badge badge-red">12</span><h2 style="margin:0;font-size:24px;">Performance &amp; Best Uses</h2>
      </div>
      
      <div class="callout callout-amber">
        <p><strong>The Golden Rule: Embrace Laziness</strong></p>
        <p>Never wrap <code>zip()</code> in <code>list()</code> unless you absolutely need the entire constructed array in memory. If you are zipping a 5-gigabyte text file with a label generator, using a <code>for x, y in zip(...)</code> loop processes one line at a time and consumes almost zero RAM. Casting it to a list will crash your machine with an Out Of Memory error.</p>
      </div>

      <ul>
        <li><strong>Avoid redundant zips:</strong> Transposing a matrix inside a loop regenerates the zip object every time. Store the transposed result if you need to iterate over it multiple times.</li>
        <li><strong>Use strict=True for ML Pipelines:</strong> When zipping <code>features</code> and <code>labels</code> for training data, ensuring array lengths perfectly match prevents silent bugs where trailing data is chopped off and ignored by the model.</li>
      </ul>
