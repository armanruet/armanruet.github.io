        <span class="badge badge-green">01</span><h3>What is a Vector?</h3>
      </div>
      <p>A <strong>vector</strong> is an ordered list of numbers. Think of it as a point in space — or an arrow with both magnitude and direction. In machine learning it holds features: one number per feature.</p>
      <div class="matrix-scene">
        <div class="mat-block"><div class="mat-label">Row vector (1 × 3)</div>
          <div class="matrix-wrap"><span class="bracket">[</span><div class="m-inner"><div class="m-row"><span class="cell c-b">1</span><span class="cell c-b">4</span><span class="cell c-b">9</span></div></div><span class="bracket">]</span></div>
        </div>
        <div class="mat-block"><div class="mat-label">Column vector (3 × 1)</div>
          <div class="matrix-wrap"><span class="bracket">[</span><div class="m-inner"><div class="m-row"><span class="cell c-g">1</span></div><div class="m-row"><span class="cell c-g">4</span></div><div class="m-row"><span class="cell c-g">9</span></div></div><span class="bracket">]</span></div>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;"><span class="kw">import</span> numpy <span class="kw">as</span> np

v = np.array([<span class="nm">1</span>, <span class="nm">4</span>, <span class="nm">9</span>])  <span class="cm"># 1-D array</span>
<span class="kw">print</span>(v.shape)            <span class="out"># (3,)</span>
<span class="kw">print</span>(v.ndim)             <span class="out"># 1</span>

<span class="cm"># Column vector (3×1)</span>
col = np.array([[<span class="nm">1</span>],[<span class="nm">4</span>],[<span class="nm">9</span>]])
<span class="kw">print</span>(col.shape)          <span class="out"># (3, 1)</span></pre>
        </div>

          <pre style="margin:0;"><span class="cm"># Row vector — a plain list</span>
v = [<span class="nm">1</span>, <span class="nm">4</span>, <span class="nm">9</span>]

      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-blue">02</span><h3>What is a Matrix?</h3>
      </div>
      <p>A <strong>matrix</strong> is a 2-D grid of numbers arranged in <strong>rows</strong> and <strong>columns</strong>. A dataset with <em>m</em> samples and <em>n</em> features is naturally an m × n matrix.</p>
      <div class="matrix-scene">
        <div class="mat-block"><div class="mat-label">Matrix A — shape (3 × 4)</div>
          <div class="matrix-wrap"><span class="bracket">[</span><div class="m-inner">
            <div class="m-row"><span class="cell c-b">1</span><span class="cell c-b">2</span><span class="cell c-b">3</span><span class="cell c-b">4</span></div>
            <div class="m-row"><span class="cell c-g">5</span><span class="cell c-g">6</span><span class="cell c-g">7</span><span class="cell c-g">8</span></div>
            <div class="m-row"><span class="cell c-a">9</span><span class="cell c-a">10</span><span class="cell c-a">11</span><span class="cell c-a">12</span></div>
          </div><span class="bracket">]</span></div>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>,<span class="nm">4</span>],
              [<span class="nm">5</span>,<span class="nm">6</span>,<span class="nm">7</span>,<span class="nm">8</span>],
              [<span class="nm">9</span>,<span class="nm">10</span>,<span class="nm">11</span>,<span class="nm">12</span>]])
<span class="kw">print</span>(A.shape)  <span class="out"># (3, 4)</span>
<span class="kw">print</span>(A.ndim)   <span class="out"># 2</span>

<span class="cm"># Access element [row][col]</span>
A[<span class="nm">1</span>, <span class="nm">2</span>]         <span class="out"># 7</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;">A = [[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>,<span class="nm">4</span>],
     [<span class="nm">5</span>,<span class="nm">6</span>,<span class="nm">7</span>,<span class="nm">8</span>],
     [<span class="nm">9</span>,<span class="nm">10</span>,<span class="nm">11</span>,<span class="nm">12</span>]]

<span class="cm"># Shape via len()</span>
rows = len(A)       <span class="out"># 3</span>
cols = len(A[<span class="nm">0</span>])   <span class="out"># 4</span>

<span class="cm"># Access element [row][col]</span>

    <!-- 03 TRANSPOSE -->
    <a class="section-anchor" id="transpose"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-red">03</span><h3>What is a Transpose?</h3>
      </div>
      <p>The <strong>transpose</strong> of a matrix flips it over its main diagonal — rows become columns and columns become rows. An (m × n) matrix becomes (n × m).</p>
      <div class="matrix-scene">
        <div class="mat-block"><div class="mat-label">A — (2 × 3)</div>
          <div class="matrix-wrap"><span class="bracket">[</span><div class="m-inner">
            <div class="m-row"><span class="cell c-b">1</span><span class="cell c-b">2</span><span class="cell c-b">3</span></div>
            <div class="m-row"><span class="cell c-g">4</span><span class="cell c-g">5</span><span class="cell c-g">6</span></div>
          </div><span class="bracket">]</span></div>
        </div>
        <span class="arrow-label">→</span>
        <div class="mat-block"><div class="mat-label">Aᵀ — (3 × 2)</div>
          <div class="matrix-wrap"><span class="bracket">[</span><div class="m-inner">
            <div class="m-row"><span class="cell c-b">1</span><span class="cell c-g">4</span></div>
            <div class="m-row"><span class="cell c-b">2</span><span class="cell c-g">5</span></div>
            <div class="m-row"><span class="cell c-b">3</span><span class="cell c-g">6</span></div>
          </div><span class="bracket">]</span></div>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]])

<span class="cm"># Method 1 — attribute</span>
AT = A.T

<span class="cm"># Method 2 — function</span>
AT = np.transpose(A)

<span class="kw">print</span>(AT.shape)  <span class="out"># (3, 2)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;">A = [[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]]

<span class="cm"># Method 1 — explicit loops</span>
<span class="kw">def</span> transpose(A):
    rows, cols = len(A), len(A[<span class="nm">0</span>])
    T = []
    <span class="kw">for</span> j <span class="kw">in</span> range(cols):
        row = []
        <span class="kw">for</span> i <span class="kw">in</span> range(rows):
            row.append(A[i][j])
        T.append(row)
    <span class="kw">return</span> T

<span class="cm"># Method 2 — list comprehension</span>
<span class="kw">def</span> transpose(A):
    <span class="kw">return</span> [[A[i][j] <span class="kw">for</span> i <span class="kw">in</span> range(len(A))]
            <span class="kw">for</span> j <span class="kw">in</span> range(len(A[<span class="nm">0</span>]))]

<span class="cm"># Method 3 — zip trick (Pythonic)</span>
<span class="kw">def</span> transpose(A):
    <span class="kw">return</span> [list(row) <span class="kw">for</span> row <span class="kw">in</span> zip(*A)]

transpose(A)  <span class="out"># [[1,4],[2,5],[3,6]]</span></pre>
        </div>
      </div>
      <div class="callout callout-blue">Transpose is critical for aligning matrix dimensions before multiplication — a very common operation in neural networks. <code>zip(*A)</code> unpacks all rows as arguments to zip, which groups by column position — elegant and Pythonic.</div>
      <h3 style="margin:1.5rem 0 0.5rem;font-family:var(--sans);font-size:15px;font-weight:600;">Transposing 3D Tensors</h3>
      <p>In 3D (e.g., shape <code>(batch, rows, cols)</code>), you rarely reverse all dimensions. Instead, you typically transpose the rows and columns of <em>each matrix in the batch</em> independently, leaving the batch dimension intact.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">T = np.random.rand(<span class="nm">10</span>, <span class="nm">4</span>, <span class="nm">5</span>)  <span class="cm"># 10 batches of 4x5</span>

<span class="cm"># Swap axes 1 (rows) and 2 (cols), leave axis 0 (batch)</span>
T_trans = np.transpose(T, axes=(<span class="nm">0</span>, <span class="nm">2</span>, <span class="nm">1</span>))
<span class="kw">print</span>(T_trans.shape)  <span class="out"># (10, 5, 4)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="cm"># Transpose each matrix in the batch</span>
<span class="kw">def</span> transpose_3d(T):
    <span class="kw">return</span> [transpose(batch) <span class="kw">for</span> batch <span class="kw">in</span> T]
    <span class="cm"># uses the 2D transpose function defined above</span></pre>
        </div>
      </div>
    </div>

    <!-- 04 SHAPE & AXES -->
    <a class="section-anchor" id="shape-axes"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-purple">04</span><h3>Shape of a Matrix &amp; What is an Axis?</h3>
      </div>
      <p>The <strong>shape</strong> is a tuple describing how many elements exist along each <strong>axis</strong> (dimension). Axis 0 runs <em>down</em> the rows; axis 1 runs <em>across</em> the columns.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;text-align:center;">
          <div style="font-size:12px;color:var(--accent);font-weight:500;margin-bottom:8px;">axis 0 ↓ (rows)</div>
          <div style="display:flex;flex-direction:column;gap:4px;align-items:flex-start;display:inline-flex;">
            <div class="m-row"><span class="cell c-b">1</span><span class="cell c-d">2</span><span class="cell c-d">3</span><span class="cell c-d">4</span></div>
            <div class="m-row"><span class="cell c-b">5</span><span class="cell c-d">6</span><span class="cell c-d">7</span><span class="cell c-d">8</span></div>
            <div class="m-row"><span class="cell c-b">9</span><span class="cell c-d">10</span><span class="cell c-d">11</span><span class="cell c-d">12</span></div>
          </div>
          <div style="font-size:11px;color:var(--text3);margin-top:6px;">col 0 highlighted</div>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;text-align:center;">
          <div style="font-size:12px;color:var(--green);font-weight:500;margin-bottom:8px;">axis 1 → (cols)</div>
          <div style="display:inline-flex;flex-direction:column;gap:4px;">
            <div class="m-row"><span class="cell c-g">1</span><span class="cell c-g">2</span><span class="cell c-g">3</span><span class="cell c-g">4</span></div>
            <div class="m-row"><span class="cell c-d">5</span><span class="cell c-d">6</span><span class="cell c-d">7</span><span class="cell c-d">8</span></div>
            <div class="m-row"><span class="cell c-d">9</span><span class="cell c-d">10</span><span class="cell c-d">11</span><span class="cell c-d">12</span></div>
          </div>
          <div style="font-size:11px;color:var(--text3);margin-top:6px;">row 0 highlighted</div>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">A.shape            <span class="out"># (3, 4)</span>
np.sum(A, axis=<span class="nm">0</span>)  <span class="out"># sum down rows   → (4,)</span>
np.sum(A, axis=<span class="nm">1</span>)  <span class="out"># sum across cols → (3,)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="cm"># Shape</span>
<span class="kw">def</span> shape(A):
    rows = len(A)
    cols = len(A[<span class="nm">0</span>])
    <span class="kw">return</span> (rows, cols)

<span class="cm"># Sum along axis 0 (down rows)</span>
<span class="kw">def</span> sum_axis0(A):
    cols = len(A[<span class="nm">0</span>])
    <span class="kw">return</span> [sum(A[i][j] <span class="kw">for</span> i <span class="kw">in</span>
            range(len(A))) <span class="kw">for</span> j <span class="kw">in</span> range(cols)]

<span class="cm"># Sum along axis 1 (across cols)</span>
<span class="kw">def</span> sum_axis1(A):
    <span class="kw">return</span> [sum(row) <span class="kw">for</span> row <span class="kw">in</span> A]</pre>
        </div>
      </div>
      <h3 style="margin:1.5rem 0 0.5rem;font-family:var(--sans);font-size:15px;font-weight:600;">Axes in 3D Tensors</h3>
      <p>When you move to 3D, a new axis is added at the front. The standard convention in machine learning is <code>(batch, rows, columns)</code>:</p>
      <ul style="margin-left:2rem;color:var(--text2);margin-bottom:1rem;line-height:1.7;">
        <li><strong>Axis 0 (Batch):</strong> Depth or number of samples. Summing along axis 0 collapses all batches into a single matrix.</li>
        <li><strong>Axis 1 (Rows):</strong> Height of each matrix.</li>
        <li><strong>Axis 2 (Columns):</strong> Width of each matrix.</li>
      </ul>
    </div>

    <!-- 05 ELEMENT-WISE -->
    <a class="section-anchor" id="element-wise"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-teal">05</span><h3>Element-wise Operations</h3>
      </div>
      <p><strong>Element-wise operations</strong> apply a function independently to each matching pair of elements. Both arrays must have the same shape (or be broadcastable).</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]])
B = np.array([[<span class="nm">2</span>,<span class="nm">4</span>],[<span class="nm">6</span>,<span class="nm">8</span>]])

A + B      <span class="out"># [[3,6],[9,12]]</span>
A - B      <span class="out"># [[-1,-2],[-3,-4]]</span>
A * B      <span class="out"># [[2,8],[18,32]]</span>
A / B      <span class="out"># [[0.5,0.5],[0.5,0.5]]</span>
A ** <span class="nm">2</span>     <span class="out"># [[1,4],[9,16]]</span>
np.sqrt(A) <span class="out"># [[1.,1.41…],[1.73…,2.]]</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;">A = [[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]
B = [[<span class="nm">2</span>,<span class="nm">4</span>],[<span class="nm">6</span>,<span class="nm">8</span>]]

<span class="cm"># Addition</span>
<span class="kw">def</span> add(A, B):
    <span class="kw">return</span> [[A[i][j] + B[i][j]
             <span class="kw">for</span> j <span class="kw">in</span> range(len(A[<span class="nm">0</span>]))]
            <span class="kw">for</span> i <span class="kw">in</span> range(len(A))]

<span class="cm"># Element-wise multiply</span>
<span class="kw">def</span> mul(A, B):
    <span class="kw">return</span> [[A[i][j] * B[i][j]
             <span class="kw">for</span> j <span class="kw">in</span> range(len(A[<span class="nm">0</span>]))]
            <span class="kw">for</span> i <span class="kw">in</span> range(len(A))]

<span class="cm"># Power / sqrt</span>
<span class="kw">import</span> math
[[x**<span class="nm">2</span> <span class="kw">for</span> x <span class="kw">in</span> row] <span class="kw">for</span> row <span class="kw">in</span> A]
[[math.sqrt(x) <span class="kw">for</span> x <span class="kw">in</span> row] <span class="kw">for</span> row <span class="kw">in</span> A]</pre>
        </div>
      </div>
      <div class="callout callout-amber"><code>A * B</code> is element-wise multiplication. <code>A @ B</code> is matrix multiplication. They are completely different operations.</div>
    </div>

    <!-- 06 CONCATENATION -->
    <a class="section-anchor" id="concatenation"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-green">06</span><h3>Concatenation</h3>
      </div>
      <p>Joining arrays along an existing axis.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">a = np.array([[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]])   <span class="cm"># (2,2)</span>
b = np.array([[<span class="nm">5</span>,<span class="nm">6</span>]])          <span class="cm"># (1,2)</span>

<span class="cm"># Vertical stack (axis 0)</span>
np.concatenate([a, b], axis=<span class="nm">0</span>)   <span class="out"># (3,2)</span>
np.vstack([a, b])

<span class="cm"># Horizontal stack (axis 1)</span>
np.concatenate([a, b.T], axis=<span class="nm">1</span>) <span class="out"># (2,3)</span>
np.hstack([a, b.T])</pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;">a = [[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]
b = [[<span class="nm">5</span>,<span class="nm">6</span>]]

<span class="cm"># Vertical stack (add rows)</span>
<span class="kw">def</span> vstack(A, B):
    <span class="kw">return</span> A + B       <span class="out"># [[1,2],[3,4],[5,6]]</span>

<span class="cm"># Horizontal stack (add cols)</span>
<span class="kw">def</span> hstack(A, B):
    <span class="kw">return</span> [rowA + rowB
            <span class="kw">for</span> rowA, rowB <span class="kw">in</span> zip(A, B)]

<span class="cm"># transpose b first, then hstack</span>
bT = [list(row) <span class="kw">for</span> row <span class="kw">in</span> zip(*b)]
hstack(a, bT)  <span class="out"># [[1,2,5],[3,4,6]]</span></pre>
        </div>
      </div>
      <h3 style="margin:1.5rem 0 0.5rem;font-family:var(--sans);font-size:15px;font-weight:600;">Concatenating 3D Tensors</h3>
      <p>To join 3D tensors, the dimensions along all <em>non-concatenated</em> axes must perfectly match. For example, adding more samples to a batch means joining along <code>axis=0</code>.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">T1 = np.ones((<span class="nm">2</span>, <span class="nm">3</span>, <span class="nm">4</span>))  <span class="cm"># 2 batches</span>
T2 = np.ones((<span class="nm">5</span>, <span class="nm">3</span>, <span class="nm">4</span>))  <span class="cm"># 5 batches</span>

<span class="cm"># Join along batch dimension (axis 0)</span>
T_joined = np.concatenate([T1, T2], axis=<span class="nm">0</span>)
<span class="kw">print</span>(T_joined.shape)  <span class="out"># (7, 3, 4)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="cm"># Since axis 0 is the outermost list, </span>
<span class="cm"># we just add the lists together!</span>
T_joined = T1 + T2
<span class="cm"># Length goes from 2 to 7 batches</span></pre>
        </div>
      </div>
    </div>

    <!-- 07 DOT PRODUCT -->
    <a class="section-anchor" id="dot-product"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-purple">07</span><h3>The Dot Product</h3>
      </div>
      <p>The <strong>dot product</strong> of two equal-length vectors multiplies corresponding elements and sums the results into a single scalar — a measure of alignment between two vectors.</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;margin:1rem 0;text-align:center;">
        <div style="font-size:12px;color:var(--text3);margin-bottom:8px;">a · b = (1×2) + (3×4) + (5×6)</div>
        <div class="dot-calc">
          <div class="mul-pair"><span class="cell c-b">1</span><span class="dot-op" style="font-size:10px;">×</span><span class="cell c-g">2</span></div>
          <span class="dot-op">+</span>
          <div class="mul-pair"><span class="cell c-b">3</span><span class="dot-op" style="font-size:10px;">×</span><span class="cell c-g">4</span></div>
          <span class="dot-op">+</span>
          <div class="mul-pair"><span class="cell c-b">5</span><span class="dot-op" style="font-size:10px;">×</span><span class="cell c-g">6</span></div>
          <span class="dot-eq">=</span>
          <span class="dot-result">44</span>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">a = np.array([<span class="nm">1</span>, <span class="nm">3</span>, <span class="nm">5</span>])
b = np.array([<span class="nm">2</span>, <span class="nm">4</span>, <span class="nm">6</span>])

np.dot(a, b)  <span class="out"># 44</span>
a @ b         <span class="out"># 44  (@ operator)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;">a = [<span class="nm">1</span>, <span class="nm">3</span>, <span class="nm">5</span>]
b = [<span class="nm">2</span>, <span class="nm">4</span>, <span class="nm">6</span>]

<span class="cm"># Method 1 — index loop</span>
<span class="kw">def</span> dot(a, b):
    <span class="kw">return</span> sum(a[i] * b[i]
               <span class="kw">for</span> i <span class="kw">in</span> range(len(a)))

<span class="cm"># Method 2 — zip (Pythonic)</span>
<span class="kw">def</span> dot(a, b):
    <span class="kw">return</span> sum(x * y <span class="kw">for</span> x, y <span class="kw">in</span> zip(a, b))

dot(a, b)  <span class="out"># 44</span></pre>
        </div>
      </div>
      <div class="callout callout-blue">In neural networks, every layer computes: <strong>output = weights · inputs + bias</strong>. The dot product is the heartbeat of ML.</div>
    </div>

    <!-- 08 MATRIX MULTIPLICATION -->
    <a class="section-anchor" id="matmul"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-red">08</span><h3>Matrix Multiplication</h3>
      </div>
      <p><strong>Matrix multiplication</strong> is a series of dot products. It is the core operation of deep learning, driving everything from linear layers to attention mechanisms.</p>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:8px;">The golden rule</div>
        <div style="font-family:var(--mono);font-size:14px;margin-bottom:10px;">
          <span style="color:var(--text2)">A: m &times; </span><span style="color:var(--green)">n</span>
          <span style="color:var(--text3)"> &middot; </span>
          <span style="color:var(--text2)">B: </span><span style="color:var(--green)">n</span><span style="color:var(--text2)"> &times; p</span>
          <span style="color:var(--text3)"> = </span>
          <span style="color:var(--text2)">C: m &times; p</span>
        </div>
        <p style="font-size:13px;color:var(--text2);margin:0;">A's columns must equal B's rows &mdash; that shared <span style="color:var(--green)">n</span> is the dimension you "dot over". The result inherits A's row count and B's column count.</p>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:8px;">The formula</div>
        <div style="font-family:var(--mono);font-size:14px;margin-bottom:10px;">C[i][j] = &Sigma;<sub>k=0</sub><sup>n-1</sup> A[i][k] &middot; B[k][j]</div>
        <p style="font-size:13px;color:var(--text2);margin:0;">Each cell C[i][j] is the dot product of the entire <span style="color:var(--accent)">row i of A</span> with the entire <span style="color:var(--blue)">column j of B</span>. Multiply element-by-element, then sum.</p>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:8px;">Concrete trace &mdash; computing C[0][0] for a 2&times;3 &middot; 3&times;2 example</div>
        <p style="font-size:13px;color:var(--text2);margin-bottom:12px;">Here n = 3, so k runs 0 &rarr; 1 &rarr; 2 (three iterations, three pairs, three products summed):</p>
        <div class="grid-3" style="gap:10px;">
          <div style="background:var(--bg1);padding:10px;border-radius:var(--radius);text-align:center;">
            <div style="font-size:11px;color:var(--text3);margin-bottom:4px;">k = 0</div>
            <div style="font-family:var(--mono);font-size:13px;margin-bottom:4px;"><span style="color:var(--accent)">A[0][0]</span> &times; <span style="color:var(--blue)">B[0][0]</span></div>
            <div style="font-size:12px;color:var(--text2);">1 &times; 7 = 7</div>
          </div>
          <div style="background:var(--bg1);padding:10px;border-radius:var(--radius);text-align:center;">
            <div style="font-size:11px;color:var(--text3);margin-bottom:4px;">k = 1</div>
            <div style="font-family:var(--mono);font-size:13px;margin-bottom:4px;"><span style="color:var(--accent)">A[0][1]</span> &times; <span style="color:var(--blue)">B[1][0]</span></div>
            <div style="font-size:12px;color:var(--text2);">2 &times; 9 = 18</div>
          </div>
          <div style="background:var(--bg1);padding:10px;border-radius:var(--radius);text-align:center;">
            <div style="font-size:11px;color:var(--text3);margin-bottom:4px;">k = 2</div>
            <div style="font-family:var(--mono);font-size:13px;margin-bottom:4px;"><span style="color:var(--accent)">A[0][2]</span> &times; <span style="color:var(--blue)">B[2][0]</span></div>
            <div style="font-size:12px;color:var(--text2);">3 &times; 11 = 33</div>
          </div>
        </div>
        <div style="margin-top:12px;background:var(--bg1);padding:10px;border-radius:var(--radius);text-align:center;font-family:var(--mono);font-size:14px;">
          C[0][0] = 7 + 18 + 33 = <span style="color:var(--green)">58</span>
        </div>
      </div>

      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]])    <span class="cm"># (2,3)</span>
B = np.array([[<span class="nm">7</span>,<span class="nm">8</span>],[<span class="nm">9</span>,<span class="nm">10</span>],[<span class="nm">11</span>,<span class="nm">12</span>]]) <span class="cm"># (3,2)</span>

C = np.matmul(A, B)  <span class="out"># (2,2)</span>
C = A @ B            <span class="out"># identical</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python (no NumPy)</div>
          <pre style="margin:0;">A = [[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]]
B = [[<span class="nm">7</span>,<span class="nm">8</span>],[<span class="nm">9</span>,<span class="nm">10</span>],[<span class="nm">11</span>,<span class="nm">12</span>]]

<span class="kw">def</span> matmul(A, B):
    m = len(A)
    p = len(B[<span class="nm">0</span>])
    n = len(A[<span class="nm">0</span>]) <span class="cm"># shared dimension</span>
    C = [[<span class="nm">0</span>] * p <span class="kw">for</span> _ <span class="kw">in</span> range(m)]
    
    <span class="kw">for</span> i <span class="kw">in</span> range(m):         <span class="cm"># row in A</span>
        <span class="kw">for</span> j <span class="kw">in</span> range(p):     <span class="cm"># col in B</span>
            <span class="kw">for</span> k <span class="kw">in</span> range(n): <span class="cm"># dot product sum</span>
                C[i][j] += A[i][k] * B[k][j]
    <span class="kw">return</span> C</pre>
        </div>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">Why exactly three nested loops?</div>
        <div class="grid-3" style="gap:10px;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);border-top:2px solid var(--accent);">
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:6px;">for i in range(m)</div>
            <div style="font-weight:600;font-size:12px;margin-bottom:4px;">Selects a row in A</div>
            <div style="font-size:11px;color:var(--text3);">Runs m times. Determines which row of the result C we're filling.</div>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);border-top:2px solid var(--blue);">
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:6px;">for j in range(p)</div>
            <div style="font-weight:600;font-size:12px;margin-bottom:4px;">Selects a col in B</div>
            <div style="font-size:11px;color:var(--text3);">Runs p times. Determines which column of C we're computing.</div>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);border-top:2px solid var(--green);">
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:6px;">for k in range(n)</div>
            <div style="font-weight:600;font-size:12px;margin-bottom:4px;">The dot product</div>
            <div style="font-size:11px;color:var(--text3);">Runs n times. Multiplies A[i][k] &times; B[k][j] and accumulates.</div>
          </div>
        </div>
        <p style="font-size:12px;color:var(--text2);margin:12px 0 0 0;">Total operations: m &times; p &times; n multiplications. For a 2&times;3 and 3&times;2 matrix: 2&times;2&times;3 = 12 multiplications to produce 4 result cells.</p>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">2D vs 3D &mdash; what changes and what stays the same</div>
        <div class="grid-2" style="margin-bottom:12px;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;text-transform:uppercase;">2D Matmul</div>
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:4px;">A: m &times; <span style="color:var(--green)">n</span></div>
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:4px;">B: <span style="color:var(--green)">n</span> &times; p</div>
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:8px;">C: m &times; p</div>
            <div style="font-size:11px;color:var(--text2);">k runs over <span style="color:var(--green)">n</span></div>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);border:1px solid var(--accent);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;text-transform:uppercase;">3D Batched Matmul</div>
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:4px;">A: <span style="color:var(--accent)">b</span> &times; m &times; <span style="color:var(--green)">n</span></div>
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:4px;">B: <span style="color:var(--accent)">b</span> &times; <span style="color:var(--green)">n</span> &times; p</div>
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:8px;">C: <span style="color:var(--accent)">b</span> &times; m &times; p</div>
            <div style="font-size:11px;color:var(--text2);">k still runs over <span style="color:var(--green)">n</span> &mdash; batch <span style="color:var(--accent)">b</span> is just a new outer loop.</div>
          </div>
        </div>
        <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
          <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;text-transform:uppercase;">The four nested loops &mdash; batch wraps everything</div>
          <pre style="margin:0;font-size:12px;background:transparent;padding:0;">
<span style="color:var(--accent)">for b in range(batch):</span>  <span class="cm"># NEW OUTER LOOP</span>
    <span class="kw">for</span> i <span class="kw">in</span> range(m):  <span class="cm"># selects row in A[b]</span>
        <span class="kw">for</span> j <span class="kw">in</span> range(p):  <span class="cm"># selects col in B[b]</span>
            <span class="kw">for</span> k <span class="kw">in</span> range(n):  <span class="cm"># dot product</span>
                C<span style="color:var(--accent)">[b]</span>[i][j] += A<span style="color:var(--accent)">[b]</span>[i][k] * B<span style="color:var(--accent)">[b]</span>[k][j]</pre>
        </div>
      </div>

      <div class="callout callout-amber">The inner dimensions <strong>must match</strong>: (m × <strong>n</strong>) @ (<strong>n</strong> × p). If they don't, NumPy raises a ValueError. Always check shapes first.</div>
    </div>

    <!-- 09 NUMPY -->
    <a class="section-anchor" id="numpy"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-blue">09</span><h3>What is NumPy?</h3>
      </div>
      <p><strong>NumPy</strong> (Numerical Python) is the core library for numerical computing in Python. Its star is the <code>ndarray</code> — an N-dimensional array stored as a contiguous block of typed memory, enabling vectorised operations orders of magnitude faster than Python loops.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:0.85rem;">
          <div style="font-size:11px;color:var(--text3);font-weight:500;margin-bottom:6px;">Without NumPy (slow)</div>
          <pre style="margin:0;font-size:12px;">result = []
<span class="kw">for</span> i <span class="kw">in</span> range(len(a)):
    result.append(a[i] * b[i])</pre>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:0.85rem;">
          <div style="font-size:11px;color:var(--green);font-weight:500;margin-bottom:6px;">With NumPy (fast)</div>
          <pre style="margin:0;font-size:12px;">result = a * b
<span class="cm"># vectorised — runs in C</span></pre>
        </div>
      </div>
      <div class="callout callout-blue">NumPy is not just convenient — it is <strong>parallelised</strong>. Operations run on BLAS/LAPACK routines that exploit CPU vector instructions (SIMD), making them 10–100× faster than pure Python.</div>
    </div>

    <!-- 10 PARALLELIZATION -->
    <a class="section-anchor" id="parallelization"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-teal">10</span><h3>What is Parallelization &amp; Why it Matters</h3>
      </div>
      <p><strong>Parallelization</strong> means splitting a computation across multiple processing units (CPU cores, GPU threads) so they run simultaneously. Instead of doing 1 million multiplications one after another, you do them all at once.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;text-align:center;">
          <div style="font-size:12px;color:var(--text3);margin-bottom:8px;">Sequential</div>
          <div style="display:flex;gap:3px;justify-content:center;">
            <span class="cell c-d" style="font-size:10px;width:24px;height:22px;">1</span><span class="cell c-d" style="font-size:10px;width:24px;height:22px;">2</span><span class="cell c-d" style="font-size:10px;width:24px;height:22px;">3</span><span class="cell c-d" style="font-size:10px;width:24px;height:22px;">4</span><span class="cell c-d" style="font-size:10px;width:24px;height:22px;">5</span><span class="cell c-d" style="font-size:10px;width:24px;height:22px;">6</span>
          </div>
          <div style="font-size:11px;color:var(--text3);margin-top:6px;">→ → → → → → (one by one)</div>
          <div style="font-size:12px;color:var(--red);margin-top:4px;">slow</div>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;text-align:center;">
          <div style="font-size:12px;color:var(--text3);margin-bottom:8px;">Parallel (vectorised)</div>
          <div style="display:flex;gap:3px;justify-content:center;">
            <span class="cell c-g" style="font-size:10px;width:24px;height:22px;">1</span><span class="cell c-g" style="font-size:10px;width:24px;height:22px;">2</span><span class="cell c-g" style="font-size:10px;width:24px;height:22px;">3</span><span class="cell c-g" style="font-size:10px;width:24px;height:22px;">4</span><span class="cell c-g" style="font-size:10px;width:24px;height:22px;">5</span><span class="cell c-g" style="font-size:10px;width:24px;height:22px;">6</span>
          </div>
          <div style="font-size:11px;color:var(--green);margin-top:6px;">↓ ↓ ↓ ↓ ↓ ↓ (all at once)</div>
          <div style="font-size:12px;color:var(--green);margin-top:4px;">fast</div>
        </div>
      </div>
      <p>In ML, training on millions of examples would take <em>days</em> without parallelisation. NumPy's vectorised API, and frameworks like TensorFlow/PyTorch, express computation as array operations so the hardware (CPU/GPU/TPU) can parallelise automatically.</p>
    </div>

    <!-- 11 BROADCASTING -->
    <a class="section-anchor" id="broadcasting"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-purple">11</span><h3>Broadcasting</h3>
      </div>
      <p><strong>Broadcasting</strong> lets NumPy perform operations on arrays with different shapes by <em>virtually stretching</em> the smaller array to match the larger one — without copying data in memory.</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;text-align:center;">
        <div style="font-size:12px;color:var(--text3);margin-bottom:10px;">Add a (3,) row vector to a (3×3) matrix</div>
        <div style="display:flex;align-items:center;justify-content:center;gap:10px;flex-wrap:wrap;">
          <div class="matrix-wrap"><span class="bracket">[</span><div class="m-inner">
            <div class="m-row"><span class="cell c-d">1</span><span class="cell c-d">2</span><span class="cell c-d">3</span></div>
            <div class="m-row"><span class="cell c-d">4</span><span class="cell c-d">5</span><span class="cell c-d">6</span></div>
            <div class="m-row"><span class="cell c-d">7</span><span class="cell c-d">8</span><span class="cell c-d">9</span></div>
          </div><span class="bracket">]</span></div>
          <span class="arrow-label">+</span>
          <div class="matrix-wrap"><span class="bracket">[</span><div class="m-inner">
            <div class="m-row"><span class="cell c-g">10</span><span class="cell c-g">20</span><span class="cell c-g">30</span></div>
          </div><span class="bracket">]</span></div>
          <span class="arrow-label">=</span>
          <div class="matrix-wrap"><span class="bracket">[</span><div class="m-inner">
            <div class="m-row"><span class="cell c-b">11</span><span class="cell c-b">22</span><span class="cell c-b">33</span></div>
            <div class="m-row"><span class="cell c-b">14</span><span class="cell c-b">25</span><span class="cell c-b">36</span></div>
            <div class="m-row"><span class="cell c-b">17</span><span class="cell c-b">28</span><span class="cell c-b">39</span></div>
          </div><span class="bracket">]</span></div>
        </div>
        <div style="font-size:12px;color:var(--text3);font-family:var(--mono);margin-top:8px;">[10,20,30] was broadcast across all 3 rows automatically</div>
      </div>
      <pre>A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>],[<span class="nm">7</span>,<span class="nm">8</span>,<span class="nm">9</span>]])  <span class="cm"># (3,3)</span>
b = np.array([<span class="nm">10</span>, <span class="nm">20</span>, <span class="nm">30</span>])                   <span class="cm"># (3,)</span>
A + b   <span class="out"># b is broadcast row-wise</span>

<span class="cm"># Rule: align shapes from the RIGHT.</span>
<span class="cm"># Each dim must be equal OR one of them must be 1.</span>
<span class="cm"># (3,3) vs (3,) → (3,3) vs (1,3) → compatible ✓</span></pre>
      <div class="callout callout-amber">Always <code>print(A.shape, b.shape)</code> before operating on unfamiliar arrays. Silent shape mismatches produce wrong results, not errors.</div>
    </div>

    <!-- 12 NUMPY SLICING -->
    <a class="section-anchor" id="numpy-slicing"></a>

      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-blue">12</span><h3>NumPy Slicing</h3>
      </div>
      <p>NumPy slicing uses <code>start:stop:step</code> — stop is always <em>excluded</em>. For 2-D arrays, slice both dimensions at once: <code>A[rows, cols]</code>.</p>
      <pre>v = np.array([<span class="nm">10</span>, <span class="nm">20</span>, <span class="nm">30</span>, <span class="nm">40</span>, <span class="nm">50</span>])
v[<span class="nm">1</span>:<span class="nm">4</span>]        <span class="out"># [20, 30, 40]</span>
v[::<span class="nm">2</span>]        <span class="out"># [10, 30, 50]  (every other)</span>
v[<span class="nm">-1</span>]         <span class="out"># 50            (last element)</span>

A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>,<span class="nm">4</span>],[<span class="nm">5</span>,<span class="nm">6</span>,<span class="nm">7</span>,<span class="nm">8</span>],[<span class="nm">9</span>,<span class="nm">10</span>,<span class="nm">11</span>,<span class="nm">12</span>]])
A[<span class="nm">1</span>:, <span class="nm">1</span>:<span class="nm">3</span>]   <span class="out"># [[6,7],[10,11]]</span>
A[:, <span class="nm">0</span>]     <span class="out"># [1, 5, 9]    (entire first column)</span>
A[<span class="nm">0</span>, :]     <span class="out"># [1, 2, 3, 4] (entire first row)</span>
A[::<span class="nm">-1</span>]     <span class="out"># reverse rows</span>
A[:, ::<span class="nm">-1</span>]  <span class="out"># reverse columns</span>
A[A > <span class="nm">5</span>]    <span class="out"># boolean mask → [6,7,8,9,10,11,12]</span></pre>
      <div class="callout callout-amber">NumPy slices are <strong>views</strong>, not copies. Modifying a slice modifies the original array. Use <code>.copy()</code> to avoid surprises.</div>
    </div>

    <a class="section-anchor" id="py-representation"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-green">14</span><h3>Representing Vectors &amp; Matrices</h3>
      </div>
      <p>A <strong>vector</strong> is a plain list. A <strong>matrix</strong> is a list of lists — each inner list is one row.</p>
      <pre><span class="cm"># Vector</span>
v = [<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>, <span class="nm">4</span>]

<span class="cm"># Matrix (2D) — list of rows</span>
A = [[<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>],
     [<span class="nm">4</span>, <span class="nm">5</span>, <span class="nm">6</span>]]

<span class="cm"># Tensor (3D) — list of matrices (batches)</span>
T = [
  [[<span class="nm">1</span>, <span class="nm">2</span>], [<span class="nm">3</span>, <span class="nm">4</span>]],   <span class="cm"># batch 0</span>
  [[<span class="nm">5</span>, <span class="nm">6</span>], [<span class="nm">7</span>, <span class="nm">8</span>]]    <span class="cm"># batch 1</span>
]

A[<span class="nm">1</span>][<span class="nm">2</span>]      <span class="out"># 6  — row 1, col 2  (NOT A[1,2]!)</span>
T[<span class="nm">1</span>][<span class="nm">0</span>][<span class="nm">1</span>]   <span class="out"># 6  — batch 1, row 0, col 1</span></pre>
      <div class="callout callout-amber">Pure Python uses <code>A[row][col]</code> — NOT <code>A[row, col]</code>. That comma syntax is NumPy only and will raise a <strong>TypeError</strong> on plain lists.</div>
    </div>

<span class="kw">def</span> shape_3d(tensor):
    batches = len(tensor)
    rows = len(tensor[<span class="nm">0</span>])
    cols = len(tensor[<span class="nm">0</span>][<span class="nm">0</span>])
    <span class="kw">return</span> (batches, rows, cols)

A = [[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]]
T = [[[<span class="nm">1</span>,<span class="nm">2</span>]],[[<span class="nm">3</span>,<span class="nm">4</span>]],[[<span class="nm">5</span>,<span class="nm">6</span>]]]

shape_3d(T)    <span class="out"># (3, 1, 2)</span></pre>

    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-blue">16</span><h3>Transpose (Pure Python)</h3>
      </div>
      <p>Swap rows and columns: <code>T[j][i] = A[i][j]</code>.</p>
      <pre><span class="cm"># Explicit loops</span>
<span class="kw">def</span> transpose(A):
    rows, cols = len(A), len(A[<span class="nm">0</span>])
    T = []
    <span class="kw">for</span> j <span class="kw">in</span> range(cols):
        row = []
        <span class="kw">for</span> i <span class="kw">in</span> range(rows):
            row.append(A[i][j])
        T.append(row)
    <span class="kw">return</span> T

<span class="cm"># List comprehension</span>
<span class="kw">def</span> transpose(A):
    <span class="kw">return</span> [[A[i][j] <span class="kw">for</span> i <span class="kw">in</span> range(len(A))]
            <span class="kw">for</span> j <span class="kw">in</span> range(len(A[<span class="nm">0</span>]))]

<span class="cm"># Elegant zip trick</span>
<span class="kw">def</span> transpose(A):
    <span class="kw">return</span> [list(row) <span class="kw">for</span> row <span class="kw">in</span> zip(*A)]</pre>
      <div class="callout callout-blue">See <a href="#py-zip" style="color:var(--accent);font-weight:500;">§23 Zip &amp; Unpacking</a> for a deep dive into how <code>zip(*A)</code> works for transpose, dot products, and matrix multiplication.</div>

<span class="kw">def</span> add_vectors(a, b):

              <span class="kw">for</span> j <span class="kw">in</span> range(len(A[<span class="nm">0</span>]))]

<span class="cm"># Tensor addition (3D) — triple comprehension</span>
<span class="kw">def</span> add_3d(T1, T2):
    <span class="kw">return</span> [[[T1[b][i][j] + T2[b][i][j]
               <span class="kw">for</span> j <span class="kw">in</span> range(len(T1[<span class="nm">0</span>][<span class="nm">0</span>]))]
              <span class="kw">for</span> i <span class="kw">in</span> range(len(T1[<span class="nm">0</span>]))]

      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-green">20</span><h3>Matrix Multiplication (Pure Python)</h3>
      </div>
      <p>C[i][j] = dot product of row i of A with column j of B. The classic <strong>triple nested loop</strong>.</p>
      <pre><span class="kw">def</span> mat_mul(A, B):
    m, k, n = len(A), len(B), len(B[<span class="nm">0</span>])
    C = [[<span class="nm">0</span>] * n <span class="kw">for</span> _ <span class="kw">in</span> range(m)]
    <span class="kw">for</span> i <span class="kw">in</span> range(m):          <span class="cm"># each row of A</span>
        <span class="kw">for</span> j <span class="kw">in</span> range(n):      <span class="cm"># each col of B</span>
            <span class="kw">for</span> p <span class="kw">in</span> range(k):  <span class="cm"># shared dimension</span>
                C[i][j] += A[i][p] * B[p][j]
    <span class="kw">return</span> C

<span class="cm"># Batched Matrix Multiply (3D)</span>
<span class="cm"># shape: (batch, m, k) @ (batch, k, n) -> (batch, m, n)</span>
<span class="kw">def</span> bmm(T1, T2):
    <span class="kw">return</span> [mat_mul(T1[b], T2[b]) <span class="kw">for</span> b <span class="kw">in</span> range(len(T1))]</pre>
      <div class="callout callout-blue">This triple loop is O(m·k·n) — exactly why NumPy's C-backed matmul is so much faster. For 3D tensors, we do a "batched matmul" (bmm): multiplying corresponding matrices per batch.</div>
    </div>

      <p>Row slicing works directly on the outer list. <strong>Column slicing requires a list comprehension</strong> — there is no direct column slice on a plain list.</p>

      <div class="card">
        <div class="card-head"><span class="badge badge-teal">Setup</span><span>Reference matrix (used below)</span></div>
        <div class="card-body">
          <pre>A = [
    [<span class="nm"> 1</span>, <span class="nm"> 2</span>, <span class="nm"> 3</span>, <span class="nm"> 4</span>, <span class="nm"> 5</span>],   <span class="cm"># row 0</span>
    [<span class="nm"> 6</span>, <span class="nm"> 7</span>, <span class="nm"> 8</span>, <span class="nm"> 9</span>, <span class="nm">10</span>],   <span class="cm"># row 1</span>
    [<span class="nm">11</span>, <span class="nm">12</span>, <span class="nm">13</span>, <span class="nm">14</span>, <span class="nm">15</span>],   <span class="cm"># row 2</span>
    [<span class="nm">16</span>, <span class="nm">17</span>, <span class="nm">18</span>, <span class="nm">19</span>, <span class="nm">20</span>]    <span class="cm"># row 3</span>
]  <span class="cm"># shape: 4 rows × 5 cols</span></pre>
        </div>
      </div>

      <h3 style="margin:1.5rem 0 0.5rem;">Single element</h3>
      <pre>A[<span class="nm">1</span>][<span class="nm">2</span>]    <span class="out"># 8   — row 1, col 2</span>
A[<span class="nm">0</span>][<span class="nm">0</span>]    <span class="out"># 1   — top-left corner</span>
A[<span class="nm">-1</span>][<span class="nm">-1</span>]  <span class="out"># 20  — bottom-right corner</span></pre>
      <div class="callout callout-amber"><code>A[1, 2]</code> → <strong>TypeError</strong> on a plain list. Always use <code>A[row][col]</code>.</div>

      <h3 style="margin:1.5rem 0 0.5rem;">Entire row / range of rows</h3>
      <pre>A[<span class="nm">1</span>]         <span class="out"># [6, 7, 8, 9, 10]     — single row</span>
A[<span class="nm">1</span>:<span class="nm">3</span>]       <span class="out"># rows 1 and 2</span>
A[:<span class="nm">2</span>]        <span class="out"># first 2 rows</span>
A[<span class="nm">2</span>:]        <span class="out"># last 2 rows</span>
A[::<span class="nm">2</span>]       <span class="out"># every other row</span>
A[<span class="nm">-1</span>]        <span class="out"># last row</span>
A[<span class="nm">-2</span>:]       <span class="out"># last 2 rows</span></pre>

      <h3 style="margin:1.5rem 0 0.5rem;">Entire column / range of columns</h3>
      <pre><span class="cm"># Single column — NumPy: A[:, 2]</span>
[row[<span class="nm">2</span>] <span class="kw">for</span> row <span class="kw">in</span> A]        <span class="out"># [3, 8, 13, 18]</span>

<span class="cm"># Column range — NumPy: A[:, 1:4]</span>
[row[<span class="nm">1</span>:<span class="nm">4</span>] <span class="kw">for</span> row <span class="kw">in</span> A]      <span class="out"># [[2,3,4],[7,8,9],[12,13,14],[17,18,19]]</span>

<span class="cm"># Last column — NumPy: A[:, -1]</span>
[row[<span class="nm">-1</span>] <span class="kw">for</span> row <span class="kw">in</span> A]       <span class="out"># [5, 10, 15, 20]</span>

<span class="cm"># Last 2 cols — NumPy: A[:, -2:]</span>
[row[<span class="nm">-2</span>:] <span class="kw">for</span> row <span class="kw">in</span> A]      <span class="out"># [[4,5],[9,10],[14,15],[19,20]]</span></pre>
      <div class="callout callout-blue"><code>[row[j] for row in A]</code> is the pure-Python equivalent of NumPy's <code>A[:, j]</code>. Memorise this pattern.</div>

      <h3 style="margin:1.5rem 0 0.5rem;">Sub-matrix (rows AND columns)</h3>
      <pre><span class="cm"># rows 1–2, cols 2–3 — NumPy: A[1:3, 2:4]</span>
[row[<span class="nm">2</span>:<span class="nm">4</span>] <span class="kw">for</span> row <span class="kw">in</span> A[<span class="nm">1</span>:<span class="nm">3</span>]]   <span class="out"># [[8,9],[13,14]]</span>

<span class="cm"># General helper</span>
<span class="kw">def</span> submatrix(A, r1, r2, c1, c2):
    <span class="kw">return</span> [row[c1:c2] <span class="kw">for</span> row <span class="kw">in</span> A[r1:r2]]

<span class="cm"># First 2 rows, last 2 cols</span>
[row[<span class="nm">-2</span>:] <span class="kw">for</span> row <span class="kw">in</span> A[:<span class="nm">2</span>]]    <span class="out"># [[4,5],[9,10]]</span></pre>

      <h3 style="margin:1.5rem 0 0.5rem;">Step slicing &amp; reversal</h3>
      <pre><span class="cm"># Every other row</span>
A[::<span class="nm">2</span>]                          <span class="out"># rows 0 and 2</span>

<span class="cm"># Every other column</span>
[row[::<span class="nm">2</span>] <span class="kw">for</span> row <span class="kw">in</span> A]        <span class="out"># cols 0, 2, 4</span>

<span class="cm"># Every other row AND column</span>
[row[::<span class="nm">2</span>] <span class="kw">for</span> row <span class="kw">in</span> A[::<span class="nm">2</span>]]  <span class="out"># [[1,3,5],[11,13,15]]</span>

<span class="cm"># Reverse rows (upside-down)</span>
A[::<span class="nm">-1</span>]

<span class="cm"># Reverse columns (mirrored)</span>
[row[::<span class="nm">-1</span>] <span class="kw">for</span> row <span class="kw">in</span> A]

<span class="cm"># Fully reversed (upside-down AND mirrored)</span>
[row[::<span class="nm">-1</span>] <span class="kw">for</span> row <span class="kw">in</span> A[::<span class="nm">-1</span>]]</pre>

      <h3 style="margin:1.5rem 0 0.5rem;">Diagonal elements</h3>
      <pre><span class="cm"># Main diagonal</span>
[A[i][i] <span class="kw">for</span> i <span class="kw">in</span> range(min(len(A), len(A[<span class="nm">0</span>])))]
<span class="out"># [1, 7, 13, 19]</span>

<span class="cm"># Anti-diagonal</span>
n = len(A[<span class="nm">0</span>])
[A[i][n - <span class="nm">1</span> - i] <span class="kw">for</span> i <span class="kw">in</span> range(len(A))]
<span class="out"># [5, 9, 13, 17]</span></pre>

    <!-- ZIP & UNPACKING -->
    <a class="section-anchor" id="py-zip"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-indigo">23</span><h3>Zip &amp; Unpacking (Pure Python)</h3>
      </div>
      <p>The <code>zip()</code> function is the cornerstone of pure Python matrix operations. It pairs elements from multiple iterables, which is essential for operations like transposing and dot products.</p>
      
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">What zip() is</div>
        <p style="font-size:13px;color:var(--text2);margin-bottom:12px;"><code>zip()</code> takes two or more iterables and pairs them up element-by-element, producing a <strong>lazy iterator</strong> of tuples. It stops as soon as the shortest iterable is exhausted. Think of a physical zipper — two rows of teeth coming together one pair at a time.</p>
        <div class="grid-2" style="gap:1rem;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;text-transform:uppercase;">Basic pairing — visualised</div>
            <div style="font-family:var(--mono);font-size:12px;margin-bottom:8px;">
              <span style="color:var(--accent)">A</span>: [<span style="color:var(--accent)">1</span>, <span style="color:var(--accent)">2</span>, <span style="color:var(--accent)">3</span>]<br>
              <span style="color:var(--blue)">B</span>: [<span style="color:var(--blue)">4</span>, <span style="color:var(--blue)">5</span>, <span style="color:var(--blue)">6</span>]
            </div>
            <div style="font-family:var(--mono);font-size:12px;">
              zip(A, B) &rarr; (<span style="color:var(--accent)">1</span>,<span style="color:var(--blue)">4</span>), (<span style="color:var(--accent)">2</span>,<span style="color:var(--blue)">5</span>), (<span style="color:var(--accent)">3</span>,<span style="color:var(--blue)">6</span>)
            </div>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;text-transform:uppercase;">Lazy evaluation — no copied data</div>
            <p style="font-size:11px;color:var(--text2);margin:0;"><code>zip()</code> returns a generator that produces one tuple per iteration, on demand. No intermediate list is built. Zipping two 10,000-row matrices costs <strong>zero extra memory</strong> until you iterate.</p>
          </div>
        </div>
        <pre style="margin-top:12px;background:var(--bg1);">a = [<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>]
b = [<span class="nm">4</span>, <span class="nm">5</span>, <span class="nm">6</span>]

<span class="kw">for</span> x, y <span class="kw">in</span> zip(a, b):
    <span class="kw">print</span>(x, y)  <span class="out"># 1 4 | 2 5 | 3 6</span>

<span class="cm"># wrap in list() to materialise</span>
pairs = list(zip(a, b))  <span class="out"># [(1,4), (2,5), (3,6)]</span>

<span class="cm"># Three iterables at once</span>
list(zip(a, b, [<span class="nm">7</span>,<span class="nm">8</span>,<span class="nm">9</span>]))  <span class="out"># [(1,4,7), (2,5,8), (3,6,9)]</span></pre>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">The star (*) unpacking operator &mdash; essential for matrices</div>
        <p style="font-size:13px;color:var(--text2);margin-bottom:12px;"><code>zip(*matrix)</code> is the most important matrix trick. The <code>*</code> unpacks the outer list, passing each row as a separate argument to zip — which then pairs up the first elements of every row, then the second, etc. This is exactly a <strong>transpose</strong>.</p>
        <pre style="background:var(--bg1);margin:0;">M = [[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>], [<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]]

<span class="cm"># Without *: zip sees ONE argument (the whole matrix)</span>
list(zip(M))   <span class="out"># [([1,2,3],), ([4,5,6],)] — wrong!</span>

<span class="cm"># With *: zip sees TWO arguments (row0 and row1)</span>
list(zip(*M))  <span class="out"># [(1,4), (2,5), (3,6)] — columns!</span>

<span class="cm"># Equivalent to calling:</span>
list(zip([<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>], [<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]))</pre>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">zip in 2D &mdash; transposing rows to columns</div>
        <p style="font-size:13px;color:var(--text2);margin-bottom:12px;">A 2D matrix is a list of lists. <code>zip(*M)</code> treats each row as one iterable, pairing corresponding elements across all rows.</p>
        <div class="grid-2" style="margin-bottom:12px;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">Original M (3×3)</div>
            <div style="display:flex;flex-direction:column;gap:4px;">
              <div style="display:flex;gap:4px;"><span class="cell c-hi" style="width:24px;height:24px;">1</span><span class="cell c-hi" style="width:24px;height:24px;">2</span><span class="cell c-hi" style="width:24px;height:24px;">3</span></div>
              <div style="display:flex;gap:4px;"><span class="cell c-b" style="width:24px;height:24px;">4</span><span class="cell c-b" style="width:24px;height:24px;">5</span><span class="cell c-b" style="width:24px;height:24px;">6</span></div>
              <div style="display:flex;gap:4px;"><span class="cell c-g" style="width:24px;height:24px;">7</span><span class="cell c-g" style="width:24px;height:24px;">8</span><span class="cell c-g" style="width:24px;height:24px;">9</span></div>
            </div>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">Transposed T (3×3)</div>
            <div style="display:flex;flex-direction:column;gap:4px;">
              <div style="display:flex;gap:4px;"><span class="cell c-hi" style="width:24px;height:24px;">1</span><span class="cell c-b" style="width:24px;height:24px;">4</span><span class="cell c-g" style="width:24px;height:24px;">7</span></div>
              <div style="display:flex;gap:4px;"><span class="cell c-hi" style="width:24px;height:24px;">2</span><span class="cell c-b" style="width:24px;height:24px;">5</span><span class="cell c-g" style="width:24px;height:24px;">8</span></div>
              <div style="display:flex;gap:4px;"><span class="cell c-hi" style="width:24px;height:24px;">3</span><span class="cell c-b" style="width:24px;height:24px;">6</span><span class="cell c-g" style="width:24px;height:24px;">9</span></div>
            </div>
          </div>
        </div>
        <pre style="background:var(--bg1);margin:0;"><span class="cm"># zip(*M) yields tuples — convert each to list</span>
T = [list(col) <span class="kw">for</span> col <span class="kw">in</span> zip(*M)]
<span class="out"># [[1,4,7], [2,5,8], [3,6,9]]</span>

<span class="cm"># Or with map:</span>
T = list(map(list, zip(*M)))</pre>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">zip in 3D &mdash; pairing slices and rows</div>
        <p style="font-size:13px;color:var(--text2);margin-bottom:12px;">A 3D matrix is a list of 2D matrices (slices). <code>zip()</code> at the outermost level pairs batch slices together.</p>
        <div class="grid-2" style="gap:1rem;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">1. zip over batch dimension</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">Pairs 2D matrices from the same batch index. Foundation of batched operations.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;"><span class="kw">for</span> a_slice, b_slice <span class="kw">in</span> zip(A3, B3):
    <span class="cm"># a_slice and b_slice are 2D</span>
    <span class="kw">pass</span></pre>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">2. zip over rows within a 3D slice</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">Pairs rows, then pairs elements within rows.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">summed = [
  [a + b <span class="kw">for</span> a, b <span class="kw">in</span> zip(row_a, row_b)]
  <span class="kw">for</span> row_a, row_b <span class="kw">in</span> zip(slice_0, slice_1)
]</pre>
          </div>
        </div>
        <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);margin-top:12px;">
          <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">Transposing axes of a 3D matrix</div>
          <p style="font-size:11px;color:var(--text2);margin-bottom:8px;"><code>zip(*tensor)</code> swaps the <strong>batch axis</strong> with the <strong>row axis</strong> (equivalent to numpy transpose axes=(1,0,2)).</p>
          <pre style="margin:0;font-size:11px;background:transparent;padding:0;">T = [[[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]], [[<span class="nm">5</span>,<span class="nm">6</span>],[<span class="nm">7</span>,<span class="nm">8</span>]]]  <span class="cm"># shape (2,2,2)</span>
swapped = [list(pair) <span class="kw">for</span> pair <span class="kw">in</span> zip(*T)]
<span class="cm"># swapped shape is now (2,2,2) with axes swapped</span>
<span class="cm"># pair 0: ([1,2], [5,6]) &rarr; first row of each batch</span></pre>
        </div>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">Advanced ML Idioms &amp; Gotchas</div>
        
        <div class="grid-2" style="gap:1rem;margin-bottom:1rem;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">1. The Chunking Idiom</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">Using <code>iter()</code> with <code>zip</code> to group a flat list into chunks of <i>n</i>.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">data = [<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>, <span class="nm">4</span>, <span class="nm">5</span>, <span class="nm">6</span>, <span class="nm">7</span>, <span class="nm">8</span>, <span class="nm">9</span>]
chunked = list(zip(*[iter(data)] * <span class="nm">3</span>))
<span class="cm"># [(1, 2, 3), (4, 5, 6), (7, 8, 9)]</span></pre>
          </div>
          
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">2. Sliding Windows / Derivatives</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">Comparing consecutive elements (discrete derivatives) by zipping a list with a shifted version of itself.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">t_series = [<span class="nm">10</span>, <span class="nm">12</span>, <span class="nm">15</span>, <span class="nm">20</span>, <span class="nm">26</span>]
diffs = [n - p <span class="kw">for</span> p, n <span class="kw">in</span> zip(t_series, t_series[<span class="nm">1</span>:])]
<span class="cm"># [2, 3, 5, 6]  — represents rate of change</span></pre>
          </div>
        </div>

        <div class="grid-2" style="gap:1rem;margin-bottom:1rem;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">3. ML Application: Data Loaders</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">Iterating through features and labels simultaneously during training loops.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;"><span class="kw">for</span> epoch <span class="kw">in</span> range(epochs):
    <span class="kw">for</span> x_batch, y_batch <span class="kw">in</span> zip(X_train, y_train):
        prediction = model.forward(x_batch)
        loss = compute_loss(prediction, y_batch)
        <span class="cm"># backprop...</span></pre>
          </div>
          
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">4. zip vs map</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;"><code>map</code> can be used instead of list comprehension + zip for element-wise functions.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;"><span class="kw">import</span> operator
A, B = [<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">3</span>], [<span class="nm">4</span>, <span class="nm">5</span>, <span class="nm">6</span>]
<span class="cm"># zip</span>
add_zip = [a + b <span class="kw">for</span> a, b <span class="kw">in</span> zip(A, B)]
<span class="cm"># map</span>
add_map = list(map(operator.add, A, B))</pre>
          </div>
        </div>

        <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
          <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">5. Proving "Lazy Evaluation" with sys.getsizeof</div>
          <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">Why is lazy evaluation essential? Zipping two massive arrays takes almost zero memory until materialized into a list.</p>
          <pre style="margin:0;font-size:11px;background:transparent;padding:0;"><span class="kw">import</span> sys
a, b = range(<span class="nm">1_000_000</span>), range(<span class="nm">1_000_000</span>)

z = zip(a, b)
<span class="kw">print</span>(sys.getsizeof(z))  <span class="out"># 56 bytes! (Stores only iterator state)</span>

materialised = list(z)
<span class="kw">print</span>(sys.getsizeof(materialised))  <span class="out"># 8,000,056 bytes (8MB)</span></pre>
        </div>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">zip in Matrix Multiplication &mdash; transpose B, then dot</div>
        <p style="font-size:13px;color:var(--text2);margin-bottom:12px;">The classic matmul uses a triple nested loop (<code>i, j, k</code>). With <code>zip</code>, you transpose B so its columns become rows, then dot-product each row of A with each (former) column of B &mdash; eliminating the explicit <code>k</code> loop entirely.</p>
        <div class="grid-2" style="gap:1rem;margin-bottom:12px;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">Why zip(*B)?</div>
            <p style="font-size:11px;color:var(--text2);margin:0;">Columns of B can't be directly iterated in Python &mdash; there's no <code>B[:, j]</code> on a list-of-lists. <code>zip(*B)</code> transposes B, turning each column into an iterable row. Now <code>zip(row_A, col_B)</code> pairs elements for the dot product.</p>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">The formula</div>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">C[i][j] = &Sigma; A[i][k] &middot; B[k][j]
        = sum(a*b <span class="kw">for</span> a,b <span class="kw">in</span> zip(row_i, col_j))</pre>
          </div>
        </div>
        <pre style="background:var(--bg1);margin:0;"><span class="cm"># ── zip-based matmul (no explicit k loop) ─────────────────</span>
<span class="kw">def</span> matmul_zip(A, B):
    <span class="kw">assert</span> len(A[<span class="nm">0</span>]) == len(B), <span class="out">"cols_A must equal rows_B"</span>
    B_T = list(zip(*B))           <span class="cm"># transpose B: columns → rows</span>
    <span class="kw">return</span> [
        [sum(a * b <span class="kw">for</span> a, b <span class="kw">in</span> zip(row, col))  <span class="cm"># dot product</span>
         <span class="kw">for</span> col <span class="kw">in</span> B_T]                        <span class="cm"># each column of B</span>
        <span class="kw">for</span> row <span class="kw">in</span> A                             <span class="cm"># each row of A</span>
    ]

A = [[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]
B = [[<span class="nm">5</span>,<span class="nm">6</span>],[<span class="nm">7</span>,<span class="nm">8</span>]]
matmul_zip(A, B)  <span class="out"># [[19, 22], [43, 50]]</span>

<span class="cm"># ── Step trace for C[0][0]: ───────────────────────────────</span>
<span class="cm"># B_T = list(zip(*B)) → [(5,7), (6,8)]</span>
<span class="cm"># row = A[0] = [1,2],  col = B_T[0] = (5,7)</span>
<span class="cm"># zip(row, col) → (1,5) (2,7)</span>
<span class="cm"># products     →  5     14</span>
<span class="cm"># sum          →  19  ← C[0][0] ✓</span>

<span class="cm"># ── Non-square: (2×3) @ (3×2) → (2×2) ────────────────────</span>
A = [[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>]]
B = [[<span class="nm">7</span>,<span class="nm">8</span>],[<span class="nm">9</span>,<span class="nm">10</span>],[<span class="nm">11</span>,<span class="nm">12</span>]]
matmul_zip(A, B)  <span class="out"># [[58, 64], [139, 154]]</span>

<span class="cm"># ── Batched 3D matmul using zip to pair slices ────────────</span>
<span class="kw">def</span> batched_matmul(A3, B3):
    <span class="kw">return</span> [matmul_zip(a, b) <span class="kw">for</span> a, b <span class="kw">in</span> zip(A3, B3)]</pre>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">Gotchas &mdash; 6 traps every beginner hits</div>
        <div class="grid-2" style="gap:1rem;margin-bottom:1rem;">
          <div style="background:var(--red-bg);padding:12px;border-radius:var(--radius);border:1px solid rgba(139,42,26,.2);">
            <div style="font-size:11px;font-weight:600;color:var(--red);margin-bottom:6px;">⚠ 1. Iterator exhaustion</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:6px;">A zip object can only be iterated <strong>once</strong>. The second pass silently yields nothing.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">z = zip([<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>])
list(z)  <span class="out"># [(1,3),(2,4)]</span>
list(z)  <span class="out"># []  ← empty!</span>
<span class="cm"># Fix: materialise first</span>
z = list(zip([<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]))</pre>
          </div>
          <div style="background:var(--red-bg);padding:12px;border-radius:var(--radius);border:1px solid rgba(139,42,26,.2);">
            <div style="font-size:11px;font-weight:600;color:var(--red);margin-bottom:6px;">⚠ 2. Forgetting * on a matrix</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:6px;"><code>zip(M)</code> and <code>zip(*M)</code> are completely different.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">list(zip([[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]))
<span class="out"># [([1,2],), ([3,4],)] ← WRONG</span>
list(zip(*[[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]))
<span class="out"># [(1,3),(2,4)] ← correct</span></pre>
          </div>
        </div>
        <div class="grid-2" style="gap:1rem;margin-bottom:1rem;">
          <div style="background:var(--red-bg);padding:12px;border-radius:var(--radius);border:1px solid rgba(139,42,26,.2);">
            <div style="font-size:11px;font-weight:600;color:var(--red);margin-bottom:6px;">⚠ 3. Silent truncation</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:6px;">Mismatched lengths drop elements with no warning.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">list(zip([<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">10</span>,<span class="nm">20</span>]))
<span class="out"># [(1,10),(2,20)] ← 3 dropped!</span>
<span class="cm"># Fix: Python 3.10+</span>
zip([<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">10</span>,<span class="nm">20</span>], strict=<span class="kw">True</span>)
<span class="out"># ValueError raised</span></pre>
          </div>
          <div style="background:var(--red-bg);padding:12px;border-radius:var(--radius);border:1px solid rgba(139,42,26,.2);">
            <div style="font-size:11px;font-weight:600;color:var(--red);margin-bottom:6px;">⚠ 4. Tuples, not lists</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:6px;"><code>zip(*B)</code> yields tuples. Tuples are immutable — assignment crashes.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">B_T = list(zip(*[[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]))
B_T[<span class="nm">0</span>][<span class="nm">0</span>] = <span class="nm">99</span>  <span class="out"># TypeError!</span>
<span class="cm"># Fix: convert to lists</span>
B_T = [list(c) <span class="kw">for</span> c <span class="kw">in</span> zip(*B)]</pre>
          </div>
        </div>
        <div class="grid-2" style="gap:1rem;">
          <div style="background:var(--red-bg);padding:12px;border-radius:var(--radius);border:1px solid rgba(139,42,26,.2);">
            <div style="font-size:11px;font-weight:600;color:var(--red);margin-bottom:6px;">⚠ 5. zip never mutates</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:6px;">zip produces new output. The original is unchanged.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">M = [[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]
M = zip(*M)  <span class="out"># M is now a zip object!</span>
<span class="cm"># Fix: keep original, assign to new var</span>
M_T = [list(r) <span class="kw">for</span> r <span class="kw">in</span> zip(*M)]</pre>
          </div>
          <div style="background:var(--red-bg);padding:12px;border-radius:var(--radius);border:1px solid rgba(139,42,26,.2);">
            <div style="font-size:11px;font-weight:600;color:var(--red);margin-bottom:6px;">⚠ 6. Empty matrix edge case</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:6px;"><code>zip(*[])</code> is safe but returns nothing.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;"><span class="kw">def</span> safe_transpose(M):
    <span class="kw">if not</span> M <span class="kw">or not</span> M[<span class="nm">0</span>]:
        <span class="kw">return</span> []
    <span class="kw">return</span> [list(c) <span class="kw">for</span> c <span class="kw">in</span> zip(*M)]</pre>
          </div>
        </div>
      </div>

      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;">
        <div style="font-size:13px;font-weight:600;margin-bottom:12px;">Unzipping, zip_longest, strict mode &amp; enumerate</div>
        <div class="grid-2" style="gap:1rem;margin-bottom:1rem;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">Unzipping &mdash; zip is its own inverse</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;"><code>zip(*zipped)</code> reverses a previous zip. Same transpose trick &mdash; applied twice returns the original.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">zipped = [(<span class="nm">1</span>,<span class="nm">4</span>),(<span class="nm">2</span>,<span class="nm">5</span>),(<span class="nm">3</span>,<span class="nm">6</span>)]
a, b = zip(*zipped)
<span class="cm"># a = (1,2,3), b = (4,5,6)</span>

<span class="cm"># Matrix round-trip:</span>
M = [[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]
T = list(zip(*M))     <span class="cm"># transpose</span>
O = list(zip(*T))     <span class="cm"># back to original</span></pre>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">zip_longest &mdash; no truncation</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">Pads shorter iterables with a fill value instead of dropping elements. Essential for ragged data.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;"><span class="kw">from</span> itertools <span class="kw">import</span> zip_longest

list(zip_longest(
    [<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>], [<span class="nm">10</span>,<span class="nm">20</span>],
    fillvalue=<span class="nm">0</span>))
<span class="cm"># [(1,10),(2,20),(3,0)]</span></pre>
          </div>
        </div>
        <div class="grid-2" style="gap:1rem;">
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">strict=True (Python 3.10+)</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">Raises <code>ValueError</code> on length mismatch. Use for shape validation.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;"><span class="kw">def</span> safe_add(A, B):
    <span class="kw">return</span> [
      [a+b <span class="kw">for</span> a,b <span class="kw">in</span>
       zip(ra, rb, strict=<span class="kw">True</span>)]
      <span class="kw">for</span> ra,rb <span class="kw">in</span>
       zip(A, B, strict=<span class="kw">True</span>)
    ]</pre>
          </div>
          <div style="background:var(--bg1);padding:12px;border-radius:var(--radius);">
            <div style="font-size:11px;font-weight:600;color:var(--text3);margin-bottom:8px;">enumerate + zip</div>
            <p style="font-size:11px;color:var(--text2);margin-bottom:8px;">When you need the index and parallel values simultaneously.</p>
            <pre style="margin:0;font-size:11px;background:transparent;padding:0;">A = [[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]
B = [[<span class="nm">5</span>,<span class="nm">6</span>],[<span class="nm">7</span>,<span class="nm">8</span>]]
<span class="kw">for</span> i, (ra, rb) <span class="kw">in</span> enumerate(zip(A,B)):
    <span class="kw">print</span>(f<span class="out">"row {i}: {ra} + {rb}"</span>)
<span class="cm"># row 0: [1,2] + [5,6]</span>
<span class="cm"># row 1: [3,4] + [7,8]</span></pre>
          </div>
        </div>
      </div>

      <div class="callout callout-blue">
        <strong>Quick Reference:</strong>
        <code>zip(a,b)</code> = pair elements &nbsp;|&nbsp;
        <code>zip(*M)</code> = transpose &nbsp;|&nbsp;
        <code>sum(a*b for a,b in zip(r,c))</code> = dot product &nbsp;|&nbsp;
        <code>zip(*zipped)</code> = unzip &nbsp;|&nbsp;
        <code>zip_longest</code> = no truncation &nbsp;|&nbsp;
        <code>strict=True</code> = enforce equal length
      </div>

    <!-- 23 NORMS -->
    <a class="section-anchor" id="norms"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-green">24</span><h3>Norms (L1, L2, Frobenius)</h3>
      </div>
      <p>A <strong>norm</strong> is a function that measures the "size" or "length" of a vector or matrix. In machine learning, norms are heavily used in <strong>regularization</strong> to penalize large weights and prevent overfitting.</p>
      <div class="grid-3" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:8px;">L1 Norm (Manhattan)</div>
          <p style="font-size:12px;margin-bottom:8px;">Sum of absolute values. Used in Lasso regression. Drives weights to exactly zero (sparsity).</p>
          <div style="font-size:12px;font-family:var(--mono);">||v||₁ = Σ |v_i|</div>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:8px;">L2 Norm (Euclidean)</div>
          <p style="font-size:12px;margin-bottom:8px;">Square root of sum of squared values. Used in Ridge regression. Spreads weights out evenly.</p>
          <div style="font-size:12px;font-family:var(--mono);">||v||₂ = √(Σ v_i²)</div>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:8px;">Frobenius Norm</div>
          <p style="font-size:12px;margin-bottom:8px;">The L2 norm equivalent for matrices. Square root of sum of all squared elements.</p>
          <div style="font-size:12px;font-family:var(--mono);">||A||_F = √(ΣΣ A_ij²)</div>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">v = np.array([<span class="nm">-3</span>, <span class="nm">4</span>])
A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]])

<span class="cm"># Vector Norms</span>
np.linalg.norm(v, ord=<span class="nm">1</span>)  <span class="out"># L1: |-3| + |4| = 7.0</span>
np.linalg.norm(v)         <span class="out"># L2 (default): sqrt(9+16) = 5.0</span>

<span class="cm"># Matrix Norm</span>
np.linalg.norm(A)         <span class="out"># Frobenius (default for 2D)</span>
                          <span class="out"># sqrt(1+4+9+16) = 5.477</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="kw">import</span> math
v = [<span class="nm">-3</span>, <span class="nm">4</span>]
A = [[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]]

<span class="kw">def</span> l1_norm(v):
    <span class="kw">return</span> sum(abs(x) <span class="kw">for</span> x <span class="kw">in</span> v)

<span class="kw">def</span> l2_norm(v):
    <span class="kw">return</span> math.sqrt(sum(x**<span class="nm">2</span> <span class="kw">for</span> x <span class="kw">in</span> v))

<span class="kw">def</span> frobenius_norm(A):
    <span class="kw">return</span> math.sqrt(sum(x**<span class="nm">2</span> 
                         <span class="kw">for</span> row <span class="kw">in</span> A 
                         <span class="kw">for</span> x <span class="kw">in</span> row))</pre>
        </div>
      </div>
    </div>

    <!-- 24 EIGENVALUES & EIGENVECTORS -->
    <a class="section-anchor" id="eigen"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-teal">25</span><h3>Eigenvalues &amp; Eigenvectors</h3>
      </div>
      <p>When a matrix $A$ acts on a vector $v$, it generally rotates and stretches it. An <strong>eigenvector</strong> is a special vector that <em>only gets stretched</em>, not rotated, by $A$. The amount it stretches is the <strong>eigenvalue</strong> ($\lambda$).</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;text-align:center;">
        <div style="font-size:16px;font-family:var(--serif);margin-bottom:10px;">A · v = λ · v</div>
        <div style="font-size:12px;color:var(--text3);">Matrix transformation = Scalar scaling</div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">4</span>, <span class="nm">1</span>],
              [<span class="nm">2</span>, <span class="nm">3</span>]])

<span class="cm"># eigenvalues and eigenvectors</span>
eigenvalues, eigenvectors = np.linalg.eig(A)

<span class="kw">print</span>(eigenvalues) 
<span class="out"># [5. 2.]</span>

<span class="kw">print</span>(eigenvectors) 
<span class="out"># [[ 0.707 -0.447]</span>
<span class="out">#  [ 0.707  0.894]]</span>
<span class="cm"># Note: eigenvectors are COLUMNS in the result.</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Why it matters in ML</div>
          <p style="font-size:13px;line-height:1.6;"><strong>1. Dimensionality Reduction (PCA)</strong><br>The eigenvectors of a covariance matrix tell us the directions of maximum variance in our data. The eigenvalues tell us how much variance is in those directions.</p>
          <p style="font-size:13px;line-height:1.6;margin-top:10px;"><strong>2. Graph AI &amp; PageRank</strong><br>Google's original PageRank algorithm essentially computes the dominant eigenvector of the web's hyperlink matrix.</p>
        </div>
      </div>
      <div class="callout callout-amber"><code>np.linalg.eig</code> returns eigenvectors as columns of a matrix, so the first eigenvector is <code>eigenvectors[:, 0]</code>, not <code>eigenvectors[0]</code>.</div>
    </div>

    <!-- 25 SVD & PCA -->
    <a class="section-anchor" id="svd-pca"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-purple">26</span><h3>SVD &amp; PCA</h3>
      </div>
      <p><strong>Singular Value Decomposition (SVD)</strong> generalizes eigendecomposition to <em>any</em> matrix (not just square ones). It factors a matrix $A$ into three matrices: $U \Sigma V^T$.</p>
      <div class="grid-3" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:8px;">U (Left Singular Vectors)</div>
          <p style="font-size:12px;">Orthonormal basis for the output space. Describes row relationships.</p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:8px;">Σ (Singular Values)</div>
          <p style="font-size:12px;">Diagonal matrix. Values tell us the importance of each vector, sorted descending.</p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:8px;">V^T (Right Singular Vectors)</div>
          <p style="font-size:12px;">Orthonormal basis for the input space. Describes column relationships.</p>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ With NumPy (SVD)</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>, <span class="nm">2</span>],
              [<span class="nm">3</span>, <span class="nm">4</span>],
              [<span class="nm">5</span>, <span class="nm">6</span>]])

U, S, VT = np.linalg.svd(A, full_matrices=False)

<span class="cm"># U shape: (3, 2)</span>
<span class="cm"># S shape: (2,)  -> 1D array of singular values</span>
<span class="cm"># VT shape: (2, 2)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">▶ Connection to PCA</div>
          <p style="font-size:13px;line-height:1.6;"><strong>Principal Component Analysis (PCA)</strong> is an algorithm that uses SVD under the hood.</p>
          <pre style="margin:0;"><span class="cm"># 1. Center the data</span>
X_centered = X - np.mean(X, axis=<span class="nm">0</span>)

<span class="cm"># 2. Compute SVD</span>
U, S, VT = np.linalg.svd(X_centered, full_matrices=False)

<span class="cm"># 3. Project onto top k components</span>
<span class="cm"># VT holds the principal components in its rows</span>
components = VT[:k] 
X_reduced = np.dot(X_centered, components.T)</pre>
        </div>
      </div>
    </div>

    <!-- 26 DETERMINANTS -->
    <a class="section-anchor" id="determinants"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-red">27</span><h3>Determinants</h3>
      </div>
      <p>The <strong>determinant</strong> is a scalar value that encodes key properties of a square matrix. It tells you whether a matrix is <strong>invertible</strong> (det &ne; 0), and geometrically it measures the <strong>signed volume scaling</strong> factor of the linear transformation.</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;text-align:center;">
        <div style="font-size:14px;font-family:var(--mono);margin-bottom:8px;">2&times;2: det(A) = ad &minus; bc</div>
        <div style="font-size:12px;color:var(--text3);">For A = [[a,b],[c,d]]</div>
      </div>
      <p>For larger matrices, we use <strong>cofactor expansion</strong> (Laplace expansion) &mdash; recursively breaking an n&times;n determinant into smaller (n-1)&times;(n-1) determinants.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>, <span class="nm">2</span>],
              [<span class="nm">3</span>, <span class="nm">4</span>]])
np.linalg.det(A)  <span class="out"># -2.0  (1*4 - 2*3)</span>

B = np.array([[<span class="nm">6</span>,<span class="nm">1</span>,<span class="nm">1</span>],
              [<span class="nm">4</span>,<span class="nm">-2</span>,<span class="nm">5</span>],
              [<span class="nm">2</span>,<span class="nm">8</span>,<span class="nm">7</span>]])
np.linalg.det(B)  <span class="out"># -306.0</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="kw">def</span> det(A):
    n = len(A)
    <span class="kw">if</span> n == <span class="nm">1</span>:
        <span class="kw">return</span> A[<span class="nm">0</span>][<span class="nm">0</span>]
    <span class="kw">if</span> n == <span class="nm">2</span>:
        <span class="kw">return</span> A[<span class="nm">0</span>][<span class="nm">0</span>]*A[<span class="nm">1</span>][<span class="nm">1</span>] - A[<span class="nm">0</span>][<span class="nm">1</span>]*A[<span class="nm">1</span>][<span class="nm">0</span>]
    total = <span class="nm">0</span>
    <span class="kw">for</span> j <span class="kw">in</span> range(n):
        <span class="cm"># Minor: delete row 0, col j</span>
        minor = [[A[i][k] <span class="kw">for</span> k <span class="kw">in</span> range(n)
                  <span class="kw">if</span> k != j]
                 <span class="kw">for</span> i <span class="kw">in</span> range(<span class="nm">1</span>, n)]
        sign = (-<span class="nm">1</span>) ** j
        total += sign * A[<span class="nm">0</span>][j] * det(minor)
    <span class="kw">return</span> total</pre>
        </div>
      </div>
      <div class="callout callout-amber">det(A) = 0 means the matrix is <strong>singular</strong> (not invertible). Rows/columns are linearly dependent &mdash; the transformation collapses space into a lower dimension.</div>
    </div>

    <!-- 27 MINORS & COFACTORS -->
    <a class="section-anchor" id="minors-cofactors"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-teal">28</span><h3>Minors &amp; Cofactors</h3>
      </div>
      <p>The <strong>minor</strong> M<sub>ij</sub> is the determinant of the sub-matrix formed by deleting row i and column j. The <strong>cofactor</strong> C<sub>ij</sub> = (-1)<sup>i+j</sup> &times; M<sub>ij</sub> adds a checkerboard sign pattern.</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;text-align:center;">
        <div style="font-size:14px;font-family:var(--mono);margin-bottom:8px;">C<sub>ij</sub> = (-1)<sup>i+j</sup> &times; det(A with row i, col j removed)</div>
        <div style="font-size:12px;color:var(--text3);">Sign pattern: + &minus; + / &minus; + &minus; / + &minus; +</div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],
              [<span class="nm">0</span>,<span class="nm">4</span>,<span class="nm">5</span>],
              [<span class="nm">1</span>,<span class="nm">0</span>,<span class="nm">6</span>]])

<span class="kw">def</span> minor_matrix(A):
    n = A.shape[<span class="nm">0</span>]
    M = np.zeros((n, n))
    <span class="kw">for</span> i <span class="kw">in</span> range(n):
        <span class="kw">for</span> j <span class="kw">in</span> range(n):
            sub = np.delete(np.delete(A, i, <span class="nm">0</span>), j, <span class="nm">1</span>)
            M[i,j] = np.linalg.det(sub)
    <span class="kw">return</span> M

<span class="kw">def</span> cofactor_matrix(A):
    M = minor_matrix(A)
    n = A.shape[<span class="nm">0</span>]
    signs = np.array([[(-<span class="nm">1</span>)**(i+j)
             <span class="kw">for</span> j <span class="kw">in</span> range(n)]
            <span class="kw">for</span> i <span class="kw">in</span> range(n)])
    <span class="kw">return</span> signs * M</pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="kw">def</span> get_minor(A, i, j):
    <span class="st">"""Sub-matrix with row i, col j removed"""</span>
    <span class="kw">return</span> [[A[r][c] <span class="kw">for</span> c <span class="kw">in</span> range(len(A))
             <span class="kw">if</span> c != j]
            <span class="kw">for</span> r <span class="kw">in</span> range(len(A))
            <span class="kw">if</span> r != i]

<span class="kw">def</span> minor_matrix(A):
    n = len(A)
    <span class="kw">return</span> [[det(get_minor(A, i, j))
             <span class="kw">for</span> j <span class="kw">in</span> range(n)]
            <span class="kw">for</span> i <span class="kw">in</span> range(n)]

<span class="kw">def</span> cofactor_matrix(A):
    n = len(A)
    M = minor_matrix(A)
    <span class="kw">return</span> [[(-<span class="nm">1</span>)**(i+j) * M[i][j]
             <span class="kw">for</span> j <span class="kw">in</span> range(n)]
            <span class="kw">for</span> i <span class="kw">in</span> range(n)]</pre>
        </div>
      </div>
      <div class="callout callout-blue">Cofactors are the building blocks for both the <strong>determinant</strong> (cofactor expansion) and the <strong>adjugate matrix</strong> (used to compute the inverse).</div>
    </div>

    <!-- 28 ADJUGATE & INVERSE -->
    <a class="section-anchor" id="adjugate-inverse"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-purple">29</span><h3>Adjugate &amp; Inverse</h3>
      </div>
      <p>The <strong>adjugate</strong> (or classical adjoint) is the transpose of the cofactor matrix. The <strong>inverse</strong> A<sup>-1</sup> satisfies A &middot; A<sup>-1</sup> = I (identity). It only exists when det(A) &ne; 0.</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;text-align:center;">
        <div style="font-size:14px;font-family:var(--mono);margin-bottom:4px;">adj(A) = cofactor(A)<sup>T</sup></div>
        <div style="font-size:14px;font-family:var(--mono);margin-bottom:8px;">A<sup>-1</sup> = (1 / det(A)) &times; adj(A)</div>
        <div style="font-size:12px;color:var(--text3);">For 2&times;2: A<sup>-1</sup> = (1/(ad-bc)) &times; [[d,-b],[-c,a]]</div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>, <span class="nm">2</span>],
              [<span class="nm">3</span>, <span class="nm">4</span>]])

A_inv = np.linalg.inv(A)
<span class="kw">print</span>(A_inv)
<span class="out"># [[-2. ,  1. ],</span>
<span class="out">#  [ 1.5, -0.5]]</span>

<span class="cm"># Verify: A @ A_inv = Identity</span>
np.allclose(A @ A_inv, np.eye(<span class="nm">2</span>))  <span class="out"># True</span>

<span class="cm"># Pseudo-inverse (works for non-square)</span>
np.linalg.pinv(A)</pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="kw">def</span> adjugate(A):
    C = cofactor_matrix(A)
    <span class="kw">return</span> transpose(C)

<span class="kw">def</span> inverse(A):
    d = det(A)
    <span class="kw">if</span> d == <span class="nm">0</span>:
        <span class="kw">raise</span> ValueError(<span class="st">"Singular matrix"</span>)
    adj = adjugate(A)
    n = len(A)
    <span class="kw">return</span> [[adj[i][j] / d
             <span class="kw">for</span> j <span class="kw">in</span> range(n)]
            <span class="kw">for</span> i <span class="kw">in</span> range(n)]

<span class="cm"># Quick 2x2 shortcut</span>
<span class="kw">def</span> inv_2x2(A):
    a,b = A[<span class="nm">0</span>]; c,d = A[<span class="nm">1</span>]
    D = a*d - b*c
    <span class="kw">return</span> [[d/D, -b/D],
            [-c/D, a/D]]</pre>
        </div>
      </div>
      <div class="callout callout-amber">In ML, the <strong>normal equation</strong> for linear regression is &beta; = (X<sup>T</sup>X)<sup>-1</sup>X<sup>T</sup>y. Computing the inverse directly is O(n&sup3;) and numerically unstable &mdash; use <code>np.linalg.solve</code> or <code>np.linalg.lstsq</code> in practice.</div>
    </div>

    <!-- 29 MATRIX POWER -->
    <a class="section-anchor" id="matrix-power"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-blue">30</span><h3>Matrix Power</h3>
      </div>
      <p>Raising a square matrix to an integer power means multiplying it by itself k times: A<sup>k</sup> = A &times; A &times; &hellip; &times; A. Negative powers use the inverse: A<sup>-k</sup> = (A<sup>-1</sup>)<sup>k</sup>.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">2</span>, <span class="nm">1</span>],
              [<span class="nm">0</span>, <span class="nm">3</span>]])

np.linalg.matrix_power(A, <span class="nm">3</span>)
<span class="out"># [[8, 13],</span>
<span class="out">#  [0, 27]]</span>

<span class="cm"># A^0 = Identity</span>
np.linalg.matrix_power(A, <span class="nm">0</span>)  <span class="out"># [[1,0],[0,1]]</span>

<span class="cm"># Negative power = inverse raised</span>
np.linalg.matrix_power(A, -<span class="nm">1</span>)  <span class="out"># same as inv(A)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="kw">def</span> mat_power(A, k):
    n = len(A)
    <span class="cm"># Identity matrix</span>
    result = [[<span class="nm">1</span> <span class="kw">if</span> i==j <span class="kw">else</span> <span class="nm">0</span>
               <span class="kw">for</span> j <span class="kw">in</span> range(n)]
              <span class="kw">for</span> i <span class="kw">in</span> range(n)]
    <span class="kw">if</span> k < <span class="nm">0</span>:
        A = inverse(A)
        k = -k
    <span class="kw">for</span> _ <span class="kw">in</span> range(k):
        result = mat_mul(result, A)
    <span class="kw">return</span> result

mat_power([[<span class="nm">2</span>,<span class="nm">1</span>],[<span class="nm">0</span>,<span class="nm">3</span>]], <span class="nm">3</span>)
<span class="out"># [[8, 13], [0, 27]]</span></pre>
        </div>
      </div>
      <div class="callout callout-blue">Matrix powers appear in <strong>Markov chains</strong> (state transition after k steps), <strong>graph theory</strong> (counting k-length paths), and solving systems of linear differential equations.</div>
    </div>

    <!-- 30 QR & CHOLESKY -->
    <a class="section-anchor" id="qr-cholesky"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-green">31</span><h3>QR &amp; Cholesky Decompositions</h3>
      </div>
      <p><strong>QR decomposition</strong> factors A = QR where Q is orthogonal (Q<sup>T</sup>Q = I) and R is upper-triangular. <strong>Cholesky decomposition</strong> factors a symmetric positive-definite matrix as A = LL<sup>T</sup> where L is lower-triangular.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:8px;">QR Decomposition</div>
          <p style="font-size:12px;margin-bottom:4px;">A = Q &times; R</p>
          <p style="font-size:12px;color:var(--text3);">Used in: least-squares, eigenvalue algorithms, solving Ax=b numerically stably</p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:8px;">Cholesky Decomposition</div>
          <p style="font-size:12px;margin-bottom:4px;">A = L &times; L<sup>T</sup></p>
          <p style="font-size:12px;color:var(--text3);">Used in: sampling from multivariate Gaussians, solving positive-definite systems (~2&times; faster than LU)</p>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; QR with NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">1</span>],[<span class="nm">1</span>,<span class="nm">0</span>],[<span class="nm">0</span>,<span class="nm">1</span>]])
Q, R = np.linalg.qr(A)
<span class="kw">print</span>(Q.shape)  <span class="out"># (3, 2)</span>
<span class="kw">print</span>(R.shape)  <span class="out"># (2, 2)</span>

<span class="cm"># Verify: Q @ R ≈ A</span>
np.allclose(Q @ R, A)  <span class="out"># True</span>
<span class="cm"># Q columns are orthonormal</span>
np.allclose(Q.T @ Q, np.eye(<span class="nm">2</span>))  <span class="out"># True</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Cholesky with NumPy</div>
          <pre style="margin:0;"><span class="cm"># Must be symmetric positive-definite</span>
A = np.array([[<span class="nm">4</span>,<span class="nm">2</span>],[<span class="nm">2</span>,<span class="nm">3</span>]])

L = np.linalg.cholesky(A)
<span class="kw">print</span>(L)
<span class="out"># [[2.  , 0.  ],</span>
<span class="out">#  [1.  , 1.41]]</span>

<span class="cm"># Verify: L @ L.T ≈ A</span>
np.allclose(L @ L.T, A)  <span class="out"># True</span></pre>
        </div>
      </div>
      <div class="callout callout-blue"><strong>Cholesky</strong> is essential in Bayesian ML: to sample from N(&mu;, &Sigma;), compute L = cholesky(&Sigma;), then sample z ~ N(0,I) and return &mu; + Lz.</div>
    </div>

    <!-- 31 GRAM-SCHMIDT -->
    <a class="section-anchor" id="gram-schmidt"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-amber">32</span><h3>Gram-Schmidt Orthogonalization</h3>
      </div>
      <p>The <strong>Gram-Schmidt</strong> process takes a set of linearly independent vectors and produces an <strong>orthonormal</strong> set (mutually perpendicular unit vectors). It is the algorithm behind QR decomposition.</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;text-align:center;">
        <div style="font-size:13px;font-family:var(--mono);margin-bottom:4px;">u<sub>k</sub> = v<sub>k</sub> &minus; &Sigma; proj(u<sub>j</sub>, v<sub>k</sub>)</div>
        <div style="font-size:13px;font-family:var(--mono);margin-bottom:8px;">e<sub>k</sub> = u<sub>k</sub> / ||u<sub>k</sub>||</div>
        <div style="font-size:12px;color:var(--text3);">Subtract all projections onto previous vectors, then normalise</div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;"><span class="kw">def</span> gram_schmidt_np(V):
    <span class="st">"""V: columns are input vectors"""</span>
    n = V.shape[<span class="nm">1</span>]
    U = np.zeros_like(V, dtype=float)
    <span class="kw">for</span> i <span class="kw">in</span> range(n):
        u = V[:, i].astype(float)
        <span class="kw">for</span> j <span class="kw">in</span> range(i):
            u -= np.dot(U[:,j], V[:,i]) * U[:,j]
        U[:, i] = u / np.linalg.norm(u)
    <span class="kw">return</span> U

V = np.array([[<span class="nm">1</span>,<span class="nm">1</span>],[<span class="nm">1</span>,<span class="nm">0</span>]])
Q = gram_schmidt_np(V)
<span class="cm"># Q columns are orthonormal</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="kw">def</span> proj(u, v):
    <span class="st">"""Project v onto u"""</span>
    dot_uv = sum(a*b <span class="kw">for</span> a,b <span class="kw">in</span> zip(u,v))
    dot_uu = sum(a*a <span class="kw">for</span> a <span class="kw">in</span> u)
    <span class="kw">return</span> [dot_uv/dot_uu * a <span class="kw">for</span> a <span class="kw">in</span> u]

<span class="kw">def</span> gram_schmidt(vectors):
    ortho = []
    <span class="kw">for</span> v <span class="kw">in</span> vectors:
        u = v[:]
        <span class="kw">for</span> o <span class="kw">in</span> ortho:
            p = proj(o, v)
            u = [u[i]-p[i] <span class="kw">for</span> i <span class="kw">in</span> range(len(u))]
        norm = l2_norm(u)
        ortho.append([x/norm <span class="kw">for</span> x <span class="kw">in</span> u])
    <span class="kw">return</span> ortho</pre>
        </div>
      </div>
    </div>

    <!-- 32 SOLVING LINEAR SYSTEMS -->
    <a class="section-anchor" id="linear-systems"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-red">33</span><h3>Solving Linear Systems</h3>
      </div>
      <p>A <strong>linear system</strong> Ax = b asks: given matrix A and vector b, find vector x. This is the mathematical core of regression, curve fitting, and engineering.</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;text-align:center;">
        <div style="font-size:14px;font-family:var(--mono);margin-bottom:8px;">Ax = b &rarr; x = A<sup>-1</sup>b</div>
        <div style="font-size:12px;color:var(--text3);">Direct inverse is O(n&sup3;) and unstable. Prefer factorisation-based solvers.</div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;"><span class="cm"># Exact solution (square A)</span>
A = np.array([[<span class="nm">3</span>,<span class="nm">1</span>],[<span class="nm">1</span>,<span class="nm">2</span>]])
b = np.array([<span class="nm">9</span>, <span class="nm">8</span>])
x = np.linalg.solve(A, b)
<span class="kw">print</span>(x)  <span class="out"># [2. 3.]</span>

<span class="cm"># Least-squares (overdetermined)</span>
A = np.array([[<span class="nm">1</span>,<span class="nm">1</span>],[<span class="nm">2</span>,<span class="nm">1</span>],[<span class="nm">3</span>,<span class="nm">1</span>]])
b = np.array([<span class="nm">1</span>, <span class="nm">2</span>, <span class="nm">2</span>])
x, res, rank, sv = np.linalg.lstsq(A, b, rcond=None)
<span class="kw">print</span>(x)  <span class="out"># [0.5, 0.333...]</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python (no NumPy)</div>
          <pre style="margin:0;"><span class="kw">def</span> solve_2x2(A, b):
    <span class="st">"""Cramer's rule for 2x2"""</span>
    D  = det(A)
    Dx = [[b[<span class="nm">0</span>], A[<span class="nm">0</span>][<span class="nm">1</span>]],
          [b[<span class="nm">1</span>], A[<span class="nm">1</span>][<span class="nm">1</span>]]]
    Dy = [[A[<span class="nm">0</span>][<span class="nm">0</span>], b[<span class="nm">0</span>]],
          [A[<span class="nm">1</span>][<span class="nm">0</span>], b[<span class="nm">1</span>]]]
    <span class="kw">return</span> [det(Dx)/D, det(Dy)/D]

<span class="kw">def</span> solve_inverse(A, b):
    <span class="st">"""General: x = A_inv @ b"""</span>
    A_inv = inverse(A)
    n = len(b)
    <span class="kw">return</span> [sum(A_inv[i][j]*b[j]
                <span class="kw">for</span> j <span class="kw">in</span> range(n))
            <span class="kw">for</span> i <span class="kw">in</span> range(n)]

solve_2x2([[<span class="nm">3</span>,<span class="nm">1</span>],[<span class="nm">1</span>,<span class="nm">2</span>]], [<span class="nm">9</span>,<span class="nm">8</span>])
<span class="out"># [2.0, 3.0]</span></pre>
        </div>
      </div>
      <div class="callout callout-amber"><code>np.linalg.solve</code> uses LU decomposition internally and is both faster and more numerically stable than computing <code>inv(A) @ b</code>. <strong>Always prefer solve over inv.</strong></div>
    </div>

    <!-- 33 TRACE -->
    <a class="section-anchor" id="trace"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-blue">34</span><h3>Trace</h3>
      </div>
      <p>The <strong>trace</strong> of a square matrix is the sum of its diagonal elements. It appears constantly in matrix calculus — the Frobenius inner product is tr(A<sup>T</sup>B), and many loss functions are expressed as traces.</p>
      <div style="background:var(--bg2);border-radius:var(--radius);padding:1.25rem;margin:1rem 0;text-align:center;">
        <div style="font-size:14px;font-family:var(--mono);margin-bottom:8px;">tr(A) = &Sigma; A<sub>ii</sub> = a<sub>11</sub> + a<sub>22</sub> + &hellip; + a<sub>nn</sub></div>
        <div style="font-size:12px;color:var(--text3);">Properties: tr(A+B) = tr(A) + tr(B) &bull; tr(AB) = tr(BA) &bull; tr(A) = &Sigma; eigenvalues</div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]])
np.trace(A)  <span class="out"># 5  (1 + 4)</span>

<span class="cm"># Frobenius inner product</span>
np.trace(A.T @ A)  <span class="out"># same as np.sum(A**2)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python</div>
          <pre style="margin:0;"><span class="kw">def</span> trace(A):
    <span class="kw">return</span> sum(A[i][i] <span class="kw">for</span> i <span class="kw">in</span> range(len(A)))

trace([[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>]])  <span class="out"># 5</span></pre>
        </div>
      </div>
    </div>

    <!-- 34 RANK -->
    <a class="section-anchor" id="rank"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-teal">35</span><h3>Rank &amp; Linear Independence</h3>
      </div>
      <p>The <strong>rank</strong> of a matrix is the number of linearly independent rows (or columns). Vectors are <strong>linearly independent</strong> if none can be written as a combination of the others. Rank tells you the &ldquo;true dimensionality&rdquo; of your data.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Full rank</div>
          <p style="font-size:12px;">rank(A) = min(m,n). All rows/cols independent. System has a unique solution.</p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Rank-deficient</div>
          <p style="font-size:12px;">rank(A) &lt; min(m,n). Redundant features. Infinite or no solutions.</p>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],
              [<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>],
              [<span class="nm">7</span>,<span class="nm">8</span>,<span class="nm">9</span>]])
np.linalg.matrix_rank(A)  <span class="out"># 2 (row 3 = 2*row2 - row1)</span>

B = np.eye(<span class="nm">3</span>)
np.linalg.matrix_rank(B)  <span class="out"># 3 (full rank)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Why it matters in ML</div>
          <p style="font-size:13px;line-height:1.6;">If your feature matrix X is rank-deficient, <strong>X<sup>T</sup>X</strong> is singular and the normal equation has no unique solution. This is why <strong>regularization</strong> adds &lambda;I &mdash; it guarantees full rank.</p>
        </div>
      </div>
    </div>

    <!-- 35 POSITIVE DEFINITE -->
    <a class="section-anchor" id="pos-def"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-green">36</span><h3>Positive Definite Matrices</h3>
      </div>
      <p>A symmetric matrix A is <strong>positive definite</strong> (PD) if x<sup>T</sup>Ax &gt; 0 for every non-zero vector x. Equivalently: all eigenvalues are positive. PD matrices guarantee a <strong>unique minimum</strong> in optimization.</p>
      <div class="grid-3" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Positive Definite</div>
          <p style="font-size:12px;">All eigenvalues &gt; 0. Invertible. Cholesky works.</p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Positive Semi-definite</div>
          <p style="font-size:12px;">All eigenvalues &ge; 0. Covariance matrices are always PSD.</p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Indefinite</div>
          <p style="font-size:12px;">Mixed signs. Saddle point &mdash; no guaranteed minimum.</p>
        </div>
      </div>
      <pre>A = np.array([[<span class="nm">2</span>, <span class="nm">1</span>],
              [<span class="nm">1</span>, <span class="nm">3</span>]])
eigenvalues = np.linalg.eigvalsh(A)  <span class="out"># [1.38, 3.62] — both > 0 → PD</span>

<span class="cm"># Quick test: try Cholesky — if it works, it's PD</span>
np.linalg.cholesky(A)  <span class="out"># succeeds → PD ✓</span></pre>
      <div class="callout callout-blue">In optimization, the <strong>Hessian</strong> being PD at a point means you're at a <strong>local minimum</strong>. In kernel methods (SVM), the kernel matrix must be PSD.</div>
    </div>

    <!-- 36 COVARIANCE MATRIX -->
    <a class="section-anchor" id="covariance"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-purple">37</span><h3>Covariance Matrix</h3>
      </div>
      <p>The <strong>covariance matrix</strong> &Sigma; captures how features vary together. Element &Sigma;<sub>ij</sub> = Cov(x<sub>i</sub>, x<sub>j</sub>). The diagonal holds variances; off-diagonal holds covariances. It is always <strong>symmetric positive semi-definite</strong>.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;"><span class="cm"># X: (samples, features)</span>
X = np.array([[<span class="nm">1</span>,<span class="nm">2</span>],[<span class="nm">3</span>,<span class="nm">4</span>],[<span class="nm">5</span>,<span class="nm">6</span>],[<span class="nm">7</span>,<span class="nm">8</span>]])
cov = np.cov(X, rowvar=False)
<span class="kw">print</span>(cov)
<span class="out"># [[6.667, 6.667],</span>
<span class="out">#  [6.667, 6.667]]</span>

<span class="cm"># PCA starts here: eigenvectors of cov</span>
vals, vecs = np.linalg.eigh(cov)</pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Pure Python</div>
          <pre style="margin:0;"><span class="kw">def</span> mean(col):
    <span class="kw">return</span> sum(col) / len(col)

<span class="kw">def</span> cov_matrix(X):
    <span class="cm"># X: list of rows</span>
    n = len(X); d = len(X[<span class="nm">0</span>])
    means = [mean([X[i][j] <span class="kw">for</span> i <span class="kw">in</span> range(n)])
             <span class="kw">for</span> j <span class="kw">in</span> range(d)]
    C = [[<span class="nm">0</span>]*d <span class="kw">for</span> _ <span class="kw">in</span> range(d)]
    <span class="kw">for</span> a <span class="kw">in</span> range(d):
        <span class="kw">for</span> b <span class="kw">in</span> range(d):
            C[a][b] = sum(
              (X[i][a]-means[a])*(X[i][b]-means[b])
              <span class="kw">for</span> i <span class="kw">in</span> range(n)) / (n-<span class="nm">1</span>)
    <span class="kw">return</span> C</pre>
        </div>
      </div>
    </div>

    <!-- 37 SPECIAL MATRICES -->
    <a class="section-anchor" id="special-matrices"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-amber">38</span><h3>Special Matrices</h3>
      </div>
      <p>Certain matrix types appear so often in ML that recognizing them instantly saves time and prevents bugs.</p>
      <div class="grid-3" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Identity (I)</div>
          <p style="font-size:12px;">AI = IA = A. The &ldquo;1&rdquo; of matrices. <code>np.eye(n)</code></p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Diagonal</div>
          <p style="font-size:12px;">Non-zero only on diagonal. <code>np.diag(v)</code>. Scales each axis independently.</p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Symmetric</div>
          <p style="font-size:12px;">A = A<sup>T</sup>. Covariance, kernel, and Hessian matrices are symmetric.</p>
        </div>
      </div>
      <div class="grid-2" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Orthogonal</div>
          <p style="font-size:12px;">Q<sup>T</sup>Q = I. Preserves lengths/angles. Used in weight init for RNNs. <code>np.linalg.qr</code></p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Sparse</div>
          <p style="font-size:12px;">Mostly zeros. NLP bag-of-words, graph adjacency. <code>scipy.sparse</code> saves memory.</p>
        </div>
      </div>
      <pre><span class="cm"># NumPy special matrix constructors</span>
np.eye(<span class="nm">3</span>)          <span class="out"># 3×3 identity</span>
np.diag([<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>])   <span class="out"># 3×3 diagonal</span>
np.zeros((<span class="nm">3</span>,<span class="nm">4</span>))    <span class="out"># 3×4 zeros</span>
np.ones((<span class="nm">2</span>,<span class="nm">2</span>))     <span class="out"># 2×2 ones</span>

<span class="cm"># Sparse (scipy)</span>
<span class="kw">from</span> scipy <span class="kw">import</span> sparse
S = sparse.csr_matrix(np.eye(<span class="nm">1000</span>))  <span class="out"># stores only 1000 values, not 1M</span></pre>
    </div>

    <!-- 38 PROJECTION MATRICES -->
    <a class="section-anchor" id="projection"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-red">39</span><h3>Projection Matrices</h3>
      </div>
      <p>A <strong>projection matrix</strong> P maps a vector onto a subspace. It satisfies P&sup2; = P (applying it twice changes nothing). The <strong>least-squares projection</strong> is P = X(X<sup>T</sup>X)<sup>-1</sup>X<sup>T</sup>, which projects b onto the column space of X.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;"><span class="cm"># Project b onto the column space of A</span>
A = np.array([[<span class="nm">1</span>,<span class="nm">0</span>],[<span class="nm">0</span>,<span class="nm">1</span>],[<span class="nm">0</span>,<span class="nm">0</span>]])
P = A @ np.linalg.inv(A.T @ A) @ A.T

b = np.array([<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>])
proj_b = P @ b  <span class="out"># [1, 2, 0]</span>

<span class="cm"># Verify P² = P</span>
np.allclose(P @ P, P)  <span class="out"># True</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Why it matters in ML</div>
          <p style="font-size:13px;line-height:1.6;"><strong>Linear regression</strong> is a projection: &ycirc; = Xb projects the target vector onto the column space of the feature matrix. <strong>Attention</strong> in Transformers computes Q(K<sup>T</sup>K)<sup>-1</sup>K<sup>T</sup>V &mdash; a weighted projection.</p>
        </div>
      </div>
    </div>

    <!-- 39 MATRIX CALCULUS -->
    <a class="section-anchor" id="matrix-calculus"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-teal">40</span><h3>Matrix Calculus (Jacobians &amp; Hessians)</h3>
      </div>
      <p><strong>Matrix calculus</strong> extends derivatives to vectors and matrices. The <strong>Jacobian</strong> is the matrix of all first-order partial derivatives. The <strong>Hessian</strong> is the matrix of second-order derivatives &mdash; it tells you about curvature.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Jacobian J</div>
          <p style="font-size:12px;">J<sub>ij</sub> = &part;f<sub>i</sub>/&part;x<sub>j</sub>. Maps how each output changes w.r.t. each input. Shape: (outputs &times; inputs).</p>
        </div>
        <div style="background:var(--bg2);border-radius:var(--radius);padding:1rem;">
          <div style="font-size:13px;font-weight:600;margin-bottom:6px;">Hessian H</div>
          <p style="font-size:12px;">H<sub>ij</sub> = &part;&sup2;f/&part;x<sub>i</sub>&part;x<sub>j</sub>. Always symmetric. PD &rarr; convex &rarr; guaranteed minimum.</p>
        </div>
      </div>
      <pre><span class="cm"># Key gradient rules (used in backprop)</span>
<span class="cm"># d/dx (Ax)    = A</span>
<span class="cm"># d/dx (x^T A x) = (A + A^T) x  (= 2Ax if A symmetric)</span>
<span class="cm"># d/dA tr(AB)  = B^T</span>
<span class="cm"># d/dx ||Ax-b||² = 2 A^T(Ax - b)   ← linear regression gradient!</span>

<span class="cm"># PyTorch auto-computes Jacobians:</span>
<span class="cm"># loss.backward() builds the chain of Jacobians (backpropagation)</span></pre>
      <div class="callout callout-amber">Backpropagation is just the <strong>chain rule applied to Jacobians</strong>: &part;L/&part;W<sub>1</sub> = (&part;L/&part;z<sub>n</sub>) &times; (&part;z<sub>n</sub>/&part;z<sub>n-1</sub>) &times; &hellip; &times; (&part;z<sub>2</sub>/&part;W<sub>1</sub>). Each term is a Jacobian matrix.</div>
    </div>

    <!-- 40 EINSUM -->
    <a class="section-anchor" id="einsum"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-purple">41</span><h3>Einstein Summation (einsum)</h3>
      </div>
      <p><code>np.einsum</code> expresses any tensor operation in one line using Einstein notation. Repeated indices are summed over. It's the lingua franca of PyTorch/TensorFlow for complex tensor ops.</p>
      <pre><span class="cm"># Dot product</span>
np.einsum(<span class="st">'i,i->'</span>, a, b)           <span class="out"># same as a @ b</span>

<span class="cm"># Matrix multiply</span>
np.einsum(<span class="st">'ik,kj->ij'</span>, A, B)      <span class="out"># same as A @ B</span>

<span class="cm"># Batch matrix multiply</span>
np.einsum(<span class="st">'bik,bkj->bij'</span>, T1, T2) <span class="out"># per-batch matmul</span>

<span class="cm"># Trace</span>
np.einsum(<span class="st">'ii->'</span>, A)              <span class="out"># same as np.trace(A)</span>

<span class="cm"># Transpose</span>
np.einsum(<span class="st">'ij->ji'</span>, A)            <span class="out"># same as A.T</span>

<span class="cm"># Outer product</span>
np.einsum(<span class="st">'i,j->ij'</span>, a, b)        <span class="out"># same as np.outer(a,b)</span>

<span class="cm"># Attention: Q K^T (batched)</span>
np.einsum(<span class="st">'bhid,bhjd->bhij'</span>, Q, K) <span class="out"># multi-head attention scores</span></pre>
      <div class="callout callout-blue">Rule: letters that appear on <strong>both sides</strong> of <code>-></code> are kept; letters that appear only on the <strong>left</strong> are summed over. This single rule covers dot products, matmuls, traces, and more.</div>
    </div>

    <!-- 41 NULL & COLUMN SPACE -->
    <a class="section-anchor" id="null-col-space"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-green">42</span><h3>Null Space &amp; Column Space</h3>
      </div>
      <p>The <strong>column space</strong> of A is the set of all possible outputs Ax &mdash; it's the &ldquo;reach&rdquo; of A. The <strong>null space</strong> is the set of all x where Ax = 0 &mdash; inputs that A annihilates. Together: dim(col space) + dim(null space) = n (number of columns).</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy / SciPy</div>
          <pre style="margin:0;">A = np.array([[<span class="nm">1</span>,<span class="nm">2</span>,<span class="nm">3</span>],[<span class="nm">4</span>,<span class="nm">5</span>,<span class="nm">6</span>],[<span class="nm">7</span>,<span class="nm">8</span>,<span class="nm">9</span>]])

<span class="cm"># Column space basis (via SVD)</span>
U, S, VT = np.linalg.svd(A)
rank = np.sum(S > <span class="nm">1e-10</span>)
col_space = U[:, :rank]  <span class="out"># shape (3, 2)</span>

<span class="cm"># Null space</span>
<span class="kw">from</span> scipy.linalg <span class="kw">import</span> null_space
ns = null_space(A)  <span class="out"># shape (3, 1)</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Why it matters in ML</div>
          <p style="font-size:13px;line-height:1.6;"><strong>Column space</strong> = what your model can predict. If b isn't in the column space, you can only get the closest approximation (least squares).</p>
          <p style="font-size:13px;line-height:1.6;margin-top:8px;"><strong>Null space</strong> = parameter directions that don't affect output. A non-trivial null space means infinite solutions &mdash; your model is under-constrained.</p>
        </div>
      </div>
    </div>

    <!-- 42 CONDITION NUMBER -->
    <a class="section-anchor" id="condition-number"></a>
    <div class="section">
      <div style="display:flex;align-items:baseline;gap:0.75rem;margin-bottom:0.75rem;">
        <span class="badge badge-amber">43</span><h3>Condition Number</h3>
      </div>
      <p>The <strong>condition number</strong> &kappa;(A) = &sigma;<sub>max</sub>/&sigma;<sub>min</sub> measures how sensitive a matrix computation is to numerical errors. A high condition number means tiny input changes cause huge output changes &mdash; the matrix is <strong>ill-conditioned</strong>.</p>
      <div class="grid-2" style="margin:1rem 0;">
        <div>
          <div style="font-size:11px;color:var(--accent);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; With NumPy</div>
          <pre style="margin:0;"><span class="cm"># Well-conditioned</span>
A = np.array([[<span class="nm">1</span>,<span class="nm">0</span>],[<span class="nm">0</span>,<span class="nm">1</span>]])
np.linalg.cond(A)  <span class="out"># 1.0 (perfect)</span>

<span class="cm"># Ill-conditioned</span>
B = np.array([[<span class="nm">1</span>, <span class="nm">1</span>],
              [<span class="nm">1</span>, <span class="nm">1.0001</span>]])
np.linalg.cond(B)  <span class="out"># ~40000 (dangerous!)</span>

<span class="cm"># Rule of thumb:</span>
<span class="cm"># κ ~ 1     → stable</span>
<span class="cm"># κ ~ 10^k  → lose ~k digits of precision</span>
<span class="cm"># κ = ∞     → singular</span></pre>
        </div>
        <div>
          <div style="font-size:11px;color:var(--green);font-weight:500;letter-spacing:0.04em;text-transform:uppercase;margin-bottom:6px;">&#9654; Why it matters in ML</div>
          <p style="font-size:13px;line-height:1.6;"><strong>NaN gradients?</strong> Check if your weight matrices are ill-conditioned. Features with vastly different scales create ill-conditioned X<sup>T</sup>X.</p>
          <p style="font-size:13px;line-height:1.6;margin-top:8px;"><strong>Fix:</strong> Feature scaling (standardization), regularization (&lambda;I raises the smallest singular values), or using <code>np.linalg.lstsq</code> instead of <code>inv</code>.</p>
        </div>
      </div>
      <div class="callout callout-amber">If <code>np.linalg.cond(X.T @ X)</code> is large, <strong>do not</strong> compute <code>inv(X.T @ X)</code>. Use <code>np.linalg.lstsq(X, y)</code> or add regularization instead.</div>
    </div>
